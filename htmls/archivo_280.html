    A separate post-sort processing run is required only if you are using an asynchronous flow or a custom flow in Orchestrator. To complete separate post-sort processing, use your normal method of running the engine. For more information about running engine orchestration, see section 9 "Running engine orchestration" in OpenText Communications - Web Client User Help (CCWC250100-H-UGD)section 9 "Running engine orchestration" in OpenText Communications - Web Client User Help (CCWC250100-H-UGD).

    The engine uses the data in the sort data file(s) to produce final output according to your changes in the sort index data source and other communication settings.
