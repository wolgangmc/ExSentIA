8.3.2.4.7   Comparing similar content

    To help prevent duplicate content, Rationalization automatically compares text content within your migration set. For example, if content from one PDF or DOCX file is the same or similar to content in another file in the migration set, the Rationalization tool identifies those results on the Occurrences tab.

    Additionally, Rationalization can compare your migration set text content with existing DAS object content. The comparison looks at approved text in DAS components, clauses, text messages, and graphic messages and provides information about the percentage of matching content.

    Important

    The DAS object comparison functionality is available only if Solr is part of the OpenText Communications installation.

    Some points to keep in mind:

    • If the rationalization process finds similar content, you can then decide which instances, if any, you want to include as part of the migration set.

    • If rationalization identifies similar content in an approved Designer or Content Author clause, then the description includes a designation of Clause (CD) or Clause (CA), respectively.

    • The Occurrences tab provides a link when identical matches are found on multiple pages within the file or in another file that is part of the migration set.

    • The number of content similarities, based on tolerance level, is displayed in the Fragments panel in the Count column.

    To compare content:

    1. In the Fragments panel, select a fragment.

    2. In the Properties panel, click the Occurrences tab.

      Based on the Fragment rationalization tolerance setting, any duplicate content is shown with the percent match and assigned to one of the following groups:

      • Primary—Content from the migration set. If there are multiple instances of identical content in the migration set, all instances are included here.

      • Matches—Fragments or DAS objects that contain a percentage of matching content.

      • Other—Text content in the migration set with partial matches.

      When viewing image, variable, or rule fragments, they are combined into an All Occurrences group.

    3. Optional Expand a fragment to view its full content. Differences from the primary content are highlighted.

      You can also use the Comparison panel to view the variations in content:

      1. Select one or more of the non-primary fragments.

      2. Click Comparison in the Preview panel.

        The contents of all the selected fragments are shown, with differences from the primary content highlighted.

      3. Optional Click the Compare button to alternate between showing added and removed content compared to the primary fragments.

        Note

        You can keep the Compare button active to automatically compare fragments as they are selected from the Fragments panel.

    4. Optional If there is a large amount of matched content, you can pin specific instances so that they will appear directly under the primary instances for easier comparison. To pin content, expand the fragment and click the pin button .

      A Pinned group is created just under the Primary group containing the selected fragments.

    5. Choose the occurrences of the fragment that you want to include in the migration set, thus making them available for design use. You do not have to include the primary fragment, but at least one fragment is required.

      Note

      If an existing DAS object matches at 100%, then the DAS object is selected as the default rather than the migration set content.

    6. Click Save.
