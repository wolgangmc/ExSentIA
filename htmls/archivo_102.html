2.9   Autoscaling deployments

    To accommodate your organization's business requirements, you can set your OpenText Communications deployment to automatically scale the resources that are used to maintain consistent performance.

    You use the Horizontal Pod Autoscaler to automatically scale the number of pods in a deployment. You can set up scaling when you deploy OpenText Communications with the helm install command, or set it up in an existing deployment with the helm upgrade command.

    Deployments can be scaled based on the following properties:

    • CPU utilization—To scale deployments based on CPU utilization, Kubernetes metrics server must be installed, and all deployments to be scaled must have resource requests and resource limits set.

    • Custom metrics—To scale deployments based on custom metrics, Prometheus and Prometheus adapter must be installed. If scaling should be based on message queue length, you must also enable metrics for RabbitMQ.

    For detailed information about setting up autoscaling for your deployment, see OpenText Communications - Cloud Deployment Guide (CCMCE250100-IGD)OpenText Communications - Cloud Deployment Guide (CCMCE250100-IGD).
