3.13   Associating resources with domains

    By default, when you add or create a resource, it is associated with the domain in which it is created. After you approve a resource, you can associate it with other domains as needed, for example, if you created a theme in a development domain and wanted to test it in a test domain. You can view the currently associated domains for a selected resource on the Domains tab in the Properties panel.

    You can modify a resource in one domain without affecting other versions of that resource in the other domains that it is associated with. When you associate a resource with a domain, all of its linked resources are also associated with that domain. However, when you remove a domain association from a resource, it does not remove that domain association from its linked resources.

    Tip

    If you always associate all approved resources in your domain with other domains, then you can link the other domains to your domain. After linking, your domain automatically associates approved resources to the linked domains. For more information on linking domains, see Section 2.3.2.7 “Linking domains to auto-associate resources”.

    Associating a resource with a domain

    You can associate a resource with a domain if the resource, and all of its linked resources, are approved.

    Note

    Exception: You can associate an approved application package with a domain, whether or not its subpackages are approved. Likewise, you can associate an approved subpackage with a domain if at least one version of its associated application package is approved. For more about application package and subpackage resources, see Managing application packages and subpackages.

    To associate a resource with a domain:

    1. In the asset library panel, click the resource.

    2. In the Properties panel, click the Domains tab.

    3. In the Available domains list, in the row for the domain to associate, click Associate.

    4. In the dialog box that asks you to confirm the object’s association with the domain, do one of the following:

      • Select the Grant edit permissions with association check box (default) to make the object editable in the associated domain.

      • Clear the Grant edit permissions with association to make the object noneditable (read-only) in the associated domain.

    5. Click Apply.

    Changing permissions when associating a resource with a domain

    1. Associate an approved resource with a domain.

    2. In the Currently associated domains list, point to the domain that you associated with the resource and click the More actions button in that row.

    3. Point to Permissions and click one of the following options:

      • Read-only to make the resource noneditable in the associated domain.

      • Edit to make the resource editable in the associated domain.

    4. Click OK to confirm the permission change.

    Removing a domain association from a resource

    You can remove any domain association from a resource except its association to the current domain.

    Note

    • If a resource is associated with only one domain, you cannot remove that domain association from the resource. Also, if the resource is noneditable (read-only) in the current domain, you cannot remove domain associations from the resource.

    • If you have a data source that is associated with multiple domains, and that data source includes a migration set, if you remove the data source from one domain, that action will not automatically remove the associated migration set from that same domain.

    To remove a domain association from a resource:

    1. In the asset library panel, select the resource.

    2. In the Properties panel, click the Domains tab.

    3. In the Currently associated domains list, select a domain, and then click the More actions button in that row.

    4. Click Remove Association.

    5. Click Proceed to confirm the removal.
