9.4.2   Reprocessing a flow step

  • Communication asset library view
  • Design asset library view
  • Orchestration asset library view
  • Monitor view
  • Rationalization asset library view
  • Dashboards view

    Orchestration flow steps in a failed state can be reprocessed using the same flow instance used to run the step initially. This option might be helpful if a flow step has encountered a problem during processing. After a flow step is reprocessed, any subsequent flow steps are then processed.

    Note

    This option is available only for batch jobs and is restricted to failed output channels.

    To reprocess all the failed flow steps in a flow instance:

    1. In Monitor view, on the Flows tab, click the More actions button for the flow instance row containing the flow steps that you want to reprocess.

    2. Click Reprocess flow steps.

    3. Optional In the Reprocess options dialog box, select the failed flow steps that you want to reprocess.

    4. Click Reprocess.

    To reprocess a single failed flow step:

    1. In Monitor view, on the Flows tab, select the flow instance that contains the failed flow step that you want to reprocess.

    2. In the Properties panel, select the appropriate failed flow step.

    3. Click the More actions button for the flow instance row.

    4. Click Reprocess flow step.

    Note

    On the General tab of the Properties panel, reprocessed flow steps have “(reprocess)” appended to their names. In the flow instance graphic, the status of a flow step is also updated if it is reprocessed.
