2.2.5.4   Step 4: Allocate users, groups, or partitions to the product key in OTDS

  • Communication asset library view
  • Design asset library view
  • Orchestration asset library view
  • Monitor view
  • Rationalization asset library view
  • Dashboards view

    To allow users to access the Design and Production design tools (which include Design Manager and Designer for Design and Production, among others), you’ll need to allocate specific users to the product key. You can also allocate groups or partitions to allocate all of the users in those groups or partitions to the product key. After you allocate users in OTDS, you can then configure Design Manager to use OTDS credentials for user authentication and to allow users to upload resources to DAS.

    Depending on which features (design tools) are associated with the product key, the process of allocating users to a product key also grants them permission to utilize one of the Named User Seats (known as reserved seats in OTDS) for the web client design applications (Designer, Content Author, and Interactive). However, it does not automatically secure the seat until the user accesses a feature for the first time. Once a user is allocated to a seat, that seat stays reserved to that user unless revoked by an administrator in OTDS.

    Note

    Deleting a user does not automatically release the seat that is assigned to that user. A seat must be revoked in order for the seat to be available for reallocation.

    For specific information about how licensing works with OpenText Communications, see OpenText Communications - Containerized Deployment Licensing Guide (CCDPCD250100-ULI)OpenText Communications - Containerized Deployment Licensing Guide (CCDPCD250100-ULI). For information about OTDS users, groups, and partitions, or how to allocate or revoke reserved seats, see OpenText Directory Services - Installation and Administration Guide (OTDS-IWC)OpenText Directory Services - Installation and Administration Guide (OTDS-IWC).

    Important

    You can use only temporary or non-production product keys for Design and Production allocation.

    To allocate a user, group, or partition to the product key:

    1. In the OTDS web client, sign into the following URL as a tenant administrator user: 

      https://<host>:<optionalPort>/otdstenant/<tenant>/otds-admin/

    2. In the Setup menu, click Users & Groups or Partitions, depending on the allocation type.

    3. Locate the user, group, or partition that you want to allocate to the product key.

    4. In the Actions menu for each user, group, or partition, click Allocate to License.

    5. In the License list, select the product key.

    6. If you want to change which feature the user, group, or partition has permission to use, select the appropriate feature in the Counter list.

      Counters

      The counters that appear in the Counter list are determined by which features have been licensed.

      As a best practice, allocate counters individually to specific users or user groups.

      Important

      EXSTREAM is the primary (root) counter and, if selected, allocates the license key to all of the EXSTREAM counters in the license key for that user, group of users, or all of the users in a partition. OpenText recommends that you control license key access by allocating specific feature counters to groups or individual users instead of to the EXSTREAM root counter.

      Note

      For Communications version 24.4.0 and later, the product key file retains references to “EXSTREAM” as the product name. This is to allow any previously activated licenses to remain usable due to the product name change.

      For example, if the counter list includes EXSTREAM, EXSTREAM.COMMUNICATIONS_DESIGNER, EXSTREAM.CONTENT_AUTHOR, and EXSTREAM.EMPOWER, selecting EXSTREAM will automatically provide access to Designer, Content Author, and Interactive. If you select only EXSTREAM.CONTENT_AUTHOR, then access is granted to a Content Author seat only.

      If needed, the allocation process can be repeated to provide access to more than one counter.

      Important points regarding counters:

      • A Designer seat is required for access to Designer, Communication asset library view, Design asset library view, Orchestration asset library view, and Monitor view.

      • A Content Author seat is required for access to Content Author.

      • It is not required to allocate the EXSTREAM.MESSAGES and EXSTREAM.TRANSACTIONS counters.

    7. Click Allocate to License.

    8. If necessary, repeat step 6 through step 7 to allocate a Named User Seat for any additional features.

    9. Click Close.

    With the Design and Production user, group, or partition allocated to the product key, the Integration settings for OTDS in Design Manager can then be configured so that resources can be uploaded to DAS. For information about configuring Integration settings in Design Manager, see section 1.10.3 "Configuring OTDS connection settings" in OpenText Exstream Design and Production - Installation, Upgrade, and Administration Help (CCDPICU250100-H-IGD)section 1.10.3 "Configuring OTDS connection settings" in OpenText Exstream Design and Production - Installation, Upgrade, and Administration Help (CCDPICU250100-H-IGD).
