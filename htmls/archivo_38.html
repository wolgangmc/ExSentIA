2.2.3   Creating tenants with a bootstrap configuration

    In the OpenText Communications Helm chart, the values.yaml file contains the following default configuration: 

    Example: 

    bootstrap:
    enabled: true

    To use a Helm upgrade command to set up new tenants in an OpenText Communications deployment, you specify a custom bootstrap configuration as shown in the following parameter block: 

    Example: 

    bootstrap:
    enabled: true
    config:
        <Enter bootstrap configuration parameters>

    If you are using manual schema management for your OpenText Communications databases, then you will need to first set up tenant schemas directly in the database for each of the OpenText Communications services. You can then specify your bootstrap parameters in the values.yaml file that you used to initially deploy OpenText Communications or you can create a custom bootstrap file. Regardless of which file you use, you must specify the configuration parameters in the correct format. For detailed information about creating and formatting your bootstrap file, see Bootstrap configuration parameters.

    Bootstrapping tenant configurations provides a convenient way to add multiple tenants to an existing deployment using a single Helm command. Bootstrap parameters also let you customize and select specific parts of the tenant configuration process that you want to automate. For example, you could use this process to create multiple tenants, configure domains in those tenants, add roles, and specify OTDS group assignments with a single command. Or, you could choose to create only tenants and let individual tenant administrators manually complete role and domain configurations for each tenant.

    Note

    You can also use a custom bootstrap configuration with the helm install command to create and configure individual tenants during the initial deployment. For more information about using bootstrap configurations during deployment, see OpenText Communications - Cloud Deployment Guide (CCMCE250100-IGD)OpenText Communications - Cloud Deployment Guide (CCMCE250100-IGD).

    If you enable the default bootstrap configuration and you are using manual schema management, you must specify the database schemas in the bootstrap configuration to create the sample tenant.
