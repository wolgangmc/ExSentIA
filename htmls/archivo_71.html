2.3.3.6   Configuring advanced workflow settings for a domain

  • Communication asset library view
  • Design asset library view
  • Orchestration asset library view
  • Monitor view
  • Rationalization asset library view
  • Dashboards view

    In a domain that has advanced workflow configured, when a user submits a draft for approval, OpenText Communications routes it to approver groups in an approval role set that matches the resource type in the draft. Approver groups at the first level can start their reviews when the workflow state button for the draft shows the submitted for approval symbol . Meanwhile, higher-level approver groups—and other users in the domain who can access the draft—can check the workflow status of the draft by clicking the workflow state button when it shows the pending approval symbol .

    Note

    If none of the resource types in a submitted draft are associated with an approval role set, the workflow type for that submitted draft switches to the standard workflow.

    OpenText Communications can send approval request notifications to approver groups at each level when required if the following conditions are met:

    • The tenant is configured to send email notifications for workflow changes to resource types in the selected domain.

    • The draft includes a resource type that triggers notifications when its workflow state changes.

    • The default approver distribution list in the notification settings matches the approver distribution list for the level in the advanced workflow settings.

    For more information about setting up notifications for resource type changes in a domain, see Section 2.4.4 “Setting up email notifications for workflow changes”.

    The following table summarizes the actions that occur if you submit a draft for approval in a domain with advanced workflow:

    Advanced workflow step

    Action

    1

    A user submits a draft resource for approval.

    2

    (If configured) OpenText Communications emails an approval request to the approver distribution list for the level 1 approver groups in an approval role set that matches the draft’s resource type. The email includes a link to the submitted draft resource.

    3

    If all level 1 approver groups approve the draft, the workflow completes unless the set has approver groups at level 2.

    4

    (If configured) OpenText Communications emails an approval request with the draft link to the distribution list for the level 2 groups.

    5

    If all level 2 groups approve the draft, the workflow completes unless the set has groups at level 3.

    The steps above repeat up to the highest level in the approval role set.

    The approvers in groups on the same level do parallel approvals, however they all must approve before the next workflow step can begin. If, at any level, an approver rejects the draft, OpenText Communications sends a rejection notification (if configured) to the distribution lists for all levels in the set. Later, if the author resubmits the draft for approval, the advanced workflow starts over from the beginning.

    After the highest-level groups approve the draft, the workflow completes and OpenText Communications moves the submitted draft to the approved state.

    To add approval role sets to a domain:

    1. In Tenant management view, access the workflow type settings for a domain.

    2. In the Workflow tab, click Advanced in the Workflow type list.

    3. In the Workflow tab, click the add button for Approval role sets.

    4. Perform the following tasks to finish configuring the approval role set:

      • Assign resource types to the approval role set.

      • Configure approver groups in the approval role set.

      • Configure levels for approver groups in the approval role set.

      • Assign an approval distribution list to each level in an approval role set.

    5. Optional Repeat steps 3 through 4 to add more approval role sets to the domain.

    6. Click Save.

    To assign resource types to an approval role set

    1. In Tenant management view, access the workflow type settings for a domain.

    2. In the Workflow tab, click Advanced in the Workflow type list,

    3. If no approval role sets are listed after Approval role sets, add an approval role set.

    4. In the Resource type list for an approval role set, select the check box for each resource type that you want to assign to the set.

      The approver groups in the set can approve only the assigned resource types.

      Important

      • A resource type can be assigned to only one approval role set. If you assign all resource types to one set, none are available to select for other sets.

      • If no resources are selected in the Resource type list, then the approval role set applies to all resource types. Also, the plus button to add new approval role sets is not available.

    5. Click Save.

    To configure approver groups in an approval role set

    1. In Tenant management view, access the workflow type settings for a domain.

    2. In the Workflow tab, in the Workflow type list, click Advanced.

    3. If no approval role sets are listed after Approval role sets, add an approval role set.

    4. In the Approval role sets list, click the Edit button for an approval role set.

    5. In the Approver groups dialog box, add approver groups to the workflow or remove approver groups from the workflow as follows:

      Note

      Approver groups are listed by role in the Approver groups dialog box. Each role identifies one or more approver groups (OTDS groups with Approver access) in the domain.

      For more information about configuring role-based permissions for OTDS groups in a domain, see Section 2.3.1 “Configuring roles and permissions”.

      • To add approver groups to the workflow:

        In the Available approver groups list, click the add button for a role to move OTDS groups with that role to the Applied roles list.

      • To remove approver groups from the workflow:

        In the Applied roles list, click the remove button for a role to move groups with that role to the Available approver groups list.

    6. Click Apply.

    To configure levels for approver groups in an approval role set.

    1. In Tenant management view, access the workflow type settings for a domain.

    2. In the Workflow tab, in the Workflow type list, click Advanced.

    3. If no approval role sets are listed after Approval role sets, add an approval role set, and then add available approver groups in the set to the workflow.

    4. If no approver group roles are listed in the Roles column for the approval role set, edit the set to add available approver groups in the set to the workflow.

      Note

      Each role in the Roles column is assigned to one or more approver groups (OTDS groups with Approver access) in the domain.

      For more information about configuring role-based permissions for OTDS groups in a domain, see Section 2.3.1 “Configuring roles and permissions”.

    5. In the Levels column for the approval role set, click the list in a row, and then click a level number. The level number that you select applies to all approval groups that are assigned to the role in the Roles column.

      You can assign the same level number or a different level number in each row. All approver groups in the set who have the same role and level number must approve the draft before the workflow can finish or move to the next level.

    6. Click Save.

    To assign an approval distribution list to a level in an approval role set

    1. In Tenant management view, access the workflow type settings for a domain.

    2. In the Workflow tab, in the Workflow type list, click Advanced.

    3. If no approval role sets are listed after Approval role sets, add an approval role set, and then add available approver groups in the set to the workflow.

    4. Optional In the Approver distribution list column, in the row for an approver group level, enter the email distribution list for the approver groups at that level.

    5. If the tenant is configured to send email notifications for workflow changes—and when the approver groups at the level that you are configuring can review the draft—OpenText Communications sends an approval request to the distribution list for that level. The email includes a link to the submitted draft.

      The address for the distribution list is not validated or maintained in DAS; it must exist in your mail server. If the distribution list is incorrect or does not exist, then the approval request will not be sent.

      Note

      You can enter one email address only in the Approver distribution list box. If you need to add recipients, add their email addresses to the specified distribution list on the SMTP server.

    6. Click Save.
