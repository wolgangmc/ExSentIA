2.4.6   Creating category tags

  • Communication asset library view
  • Design asset library view
  • Orchestration asset library view
  • Monitor view
  • Rationalization asset library view
  • Dashboards view

    Categories are a convenient way to tag communications so that you can easily search for them in the DAS asset library. You must be a tenant administrator to add resource categories to a tenant, which are then available to users to apply to Designer and Content Author resources in all of the domains in that tenant.

    Users can apply category tags in the following views: 

    • Communication launcher view—Lets users apply category tags when creating new communications

    • Design asset library view—Lets users apply category tags to existing resources

    • Communication Composer in Designer—Lets users apply category tags to new and existing resources

    • Communication view and Authoring view in Content Author—Lets users apply category tags to new and existing resources

    If a user's role has the required permissions for categories, then that user can also create new category tags in these views. For information about applying tags to resources in Designer and Content Author see the following documentation:

    • OpenText Communications - Designer User Help (CCCD250100-H-UGD)OpenText Communications - Designer User Help (CCCD250100-H-UGD)

    • OpenText Communications - Content Author User Help (CCCA250100-H-UGD)OpenText Communications - Content Author User Help (CCCA250100-H-UGD)

    Note

    You must create at least one category tag in Tenant management view to enable users to apply tags or create additional tags in other views.

    You can use category tags in API calls to search for and filter resources in the DAS asset library. For more information about the Design API, see the developer documentation at the following URL:

    https://host:<optionalPort>/design/swagger-ui/

    Creating a new category tag

    1. In the Categories panel in Tenant management view, click Add.

    2. In the Properties panel, specify a name for the new category, and then click Create.

    Assigning category creation permissions to a role

    1. In the Roles panel in Tenant management view, select a role.

    2. In the Properties panel, click the Permissions tab.

    3. In the Web client access area, select the Access check box for Categories in the Type list.

    4. Click Save.

    Deleting a category tag

    1. In the Categories panel in Tenant management view, select the category tag that you want to delete..

    2. In the Properties panel, click Delete.

      If the tag was applied to one or more resources, then that tag reference is removed from those resources.
