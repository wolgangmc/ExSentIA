    To migrate OpenText Communications resources from a tenant in the OpenText Communications 16.x platform to a containerized OpenText Communications 25.1 deployment, you must first create CAS export packages using either the migration tools for the OpenText Communications platform or the transport option in WorkShop. You can then use the Import option to import resources from that exported package into the destination DAS asset library.

    For detailed information about exporting CAS resources, see the following documentation on My Support for your OpenText Communications platform version:

    • OpenText Communications Migration Tools User Guide (CCM-DMC)

    • OpenText Communications Workshop User Guide (CCMWEBRMG-UGD)

    Note

    For information about OpenText product compatibility, see the OpenText compatibly information provided in the release notes for the product. Release notes are available on My Support.

    Supported export package formats

    You can import resources from the following types of CAS export packages:

    • Release package—Contains all the resources that are linked to a specific commit label of a Communications Builder project release, that is, the Communications Builder project, the project export file, and all of its dependent WorkShop resources. Release packages are created in WorkShop or using the migration tools.

      OpenText strongly recommends using this type of export package to migrate Content Author or Designer communications.

      Note

      Data files that are referenced in Communications Builder projects through OpenText Communications engine switches are not included in the exported release packages. These files must be manually uploaded into the DAS asset library.

    • Resource package—Contains resources that are selected from the default Resources view or Templates view in WorkShop, such as themes, templates, and images. Resource packages are created in WorkShop. When you select a resource to create a resource package, the selected resource and all of its dependent resources are packaged together. However, a resource package cannot include Communications Builder projects.

      This type of export package is a convenient way to migrate WorkShop resources that are related to a Communications Builder project, and were updated after you created a release package to export that project.

    Considerations when migrating CAS resources

    When you migrate CAS resources, keep the following considerations in mind: 

    • Database support—You can export resources from all databases that are supported for the OpenText Communications platform to all databases that are supported in OpenText Communications 25.1. For information about supported databases for OpenText Communications, see the OpenText Communications Release Notes .

    • Workflow status of versioned resources—The export package maintains the workflow status of your versioned resources. During import, the workflow status is retained for all resources except for templates and design packs, which are automatically approved. OpenText recommends that you approve all resources before you create a CAS export package.

    • User permissions for locked resources—If an export package contains a resource that is locked, then the import process breaks the lock and imports the resource in an unlocked state so that users in the target tenant can edit it.

    • Unsupported resource types—Resources types that are not supported in DAS are not imported, even if they exist in the export package, and reference links for the unsupported resources are removed during import.

    Resource behavior after migration

    You can migrate only those resource types that are supported in OpenText Communications 25.1. The following table lists the CAS resource types and their corresponding DAS resource types after migration:

    Table 6.1: Behavior of migrated CAS resources

    CAS resource Corresponding DAS resource Import behavior
    Communications Builder Project -

    Communications Builder projects are ignored during import. To produce output from an imported communication, you must create new orchestration flows for the OpenText Communications engine. For more information, see Adding orchestration flow resources
    OpenText Communications Application Communication (CD) Imported as-is.
    OpenText Communications Application Manifest - Manifests are not available as a separate resource in the DAS asset library. You can view the manifest for a package file from the package file properties.
    OpenText Communications Application Package Package Imported as-is.
    OpenText Communications Component Component Imported as-is.
    OpenText Communications Data Source Data source Imported as-is.
    OpenText Communications Document Document Imported as-is.
    OpenText Communications Email Email Imported as-is.
    OpenText Communications Page Page Imported as-is.
    OpenText Communications Variable Variable Imported as-is.
    OpenText Communications Web Design Web design Imported as-is.
    Clause Clause Imported as-is.
    Design Pack Design pack

    The import process automatically approves design packs that are not already in an approved state.

    Flow model -

    CAS flow models are ignored during import. However, you can create new flow models in the DAS asset library. For more information, see Adding orchestration flow resources.

    Graphic Message Graphic message Imported as-is.
    Image Image Imported as-is.
    Orchestration Server Settings - Orchestration server settings are ignored during import. Orchestration settings in OpenText Communications 25.1 are defined using flow models and flow contexts.
    Section Section Imported as-is.
    Sample file Sample file Imported as-is.
    Simulation - Simulations are ignored during import. Content Author and Designer directly use sample files for generating previews. The engine run definitions for these previews are generated from the information in the application package and manifest.
    Template Template

    The import process imports and automatically approves only the following template types:

    • OpenText Communications templates (based on Design Manager applications)

    • Designer templates

    StoryTeller templates and templates for the template engine are ignored during import.

    Text Message Text message Imported as-is.
    Theme Communication (CA)

    Only Content Author themes are migrated. StoryBoard themes that exist in the export package are ignored during import.

    Content Author communications that were created in versions prior to 16.4 Update 2 are not supported in OpenText Communications 25.1.
