8.3.2.2.3   Rationalizing migration set content

    After you have created a migration set, you must rationalize the content to create the fragments that can be used in Designer and Content Author.

    To rationalize a migration set:

    1. In the Migration sets panel in Rationalization asset library view, select a migration set.

    2. In the Properties panel, perform the following tasks:

      1. Optional In the Current rationalization set tolerance area, use the slide or enter a value from 50 to 100, and then click Save. This value determines how alike that different fragments must be before Rationalization considers them to be a match—a higher number requires them to be more alike, and a lower number requires them to be less alike.

        The default tolerance value is 50. Values below 50 are not allowed.

      2. Click Run to initiate the rationalization process.

    In the Migration sets panel, the indicator in the Status column changes to show the current status of the rationalization process. After the rationalization process is complete, you can open the migration set in the Rationalization user interface to identify and edit any content fragments.

    For information about how to work with rationalized content, see Section 8.3.2.4 “Working in Rationalization”.
