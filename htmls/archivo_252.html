    Migration sets include a tolerance value at the time of creation that determines the threshold used by Rationalization to recognize matches within text content. Based on that value, the Rationalization tool searches for all instances of text content that are part of your migration set, and then groups like text fragments together into a single instance.

    Tolerance values range from 50 to 100, with higher numbers indicating a higher threshold for a match. The tolerance level for the entire migration set can be adjusted even after the rationalization process has been run. To do this, simply adjust the tolerance value in the Properties panel of the migration set, click Save, and then click Run again.

    You can also update the tolerance value on the properties for each text fragment in Rationalization.

    When a text fragment is inserted into a Designer or Content Author communication, any formatting and text characteristics—such as font type, size, color, and styling, as well as carriage returns—are preserved if possible. If the text fragment includes fonts or styling that are not part of an approved DAS font, the content is converted to use the default font.

    To work with text fragments in Rationalization:

    1. In the Fragments panel, open a migration set.

    2. Click the Text fragments button .

    3. Select a text fragment from the list. The fragment that you select is highlighted in the Preview panel.

    4. In the Properties panel for the text fragment, adjust the properties as needed:

      • On the General tab, the following information is available:

        • Name—Update the name to help differentiate the text fragment when Designer and Content Author users add it to content.

          Names do not support the following special characters: & / : * ! ? > < | + # %

        • Description—Optionally, add a description for the text fragment.

        • Fragment type—If the selected fragment is the same for all occurrences, then this shows the value Static. If there is a dynamic component to any of the occurrences, then the value is Dynamic.

      • On the Occurrences tab, you can see all of the different instances of the text fragment that the Rationalization tool was able to identify across all of the text content in the migration set and DAS. Matches are based on the tolerance value and are grouped together into this single instance. Tolerance values range from 50 to 100, with higher numbers indicating a higher threshold for a match. You can use the slider or enter a value in the field to change the tolerance value.

        This tab also indicates when identical matches are found on multiple pages within the file or in another file that is part of the migration set. For example, you might see a message saying, “Identical text fragment pulled from 2 pages” or “Identical text fragment pulled from 7 pages within 7 files”. If you click the underlined link, a dialog appears that lets you select which instance you want to review.

    5. Click Save to save your changes, or click Ignore to hide the text fragment from Designer and Content Author users.

    To combine multiple text fragments into a single fragment:

    1. In the Fragments panel, press and hold the CTRL key while selecting the text fragments you want to combine.

    2. In the Properties panel, the selected fragments appear in the Combined fragments section. You can remove an individual fragment by clicking the X next to its name.

    3. Specify a name and, optionally, a description for the combined text fragment.

      Names do not support the following special characters: & / : * ! ? > < | + # %

    4. Click Save to save your changes, or click Ignore to hide the text fragment from Designer and Content Author users.

      The combined fragment is added to the list in the Fragments panel. The fragments that make up the combined fragment remain available individually.
