5.3   Creating design layers

  • Communication asset library view
  • Design asset library view
  • Orchestration asset library view
  • Monitor view
  • Rationalization asset library view
  • Dashboards view

    Design layers allow you to view static design components as you work on other objects within content from the Design view in Designer. Depending on the type of design layer that you use, content on a design layer can either appear in the final output, or you can suppress them so that the content is only visible during the design phase. For example, you might need a representation of a letterhead, print margin marks, or pre-printed form areas. These design elements can be useful and instruct design choices, but should not be visible in the output. Alternatively, if you deliver a design to multiple outputs, you can use a design layer to include content that should appear only for a certain output types. For example, you might specify that barcodes should be included only in printed output, and not in electronic outputs.

    Tip

    Keep in mind that design layers are not the same as design layouts or design variations.

    • A design layer allows you to create a named grouping of design objects that can be kept static as you create other content within a complex design.

    • A design layout is a pre-defined set of page standards that allow companies to enforce a set of corporate standards across page designs, such as enforcing that designs use the same page size, style sheets, and colors. See section 13.1.1 "Applying a Designer design layout" in OpenText Communications - Designer User Help (CCCD250100-H-UGD)section 13.1.1 "Applying a Designer design layout" in OpenText Communications - Designer User Help (CCCD250100-H-UGD).

    • A design variation is a simpler alternative to complicated targeting rules that determine which content goes to which customers. See section 25 "Design variations" in OpenText Communications - Designer User Help (CCCD250100-H-UGD)section 25 "Design variations" in OpenText Communications - Designer User Help (CCCD250100-H-UGD).

    To add a design layer to the asset library

    1. In the Design asset library, click the Common tab.

    2. Click Add > Design layer.

    3. In the Properties panel, enter a name and, optionally, a description for the design layer.

      Names do not support the following special characters: & / : * ! ? > < | + # %

    4. From the Use list, select when content of this design layer should be visible:

      • Always visible – The design layer can appear in any output.

      • Design only – The design layer is visible only in design view and is not visible in any output.

      • Output inclusion – The design layer can appear in one or more specified outputs. To specify affected outputs, in the Outputs area click Add, select outputs as needed, and then click OK.

      • Output exclusion – The design layer is hidden from one or more specified outputs. To specify affected outputs, in the Outputs area click Add, select outputs as needed, and then click OK.

    5. Click Create.

    6. Optional After you have created the design layer, you can define the following common object settings:

      • Domains – For information about defining the domains to which a resource is assigned, see Section 3.13 “Associating resources with domains”.

      • Workflow – For information about managing an approval workflow, see Section 3.6 “Managing approval workflows”.

      • References – For information about viewing the linked resources connected to a selected resource, see Section 3.3 “Viewing resources in asset library views”.

    After you have created a design layer you can access the design layer from the Design view in Designer.
