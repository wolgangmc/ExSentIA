    The basic process for working with content in Rationalization is as follows:

    1. Select external content

      Select the files that contain content that you want to use in communications designs. For example, select one of the following file types:

      • PDF output files that were generated using a different design tool.

      • DOCX files that were created in Microsoft Word.

      • StreamServe files that contain static text content and embedded text fragments in the form of StoryTeller designs. If those designs have a template that has been exposed to StoryBoard, then those templates can be used with Rationalization.

    2. Create a ZIP file

      Create a ZIP file of the PDF, DOCX, or StreamServe content. For PDF and DOCX, you can optionally include any XML driver files that were used to create the PDF output or DOCX.

    3. Create a migration set

      In Rationalization asset library view, create a migration set that contains one or more of the ZIP files.

    4. Open the migration set

      With a migration set in place, you can open it in OpenText Communications Rationalization from the OpenText Communications web client.

    5. Update properties and combine fragments, as needed

      In the Rationalization user interface, you can organize and update text fragments, image fragments, variables, and rules.

      Note

      Rules are only imported as part of a StreamServe template migration set.

      You can also group fragments, as well as compare them to any existing, approved DAS content.

    6. Save your changes

      After you have made any needed changes to your content, click Save in the Properties panel.

    7. Link the migration set to a data source

      Before the migration set can be used with a communication that is based on a communication set, the migration set must be associated with a driver data source within the communication set.

      In Designer, if your communication is based on a design pack, you can link the migration set directly to the communication in the Properties panel of the communication when it is open in the Communication Composer .

    8. Review your updates in Designer and Content Author

      After you have finished working with the content in your migration set, you should test it to make sure that everything appears as expected in Designer and Content Author.

    You can repeat this process if you want to make any adjustments to the content.
