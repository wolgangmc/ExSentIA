3.6   Managing approval workflows

    The approval workflow is the progression of a resource from initial version to final approval. Using the standard, simple, or advanced workflow in OpenText Communications helps to ensure that new and updated content is properly reviewed for accuracy and conforms to company standards.

    For more information about the standard, simple, and advanced workflow types, see Section 2.3.3 “Configuring domain workflows”.

    Keep the following in mind when approving resources:

    • All child resources must be approved before a parent resource. For a communication, this includes the communication and all of its associated resources, such as images that are used in designs and the sample files that are referenced by the communication. If you try to approve a resource that references other unapproved resource, you receive a message indicating that you must first approve all of the linked resources before you can approve that resource.

    • You can only approve a new design pack if it contains at least all the attributes and tags in approved variations of the communications that use the design pack. If you try to approve a design pack that is missing attributes or tags, a list of affected variations opens and prompts you to remove the attributes or tags before resubmitting the design pack for approval. For more information about using variations in a communication, see OpenText Communications - Designer User Help (CCCD250100-H-UGD)OpenText Communications - Designer User Help (CCCD250100-H-UGD).

      Note

      Alternatively, you can use Design Manager to add the missing attributes or tags to the design pack and then re-upload the design pack to DAS. For more information about uploading design packs, see section 2.5 "Packaging and uploading applications" in OpenText Exstream Design and Production - Design and Import Help (CCDPDI250100-H-UGD)section 2.5 "Packaging and uploading applications" in OpenText Exstream Design and Production - Design and Import Help (CCDPDI250100-H-UGD).

    • You can change the workflow state of a single resource or multiple resources at the same time. If you select multiple resources, only those whose workflow state matches that of the resource you actively select to move are changed.

      Note

      The following exceptions apply when you change the workflow state for multiple resources:

      • Selecting the Discard draft option discards the draft of the selected resource only.

      • In the advanced workflow, the resource types for all selected resources must be assigned to the approval role set. For more information about assigning resource types to an approval role set, see Section 2.3.3.6 “Configuring advanced workflow settings for a domain”.

    Moving through the approval process

    As you move through the approval process, keep in mind that resources are only editable in the draft state. After a resource is approved, a new draft of the resource must be created before it can be edited. Likewise, if a resource is rejected, it must be moved to the draft state before it can be edited.

    To move resources through the approval process:

    1. On the web client home page, click the toolbar button for the asset library view that contains the resources that you want to move to the next workflow state.

    2. In the asset library panel, select the check box for each resource that you want to move to the next state.

      If you are moving only one resource to the next state, you do not need to select its check box.

      Note

      You can only select resources on the active page. Selecting the check box in the heading row selects all resources on the page. Navigating away from the page clears your selections.

    3. Click the workflow state button for the selected resource, or click the button for one of the selected resources.

      If you selected multiple resources, clicking the workflow button for one of them will change the workflow state of the others only if they are currently in the same state.

      To see the workflow history for a selected resource, in the Properties panel, click the Workflow tab.

    Checking the approval status for a draft in an advanced workflow

    1. In a domain with an advanced workflow, if the workflow state button for a submitted resource shows the pending approval symbol , click the button, and then click Workflow status.

    2. In the Advanced Workflow dialog box, in the Approval column, review the mix of approved symbols to pending approval symbols for the approver groups in the active level.

    3. Click Close.

      Note

      • For more information about approvers and approver groups, see Section 2.3.3.3 “Approvers and approver groups in workflows”.

      • For more information about configuring an advanced workflow with multiple levels of approver groups, see Section 2.3.3.6 “Configuring advanced workflow settings for a domain”.
