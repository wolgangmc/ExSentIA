8.3.1.1   Converted vs. unconverted HTML output

    If your communication produces HTML output, you have the option to place it into HTML or HTML Email output, or convert the imported HTML to OpenText Communications format.

    After you have created an HTML placeholder variable, you can specify whether the imported HTML is converted in the variable properties panel, on the General tab. When HTML is selected from the Placeholder type list, the Do not convert HTML check box becomes active. The check box is selected by default, but this setting affects only HTML output. The engine ignores it for all other output types. If you clear the Do not convert HTML check box, then the imported HTML content is converted.

    In most cases, you will leave this check box selected, so that the imported HTML is not converted in the HTML output. This way, the imported HTML is displayed in the browser according to the styles applied inline within the HTML. If you do not convert the imported HTML, then the engine includes a comment in the HTML output to identify the imported, unconverted HTML as pre-composed content.

    If you decide to convert the imported HTML to OpenText Communications format, then the resulting output will be limited to only those formats that are supported within OpenText Communications. As a result, you might see some visual differences between the imported HTML and HTML output. You might also see differences in the text alignment due to the way that the engine calculates tab, margin, and padding sizes. More information on expected differences between converted and unconverted HTML in HTML output for various design elements are outlined in this section.

    If you produce accessible HTML output, you should also consider which design objects OpenText Communications supports for accessibility, and how the engine creates the HTML structures in HTML and HTML Email output.
