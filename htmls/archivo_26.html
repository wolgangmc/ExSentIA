    The OpenText Communications web client is the central user interface for interacting with DAS. From the web client, users can define tenants in their OpenText Communications deployment, upload OpenText Communications engines, manage tenants, create communications, and manage resources in their DAS asset library. The web client provides access to both the system tenant as well as other regular tenants in the OpenText Communications deployment. Use the following URL to access a tenant in the web client:

    https://<host>:<optionalPort>/design?tenant=<tenantName>

    The web client includes several different views to enable users to perform various tasks. A user's role and the permissions that are assigned to that role determines their level of access in the OpenText Communications environment. If you have questions about your access privileges, talk to your system administrator.

    System management view

    When a user accesses the system tenant, the web client opens to the System management view. The system tenant provides a purely administrative environment; it cannot be used for creating and testing communications. To access the OpenText Communications system tenant, users must be members of the otadmins group in the OTDS system tenant.

    The System management view contains the following panels:

    • Tenants and services—Lets users create and manage tenants in the OpenText Communications deployment

    • Engines—Lets users upload and manage OpenText Communications engines

    • Scheduled tasks—Lets users configure global scheduled tasks for tenants

    Tenant-specific views

    When a user accesses a regular tenant, the web client opens to the Tenant View. This view lists the domains that are available to the current user in that tenant.

    To filter domains by name:

    1. Enter the domain name or part of the name in the Search box in the top right corner of the Tenant View.

    2. Optional To perform a case-sensitive search, click the drop-down button, and then select the Case sensitive search check box. A plus sign appears in the Search box to indicate a case-sensitive search.

    3. Click the button.

    In Tenant View, tenant administrators can click the Tenant management button to open the Tenant management view.

    The Tenant management view contains the following panels:

    • Roles—Lets users create roles and assign access permissions to each role

    • Domains—Lets users create and manage domains and assign user groups to each domain

    • Configurations—Lets users create and manage application configurations

    • Categories—Lets users create and manage categories

    • Notifications—Lets users configure email notification settings

    Domain-specific views

    To open a domain to work in, click a domain card in the Tenant View. The web client switches to the Communication launcher view. This view is the domain home page for all domains. To switch to a different domain, click the Current domain list in the top right corner of this view, and select a domain. This list displays only those domains that are available to the current user.

    From Communication launcher view, authorized OpenText Communications users can launch the following applications:

    • Designer

    • Content Author

    In any domain-specific view, the web client contains an application toolbar that provides access to additional views to perform various actions in the current domain. The following table describes the domain-specific views and how to access them.

    Table 1.1: Domain-specific views in the OpenText Communications web client

    View Description

    Communication launcher view

    Contains the launcher panels for Designer and Content Author.

    To access the view, click the Communication launcher button .

    Communication asset library view

    Contains the asset library panels that are used to create and manage the DAS resources that you can use to set up new communications, such as data sources, variables, outputs, and so on.

    To access the view, click the Communication asset library button .

    From this view, you can also access Mapping view, that lets you map data sources and create and manage variables. To access Mapping view, open a data source resource from the Data panel in Communication asset library view.

    Design asset library view

    Contains the asset library panels that are used to manage Content Author and Designer resources in DAS, as well as other common design resources, such as images, fonts, barcodes and so on.

    To access the view, click the Design asset library button .

    Orchestration asset library view

    Contains the asset library panels that are used to manage the DAS resources that are required for engine orchestration.

    To access the view, click the Orchestration asset library button .

    Monitor view

    Contains the monitoring panel that lets you monitor engine orchestration jobs.

    To access the view, click the Monitoring button .

    For more information, see section 9 "Running engine orchestration" in OpenText Communications - Web Client User Help (CCWC250100-H-UGD)section 9 "Running engine orchestration" in OpenText Communications - Web Client User Help (CCWC250100-H-UGD).

    Rationalization asset library view

    Contains the asset library panel that is used to create and manage migration sets for Rationalization.

    To access the view, click the Rationalization button .

    From this view, users can also access Rationalization to create content fragments from migration sets that can then be used in Designer communications. To access Rationalization, open a migration set resource from the Migration sets panel in Rationalization asset library view.

    Dashboards view

    Provides the ability to create dashboards that can display information such as orchestration job status and licensing information.

    To access the view, click the Dashboards button .


    In addition to these views, the Import button provides access to the Import dialog box to import resources into the current DAS asset library.

    Tenant administrators can also click the Tenant management button to open Tenant management view from the application toolbar.
