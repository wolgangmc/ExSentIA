8.3.2   Rationalizing external content

    Rationalization is a tool that allows you to import external content in the form of ZIP files that contain DOCX files, PDF files, or StreamServe templates. These files are processed to identify, group, and separate blocks of content within those files. You can then use the Rationalization tool to consolidate content by identifying duplicate or similar content.

    If you include XML driver files in the same ZIP file as your PDF or DOCX files, Rationalization also detects variables in your rationalized content.

    After the content is rationalized, it can be made available for use in Designer and Content Author communications.

    Rationalization is strictly a tool to identify and organize content, not a tool to edit content. As a result, Rationalization does not provide options to format content; instead, users in Designer and Content Author will format and customize the content that you make available to them. Because Rationalization is uniquely a design tool, it cannot be used in production domains.

    Keep in mind that because of differences in how output files are produced and formatted, some content fragments in Rationalization might require more effort to separate or combine. For example, content that spans multiple pages or that is contained in a table might not be grouped or separated as expected after being rationalized.

    Note

    Charts and tables are ignored when imported as part of the static text or embedded text from a StreamServe template.

    For more information about using content fragments in Designer or Content Author, see OpenText Communications - Designer User Help (CCCD250100-H-UGD)OpenText Communications - Designer User Help (CCCD250100-H-UGD) or OpenText Communications - Content Author User Help (CCCA250100-H-UGD)OpenText Communications - Content Author User Help (CCCA250100-H-UGD).
