    The Interactive output driver produces documents that can be opened in Interactive. It contains the settings that control the appearance and format of Interactive documents.

    Note

    Using Interactive output objects created prior to OpenText Communications version 21.4.0 may result in incorrect behavior in Interactive. OpenText recommends that you create a new Interactive output object to use with your communication.

    After you create the output object, you can use the Properties panel to view and modify the following output properties. After making changes, click Save or the changes will be lost.

    General output settings

    The following sections provide information about the general output options.

    Resolution (DPI)

    You can customize the output resolution settings to ensure that output is clear and readable. The output resolution specifies the sharpness of text and graphics, measured in dots per inch (DPI). If you are generating printed output, use a DPI at or lower than the maximum resolution supported by the output device that you are using for the best processing performance.

    In the Resolution (DPI) list, select the DPI setting that you want to use from the available options.

    If you select Custom, in the Custom box, enter the custom image resolution that you want to use. The minimum valid resolution that you can set is 72, and the maximum valid resolution is 2400.

    Editor settings

    Hyperlink behavior

    In the Hyperlink behavior list, select how text-based hyperlinks are opened within an Interactive document.

    Initial behavior of hidden items

    The Initial behavior of hidden items setting controls whether hidden objects are visible or not when end users first open an Interactive document. A show/hide icon will be available in the Interactive document so that end users can toggle hidden content on or off as needed to customize the content.

    Revision tracking

    Revision tracking lets you follow changes to an Interactive document throughout its life cycle. You can track changes made to an Interactive document by a single user, but revision tracking is an especially useful tool when multiple end users are opening and editing the same Interactive document.

    There are a few things to keep in mind about using revision tracking in Interactive documents:

    • Revision tracking is available for text boxes, table cells, paragraph objects, and variables in an Interactive document. Note that formatted text variables, tagged text variables, and placeholder variables are not supported.

    • Interactive uses login credentials to track the revisions made by each end user and automatically assigns one of five different colors to each reviewer's revisions in order to help quickly differentiate between reviewers.

    • End users receive a warning in the Revisions pane if they add content that causes fixed-height text boxes and table cells to overflow.

    In the Revision tracking list, select one of the following options:

    • User selectable – End users decide whether they want to track the changes they make. The revision tracking tools are available in Interactive, including the button, which turns tracking off and on. This option is the default.

    • Always on – Revision tracking is always on. The revision tracking tools are available in Interactive.

    • Always off – Revision tracking is always off. The revision tracking tools are not available in Interactive.

    Allow user to add comments

    Enabling the Allow user to add comments feature lets end users add comments to the editable text areas of an Interactive document, so that end users can collaborate with co-workers on the changes that need to be made to a document before sending the final version to customers. Adding comments lets end users explain or add context to any changes that they make, or to ask questions about changes that other people have made. Additionally, end users can take part in discussions within an Interactive document itself by adding comments to existing comments.

    Additional settings

    This output includes additional common settings. For more information about any of these settings, see the following locations:

    • Domains – For information about defining the domains to which a resource is assigned, see Section 3.13 “Associating resources with domains”.

    • Workflow – For information about managing an approval workflow, see Section 3.6 “Managing approval workflows”.

    • References – For information about viewing the linked resources connected to a selected resource, see Section 3.3 “Viewing resources in asset library views”.
