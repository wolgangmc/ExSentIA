8.1   Creating Designer communications

    Designer is an intuitive web application with a modern approach to rapid development of personalized customer communications that are fulfilled using the OpenText Communications engine.

    Designer lets you leverage resources that are set up in Design Manager—such as fonts, styles, color families, data files, variables, and output queues—as well as the resources that are directly created in the DAS asset library by authorized users, who can use the OpenText Communications web client to add and configure DAS resources. Designers can then use these resources to create design layouts for customer communications in the design environment.

    Note

    In addition to the resources that are available in a design pack, if your communication is based on a design pack and if the Design Manager application that is associated with the design pack includes search keys with print stream information, then the search keys appear in the output as designated in the application. For more information about search keys, see section 1.6.5.1 "Using Application Search Keys" in OpenText Exstream Design and Production - Output, Production, and Delivery Help (CCDPOPD250100-H-UGD)section 1.6.5.1 "Using Application Search Keys" in OpenText Exstream Design and Production - Output, Production, and Delivery Help (CCDPOPD250100-H-UGD).

    You can also use Designer in conjunction with other OpenText Communications software components to produce customer communications. For example, you might want to use the design environment in Designer to create a design that will be used as a template for customer correspondence in Content Author, or you can design an Interactive document and then include the edited Interactive document in another communication to create customer output.

    For information about how to use Designer, see OpenText Communications - Designer User Help (CCCD250100-H-UGD)OpenText Communications - Designer User Help (CCCD250100-H-UGD).
