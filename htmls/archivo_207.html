    Resources in the DAS asset library are exclusive to the tenant in which they are created. By default, when you add or create a resource, it is associated with the domain in which it is created. However, all resources in a tenant—regardless of the domain with which they are associated—are stored in the same tenant schema in the database.

    To import resources into the asset library, you use the DAS import functionality. The import functionality supports the following options:

    • CAS to DAS—Lets you migrate resources from a CAS tenant in OpenText Communications 16.x to a DAS tenant in a containerized OpenText Communications deployment. You cannot directly import Streamserve 5.6.2 resources into OpenText Communications 25.1.

    • DAS to DAS—Lets you move resources from one DAS tenant to another in a containerized OpenText Communications deployment.

    Tip

    Do not use import functionality to move resources from one domain to another in the same DAS tenant. Instead, use the domain association option.

    To use the import functionality, users must be members of at least one OTDS group that is assigned a role that has, at a minimum, the Create permission enabled for the resources that they want to import.
