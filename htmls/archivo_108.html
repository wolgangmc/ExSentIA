3.4   Applying category tags to resources

  • Communication asset library view
  • Design asset library view
  • Orchestration asset library view
  • Monitor view
  • Rationalization asset library view
  • Dashboards view

    Categories are a convenient way to tag some DAS resources so that you can easily search and filter for them in the DAS asset library. The following table lists the resources to which you can apply category tags in Design asset library view.

    Application type Resource type

    Common

    • Image

    • PDF

    Designer

    • Communication (CD)

    • Design layout

    • Document

    • Page

    • Email

    • Web design

    • Component

    Content Author

    • Communication (CA)

    • Section

    • Clause

    • Graphic message

    • Text message

    For information about applying tags to resources in Designer and Content Author see the following documentation:

    • OpenText Communications - Designer User Help (CCCD250100-H-UGD)OpenText Communications - Designer User Help (CCCD250100-H-UGD)

    • OpenText Communications - Content Author User Help (CCCA250100-H-UGD)OpenText Communications - Content Author User Help (CCCA250100-H-UGD)

    You can use category tags in API calls to search and filter resources in the DAS asset library. For more information about the Design API, see the developer documentation at the following URL:

    https://host:<optionalPort>/design/swagger-ui/

    Note

    At least one category tag must have been created in Tenant management view to enable this feature. For more information, see Creating category tags.

    Applying category tags

    1. In the Designer panel or the Author panel, select a resource that is in a draft state.

    2. In the Properties panel, in the Category tags applied area, click on the category tags that you want to apply. To apply all available tags, click Add all.

      You can use the Search box to search for a specific tag by name. To create and apply a custom tag, enter the tag name in the Search box and click Create, and then apply the newly created tag.

      Note

      Before you can apply category tags, at least one category tag must have been created in Tenant management view. To create new tags, your user role must also be assigned the required permissions.

    3. Click Save.

    Removing category tags

    1. In the Designer panel or the Author panel, select a resource that is in a draft state.

    2. In the Properties panel, in the Category tags applied area, click x in the category tags that you want to remove. To remove all applied tags, click Clear all.

    3. Click Save.
