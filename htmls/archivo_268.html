9.4.4   Releasing a held flow resource

  • Communication asset library view
  • Design asset library view
  • Orchestration asset library view
  • Monitor view
  • Rationalization asset library view
  • Dashboards view

    If an orchestration job is running and the flow model contains delay nodes (hold mode), then you can release one or more holds manually. You can release an entire flow instance or individual flow steps within a flow instance. Once a flow step or instance is released, it is marked complete and the flow continues.

    Note

    Releasing a held flow instance or flow step may take a while. The status remains as Holding while the release operation is performed.

    To release a held flow instance:

    1. In Monitor view, on the Flows tab, click the More actions button at the end of the row of the flow instance that you want to release.

    2. Click Release flow.

      This releases all the flow steps that are currently in a holding status. If there are additional holds in the flow instance, the flow will stop at the next occurrence.

    To release a held flow step:

    1. In Monitor view, on the Flows tab, select a flow instance.

    2. In the Properties panel, click the More actions button at the end of the row of the flow step that you want to release.

    3. Click Release step.
