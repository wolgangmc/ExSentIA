2.4.2   Disabling the default approval workflow in Content Author

    Content Author includes a default approval workflow within the user interface so that business users can easily create and submit messages, clauses, sections, and communications for approval, and approvers can then accept or reject any changes. However, if your business needs require that you use an external approval workflow process for content that is created in Content Author, then you can disable the default approval workflow that Content Author provides.

    If the default workflow is enabled in Content Author, users can click workflow indicators to change workflow states. If the default workflow is disabled in Content Author, users can view the default workflow states, but they cannot change them.

    For information about the default approval workflow in Content Author, see OpenText Communications - Content Author User Help (CCCA250100-H-UGD)OpenText Communications - Content Author User Help (CCCA250100-H-UGD).

    Note

    The default approval workflow for Content Author is enabled or disabled at the tenant level, and affects all domains within that tenant.

    To disable the default approval workflow in Content Author:

    1. In the OpenText Communications web client, sign into the following URL as a tenant administrator user:

      https://<host>:<optionalPort>/design?tenant=<tenantName>

    2. To open Tenant management view, click the Tenant management button .

    3. In the Configurations panel, click Content Author.

    4. On the General tab of the Properties panel, clear the Use default workflow check box.

      If the Use default workflow check box is cleared, workflow status indicators in Content Author are still visible, but they are view-only.

    5. Click Save.
