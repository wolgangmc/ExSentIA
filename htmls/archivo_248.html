8.3.2.3   Making rationalized content available in communications

    After you have rationalized a migration set, you can make the content of that migration set available for use in a Designer communication. If you create a Content Author template from that communication, the content of the migration set will be available for Content Author communications based on that template.

    The method for including rationalized content from a migration set in a Designer communication differs depending on whether the communication is based on a communication set or a design pack.

    Note

    A migration set must be rationalized and approved before it is available for use.

    Using rationalized content with a communication set

    To use rationalized content in a communication that is based on a communication set, first add the migration set that includes the rationalized content to a data source, and then add that data source to the communication set. You can use migration sets only with Driver data source types; however, Interactive driver files are not supported.

    1. Create a Driver data source and add the migration set that you want to use. Up to five migration sets can be added per data source.

    2. Create a communication set and add the data source that includes the migration set.

    3. Create a Designer communication using the communication set.

      Any rationalized content that is part of the migration set is then available to be used in the communication.

      If you subsequently create a Content Author template from the communication, then the migration set content will be available for any Content Author communications created from that template.

    If a communication that includes a migration set is associated with multiple domains, then that migration set becomes available for use in those domains.

    Using rationalized content with a design pack

    To add a migration set to a communication based on a design pack:

    1. From Designer, in Communication Composer, click Communication.

    2. In the Properties panel, click the General tab.

    3. Use the Migration set list to select a migration set.

      If you subsequently create a Content Author template from the communication, then the migration set content will be available for any Content Author communications created from that template.

    For information about using rationalized content in designs created in Designer or Content Author, see OpenText Communications - Designer User Help (CCCD250100-H-UGD)OpenText Communications - Designer User Help (CCCD250100-H-UGD) or OpenText Communications - Content Author User Help (CCCA250100-H-UGD)OpenText Communications - Content Author User Help (CCCA250100-H-UGD).
