8.2.7   Making Content Author content available across communication sets

  • Communication asset library view
  • Design asset library view
  • Orchestration asset library view
  • Monitor view
  • Rationalization asset library view
  • Dashboards view

    In Content Author, you can add an existing content resource, such as a clause. section, or message, to a Content Author communication if the resource and the communication are both based on the same communication set or design pack.

    In addition, if the content resource is communication set-based, then you can optionally make that resource available to all communications that are communication set-based. The communication sets for the content and the communication can be the same or different.

    Note

    If two or more design packs were linked in Design Manager before they were uploaded to DAS, then resources in the linked design packs are available to all Content Author communications that are based on any one of the linked design packs.

    To make a communication set-based content resource available across all communication set-based communications:

    1. From the home page, click the Design asset library button .

    2. In Design asset library view, click the Author tab.

    3. Optional On the Author tab, perform any of the following actions to filter the list of resources by date, workflow state, or content type:

      Task

      Action

      To filter resources by date

      1. Click the Filter assets button.

      2. In the Results filter dialog box, for Created or Updated, enter a date range in the From or To boxes.

      3. Click Apply.

      To filter resources by workflow state

      1. Click the Filter assets button.

      2. In the Results filter dialog box, click the Workflow list and choose a workflow state.

      3. Click Apply.

      To filter resources content type

      1. In the Type filter list, click the arrow.

      2. In the list of content types, select the types that you want to show and clear the types that to you want to hide.

        To show all content types, clear all selections.

    4. From the list on the Author tab, select the content resource that you want to make available across communication sets.

      The resource that you select must be communication set-based and can be any one of the following Content Author resource types: Clause (CA), Graphic message, Section (CA), Text message, or Messaging (CA).

    5. In the Properties panel, on the General tab, select the Available across all communications check box.

    6. Click Save.

    Tip

    You can also access the Available across all communications check box in Authoring view when you edit applied content in a Content Author communication that is communication set-based. For more information about configuring applied content to be available across all communication sets in Content Author, see section 2.10 "Making content available across all communications" in OpenText Communications - Content Author User Help (CCCA250100-H-UGD)section 2.10 "Making content available across all communications" in OpenText Communications - Content Author User Help (CCCA250100-H-UGD).
