5.1   Creating barcodes

  • Rationalization asset library view
  • Dashboards view

    You can use the Common panel in the Design asset library view to create barcode resources in the asset library. These barcodes can then be inserted as needed into Designer designs.

    The various types of barcodes supported by OpenText Communications have different requirements for properties such as size of bars, thickness (or weight) of bars, position on the page, and support of readable characters.

    The OpenText Communications deployment supports the following barcode types:

    • Code 128 / EAN 128

    • Data Matrix

    • Four State

    • OMR

    • PDF417

    • QR Code

    • Swiss QR Code

    • UPC

    Creating a new barcode resource

    1. In the Common panel, click Add, then click Barcode in the selection list.

    2. In the Properties panel, enter a name and, optionally, a description for the barcode.

      Names do not support the following special characters: & / : * ! ? > < | + # %

    3. In the Type list, select a barcode type.

    4. Optional If you want to associate the barcode with a design pack, select the Use design pack check box and then select the design pack in the Design pack list. The design pack determines the variables that you can use as values in the barcode.

      Note

      • This option is available only during the creation of a barcode and if a design pack has been uploaded to DAS.

      • When a barcode is associated with a design pack, that barcode can only be inserted into a communication associated with the selected design pack.

    5. Click Create.

      The barcode is created with default settings, and the Properties panel displays the barcode properties.

    Specifying the two-dimensional properties for a barcode

    1. In the Common panel, click the barcode that you want to update.

      Tip

      To display only barcodes, in the Common panel, click the resource type list, and then select Barcode.

    2. On the General tab of the Properties panel, update the following settings as needed:

      Note

      The available options depend on the barcode type.

      • Design sample—The characters displayed when the barcode is used in Designer. This can help you manually position a barcode in a design, especially if you enter the same number of characters as the final barcode.

        A preview of the design sample is displayed in the Properties panel. Note that large barcodes are scaled to fit the preview area and may not be scannable.

      • Orientation—Determines the direction in which the barcode is read. You can select from the following options:

        • Normal—The barcode is positioned as designed.

        • Face left—The bottom of the barcode faces the left side of the page. The barcode is rotated 90 degrees.

        • Face right—The bottom of the barcode faces the right side of the page. The barcode is rotated 270 degrees.

        • Face up—The bottom of the barcode faces the top of the page. The barcode is rotated 180 degrees.

      • Module width—Specifies the width of the narrowest bar or space in a barcode symbol or the size for each individual module (or block).

      • Module height—Specifies the height of each row in the stack symbol.

      • Height—Specifies the height of the tallest bars for this code.

      • Content type—Specifies the code set of the content encoded in the barcode.

      • Symbol size

        • QR Code—Controls the number of modules that compose a QR Code barcode. If you select Automatic, the OpenText Communications engine encodes the content of the barcode in the smallest symbol version possible; this is the recommended option for the best performance.

        • Data Matrix—Specifies the size of the barcode in pixel blocks. If you select Automatic, the OpenText Communications engine specifies the most efficient size (the smallest number of pixel blocks) when creating the barcode; the size of the barcode might vary.

      • Compaction type—Specifies the compaction needed for the barcode:

        Option What can be encoded Maximum compaction
        Text (all printable ASCII characters) ASCII characters with the values 32–126 and select control characters Two characters per codeword
        Binary (all 256 possible 8–bit byte values) ASCII characters with the values 0–127. This option also supports international character sets. 1.2 bytes per codeword
        Numeric (numeric digits) Long digit strings 2.9 digits per codeword

      • Error correction level—Specifies an industry standard error correction level to control the amount of error recovery data in the barcode. Select the option that corresponds to the specifications of the equipment that reads the barcode.

      • Mask pattern—Specifies the mask pattern settings you want to use. To let the OpenText Communications engine determine the best choice for the mask pattern settings, select Automatic.

      • Rows (0 – automatic, range 3 – 90) and Columns (0 – automatic, range 1 – 30)—Specifies the number of rows and columns used in the barcode. If you want OpenText Communications to automatically specify the number of rows or columns, enter 0.

        PDF417 barcodes generally have an width:height ratio (or aspect ratio) of 3:1. When you enable the automatic selection of rows and columns, OpenText Communications uses this default aspect ratio to calculate the number of rows and columns.

      • Space—Specifies the size of the space between the bars.

      • Thickness—Specifies the thickness of the bars.

      • Placement reference—Specifies the starting point for the position measurements. You can select from the following options:

        • Upper left (portrait)—Measures the barcode placement from the upper left corner of a portrait page.

        • Upper left (landscape)—Measures the barcode placement from the upper left corner of a landscape page.

      • Placement reference bottom—Specifies the distance from the top of the page to the bottom of the barcode. The default is 1 inch (2.54 cm).

      • Placement reference left—Specifies the distance from the left of the page to the left edge of the barcode. The default is 1 inch (2.54 cm).

      • Human readable (HRI)—Adds a human-readable identifier (HRI) to a barcode.

    3. Click Save.

    Specifying the content of a barcode

    1. In the Common panel, click the barcode that you want to update.

      Tip

      To display only barcodes, in the Common panel, click the resource type list, and then select Barcode.

    2. In the Properties panel, click the Values tab. Each row in this tab represents a successive segment of the barcode, ordered from left to right.

    3. On the Values tab, specify the following information to assign content to each segment of the barcode: 

      • In the Digits box, enter the length of that segment of the barcode.

      • Use the Content type list and the Value box to specify how to populate the barcode segment indicated by the Digits box. The following table describes the ways in which you can populate the segment.

        Data type Settings
        Variable data

        • In the Content type list, select Variable.

        • In the Value box, click the Variable button to open a variable selector panel, and select the appropriate variable.

        Static text data

        • In the Content type list, select Text.

        • In the Value box, enter the static text that you want to use.

        Miscellaneous data To use data other than a variable or static text, select one of the available options in the Content type list. The available options depend on the type of barcode.
        No data To place an empty segment in the barcode, select Empty in the Content type list.

        Note

        Make sure that the segment length that you enter in the Digits box is sufficient to contain the information that is specified in the Content type list. If the specified length is too short to contain the information, digits are truncated from the left.

    4. Optional If you are creating an OMR-based barcode, you can select the Zero check box if the value of that barcode segment is allowed to be zero. If you select the Zero check box and the value is zero, no line appears at that position in the barcode. If you clear the Zero check box and the engine encounters a zero value, that segment is converted to a line.

      Note

      The Zero option is only enforced for integer variables. If you use a string variable or text, that segment is allowed to be zero even if the Zero check box is cleared.

    5. Optional Click Add to insert the next segment of the barcode.

    6. Click Save.

    Content type options for barcodes

    The following table describes the options that you can select in the Content type list to populate a barcode segment:

    Table 5.1: Barcode content type options

    Option Description
    Empty The digit information is generated using empty spaces. Select this option if your barcode is not used.
    Variable The digit information is generated using the variable selected in the Value box.
    Text The digit information is generated using the static text defined in the Value box.
    Total sheets in document The digit value is equal to the total sheets in your document. The value is determined by the system variable 'SYS_SheetTotalInDocument'.
    Total pages in document The digit value is equal to the total pages in your document. The value is determined by the system variable 'SYS_PageTotalInDocument'.
    Sheet in document The digit value is equal to the number of the sheet on which the barcode is placed. The value is determined by the system variable 'SYS_SheetInDoc'.
    Page in document The digit value used is equal to the current number of counted pages within the document being written. The value is determined by the system variable 'SYS_PageInDoc'.
    Page in queue The digit value is equal to the current page number in the current queue.
    Page in stream The digit value is equal to the current page number within the current break of the current queue. The value is determined by the system variable 'SYS_PageInBreak'.
    First sheet The value for the first sheet is 1. All other pages have a value of zero.
    Last sheet The value for the last sheet is 1. All other pages have a value of zero.
    Not first sheet The value of all sheets except the last sheet is 1. The first sheet has a value of zero.
    Not last sheet The value of all sheets except the last sheet is 1. The last sheet has a value of zero.
    Sheet in queue The digit value is equal to the current sheet number in the current output queue. The value is determined by the system variable 'SYS_SheetInQueue'.
    Sheet in break The digit value is equal to the current sheet number in the current break of the current output queue. The value is determined by the system variable 'SYS_SheetInBreak'.
    Document in queue The digit value is equal to the current customer number being written into the current output queue. The value is determined by the system variable 'SYS_DocumentInQueue'.
    Document in break The digit value is equal to the current customer number within the current break of the current output queue. The value is determined by the system variable 'SYS_DocumentInBreak'.
    Benchmark (always 1) The digit information is generated as a value of 1.
    Checkchar: Code 128 (Code 128 and EAN 128 only)–Adds the checksum digit used with Code 128–based barcodes. Code 128 barcodes require a checksum character that is printed after your data and before the stop character in your barcode. The checksum character is expected to be a single character. The checkchar is created using information from the encoded data and the current character set. The default values of the characters used in the checksum are based on Code 128 barcode character tables (according to character set A, B, or C, as determined by the start character). If the complete character set is mapped in the map string, the values are based on the mapped locations instead.
    Checksum: OMR Odd (OMR only)–Adds an extra line if the number of lines is even.
    Checksum: OMR Even (OMR only)–Adds an extra line if the number of lines is odd.
    Carriage Return (Swiss QR Code only)–Inserts a carriage return.
    Checksum: UPC (UPC only)–Adds checksum digits used with UPC barcodes.

    Adding a human-readable identifier to a barcode

    You can add a human-readable identifier (HRI) to some barcodes, so a production operator can read the barcode.

    In the following example of a UPC barcode, the numbers below and next to the bars are the HRI.

    Figure 5.1: HRI example

    If the barcode you are creating supports HRI, and you select the Human readable (HRI) check box, the following options appear:

    • HRI font

    • Point size

    • Styling—If applicable, allows you to select the Bold or Italic version of the chosen font.

    • HRI orientation—Specifies the direction the HRI characters face:

      • Left—The HRI is positioned so that it is read from bottom to top.

      • Normal—The HRI is positioned so that it is read from left to right.

      • Right—The HRI is positioned so that it is read from top to bottom.

      • Up—The HRI is positioned so that it is read from right to left.

    • HRI placement—Specifies where you want to place the HRI:

      • Above—Places the HRI above the barcode

      • Below—Places the HRI below the barcode

      • Positioned—Specifies the exact placement of the HRI on the output page using X/Y coordinates, where HRI Bottom is the Y coordinate and HRI Left is the X coordinate.
