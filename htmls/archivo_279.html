    After the pre-sort processing is complete, the sort index data source that is generated contains either one record for each customer or, with the 'SYS_SortIndexDocument' variable, one record per document. The third-party sorting program you use for sorting uses the information in the sort index data source to arrange your customer data. The index file contains a 20-byte key that points to the location in the sort data file for each customer's data.

    Sorting data with a synchronous sorting routine from orchestration

    You can use an on-demand (synchronous) routine in orchestration to sort data that is generated during pre-sort processing. For more information, see section 2.3.2.5 "Scheduling tasks for a domain" in OpenText Communications - Web Client User Help (CCWC250100-H-UGD)section 2.3.2.5 "Scheduling tasks for a domain" in OpenText Communications - Web Client User Help (CCWC250100-H-UGD).

    Sorting data with an asynchronous sorting routine from orchestration

    You can use an batch (asynchronous) routine in orchestration to automatically sort data that is generated during pre-sort processing. For more information, see section 2.3.2.5 "Scheduling tasks for a domain" in OpenText Communications - Web Client User Help (CCWC250100-H-UGD)section 2.3.2.5 "Scheduling tasks for a domain" in OpenText Communications - Web Client User Help (CCWC250100-H-UGD).

    Sorting data with an external program

    If you are using a custom flow, you can use one or more third-party programs to alter the sort index data source so that you can sort and bundle the customer information as needed. Third-party programs are available to help you to cleanse addresses, optimize for mailing, reprint copies, bundle customers, insert last-minute changes, and other options.

    When you sort the files with an external program, you produce a sorted copy of the sort index data source, which you use in the post-processing step for final output.

    As you sort with a third-party program, keep in mind that you must not change or move the 20-character sort index key variable values in the sort index data sources created from pre-sort processing. You can sort them, but do not change any information.

    Considerations for 'SYS_SortIndexDocument', 'SYS_PagePrintStart', and 'SYS_PagePrintEnd' –

    If you used the 'SYS_SortIndexDocument' system variable to map data, the engine generates a separate record for each document that is available to print for each customer. The engine uses encoded information to point to the document information in the sort data file. Do not change or move this value. You can delete records for documents you want to exclude from the final output if the records include a reference to another mapped variable that identifies each document with descriptive information (for example, name or publication number).

    If you mapped the 'SYS_PagePrintStart' and 'SYS_PagePrintEnd' system variables, the engine inserts a 1 for the 'SYS_PagePrintStart' variable and the value of 'SYS_PageTotalInDocument' for the 'SYS_PagePrintEnd' variable. You can change these values as needed to exclude pages from the final output. If you do not want to print the document, enter a zero value in both variables.
