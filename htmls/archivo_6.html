Usage and accessibility information for the help

The help system supports keyboard usage and assistive technologies in order to be accessible also for impaired users. The following sections describe this in more detail:

Recommendation for NVDA users

If you use the NVDA screen reader while using this help, we recommend that you disable the automated switch-over between browse mode and focus mode in NVDA and use the help in focus mode. This particularly applies if you experience problems with the navigation, for example if the help shows multiple help tabs and you cannot switch between them.

General layout of the help window

The help window consists of the following principal areas (see also   “General layout of the help window”):

  1. A banner at the top,

  2. then the primary navigation bar underneath, which consists of a search field on the left and a tab area with one or more help tabs, that means help modules, on the right,

  3. and finally, the content area, which has the Table of Contents panel on the left and the page content on the right, underneath the help tab area.

Some elements in more detail:

  1. If the search is used, the search results are listed as a popup covering the Table of Contents. This combined panel of Table of Contents and search result list is called the Navigation Panel; it also includes the search field above the Table of Contents.

    This entire navigation panel can be hidden and displayed via a toggle button that is located between the search field and the help tabs. Note that the general options menu described in the next list item includes options that can impact the behavior of this panel.

  2. Top banner: General options menu – The top banner of the help window contains a menu button on the left, used for general options and to open this page with usage information.

  3. Help tabs – The help can comprise multiple help modules, each of which appears as a tab in the help tab area; only one of these help modules can be displayed at a time.

  4. Help module drop-down – To the right of the tabs, a drop-down menu provides an alternative way to switch to a different help module. This is particularly relevant if there are more tabs than the tab area can show.

Keyboard usage, two-level TAB key navigation

The help window uses a two-level TAB key navigation approach, that is: There are two different navigation levels when you use the TAB key or the arrow keys to reach the interactive elements of the window:

  • On the top level, pressing the TAB key circulates through the different areas of the window, but does not go to the individual tabs in the help tab area or the individual links of the page content.

  • After reaching the help tab area or the page content, you press the ENTER key to reach the second TAB key navigation level.

    Now, pressing the TAB key or SHIFT+TAB moves the focus between the help tabs or the links on the help page, respectively.

    If you press ESCAPE, you get back to the first level. If the focus is on the first or last element of the second level, also pressing the TAB key or SHIFT+TAB, respectively, moves you back to the first navigation level.

  • In the Table of Contents, in the search result list and also in the drop-down menus, you can directly use the ARROW UP and ARROW DOWN keys to navigate between the sub elements, without pressing ENTER or ESCAPE to switch between the levels (mnemonic aid: These areas all have a vertical orientation).

The advantage of the two-level approach: The number of times you need to press the TAB key no longer depends on how many help tabs the window has, or how many links the current page content has. You always need to press the TAB key the same number of times to get from one point to a certain other point.

  • The starting point normally is the general options menu in the top banner.

  • Pressing the TAB key leads you to the Search input field, then to the search menu button, the Start search button and the Toggle button to show or hide the left panel with the Table of Contents.

  • Then you reach the help tab area. If you press RETURN at this point, you enter the help tab area and now can address the individual help tabs by pressing TAB or SHIFT+TAB again.

    Otherwise, pressing the TAB key again leads you to the help module drop-down. Here you can press the ARROW DOWN and UP keys to navigate to a different help tab and, once you reach the wished help module, press ENTER to switch to this help nodule.

    This drop-down also provides a short abstract per help module, which you display by pressing ARROW RIGHT (or by moving the mouse pointer to the blue i).

    If the help window comprises more than just a few help modules, the drop-down list may also provide a filter field at the top by which you can have the list show those list entries only that match a certain term. You reach this filter field by pressing SHIFT+TAB while one of the list entries is focussed.

    Note that, if you switch to a different help module, the focus generally is moved to the page content.

  • From the help module drop-down, pressing the TAB key leads you to the Table of Contents. Again you can use the ARROW DOWN and UP keys and then ENTER to navigate and switch to a different help page (in contrast to a different help module). Or you can press the TAB key again and then have the focus on the content of the current help page.

  • Also after switching to a different help page, the focus is automatically moved to the page content. Note that it does not enter the content area though; if you want to tab through the links or other interactive elements of the content area, first press ENTER to reach the second TAB navigation level.

  • If the focus is on the page content and on the first level, pressing the TAB key again will lead you to the interactive elements of the browser. Instead, you can press ESCAPE a couple times to return to the beginning; this also applies if the focus is at one of the other areas.
