    When determining how you want to configure tenants and domains in your containerized OpenText Communications deployment, consider the resource sharing needs in your organization, how transactions are counted, and the impact of each deployment approach on database management. These factors will determine how you create and manage tenants and domains.

    A domain is a logical grouping of the resources in the DAS asset library. You can have multiple domains in each tenant, all sharing the same database schema. Because all of the resources are contained within a single DAS database schema, resources can be associated with multiple domains in the tenant.

    Tenants and domains are defined as follows in OpenText Communications:

    • Tenant: A data-isolated environment that users access to complete their daily work. A tenant is associated with tables of resources for one or more OpenText Communications services, such as Design and Interactive. The tables for each service are in separate schemas. Each schema has its own name and database credentials.

    • Domain: A logical grouping of the resources in the DAS asset library. You can have multiple domains in each tenant, all sharing the same database schema. Because all of the resources are contained within a single DAS database schema, resources can be associated with multiple domains in the tenant.
