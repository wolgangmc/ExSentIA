2.2.2.1   Creating new tenants and associating service schemas

    If you are using System management view to add a new tenant, you use the tenant creation wizard to add the new tenant to OTDS first.

    Next, if your deployment has automatic schema management enabled, you use the tenant wizard to add new service schemas and their database tables to the tenant. The wizard can either create schemas and their tables for you, or you can specify existing schemas and let the wizard create the tables.

    Note

    Alternatively, you can use a Helm command and bootstrap configuration to add a new tenant to OTDS and to add new service schemas and tables to a tenant.

    If you are using manual schema management for your OpenText Communications databases, you use the tenant wizard to associate manually created service schemas and tables in the database with the new tenant.

    In a typical OpenText Communications deployment, you can add and associate schemas and tables for the following services:

    • Design

    • Interactive

    • Orchestration

    • Journey

    Adding a new tenant to OTDS in System management view

    1. Sign in to the OpenText Communications web client as an OTDS administrator user at the following URL:

      https://<host>:<optionalPort>/design?tenant=system

    2. On the Tenants and services panel, click Add tenant.

      Note

      Tenant names can contain up to 40 alphanumeric and underscore characters. OpenText recommends that tenant names begin with a letter.

    3. On the opening page of the tenant creation wizard, in the Tenant name field, enter a name for the tenant, and then click Proceed.

    4. On the OTDS page, enter a new OTDS administrator user name and password for the new OpenText Communications tenant, and then click Create.

      After you click Create, the OpenText Communications tenant exists in OTDS. However, you must add at lease one service schema to the tenant before the tenant appears on the Tenants and services panel.

    Adding service schemas to a tenant in System management view

    Note

    Your deployment must have automatic schema management enabled to perform this procedure.

    1. If you did not add a new tenant to OTDS using a Helm command and bootstrap configuration, in System management view, add a new tenant to OTDS.

    2. In the tenant wizard, on the details page for each service in your deployment, do one of the following:

      • If the service database has no schema name or database credentials configured for the tenant, click Create. The wizard creates the schema, database credentials, and resource tables for the service and then associates them with the tenant.

      • If the service database has a schema name and database credentials for the tenant, but no tables in the schema, do the following:

        1. Select the Use existing schema check box.

        2. In the Schema name, Database user name, and Database password boxes, enter the values that are configured for the tenant in the service database.

        3. Click Create. The wizard creates the tables in the schema and then associates the schema and tables with the tenant.

          If the wizard returns an error, verify that the schema name and database credentials for the tenant in the service database match the values that you entered in the wizard.

    3. Optional To update a service schema, its tables, or its database credentials in the tenant, do the following:

      1. Click Previous or Next to go to the details page for the service.

      2. Click Delete to remove the service schema, its tables, and its database credentials from the tenant wizard and the tenant configuration in the service database.

      3. Optional Manually update the schema name and database credentials for the tenant in the service database.

      4. Repeat step 2 above, but click Update instead of Create.

        If the wizard returns an error, verify that the values on the service details page match the values configured for the tenant in the service database.

    4. Optional To remove a service schema, its tables, and its database credentials from a tenant, click Previous or Next to go to the details page for the service, and then click Delete.

      Important

      Clicking Delete in the wizard also deletes the service schema, its tables, and its database credentials from the tenant configuration in the service database.

    5. On the Overview page, review the schema names for each added service and then click Close. The tenant appears in the Tenants and services panel in System management view with a green indicator for each service schema that you added.

    Associating manually created service schemas with a tenant in System management view

    Note

    Perform this procedure for a service schema only if the schema, its tables, and its database credentials exist in the service database.

    1. If you did not add a new tenant to OTDS using a Helm command and bootstrap configuration, in System management view, add a new tenant to OTDS.

    2. In the tenant wizard, on the details page for each service that has a manually created schema with tables and database credentials in the database, do the following:

      1. In the Schema name, Database user name, and Database password boxes, enter the values for these settings as configured in the database.

      2. Click Associate. The wizard associates the schema, its tables, and its database credentials with the tenant.

    3. Optional To update a manually created service schema, its tables, or its database credentials in the tenant, do the following:

      1. Click Previous or Next to go to the details page for the service.

      2. Click Delete.

        Note

        If automatic schema management is disabled, clicking Delete breaks the association between the service and the tenant. However, it does not clear the schema, tables, and database credentials for the tenant in the service database.

      3. In the service database, manually update the schema and database credentials for the tenant.

      4. Use Liquibase to create the tables in the updated schema.

      5. Repeat step 2 above.

        If the wizard returns an error, verify that the values on the service details page match the values configured for the tenant in the service database.

    4. Optional To remove a service schema, its tables, and its database credentials from a tenant, perform steps 3a and 3b above.

      If automatic schema management is disabled, clear the schema, tables, and database credentials for the tenant in the service database.

    5. In the wizard, on the final Overview page, review the newly associated service schema names and then click Close. The tenant appears in the Tenants and services panel in System management view with a green indicator for each associated service schema.
