6.3   Importing an export package

    The Import option lets you import resources from export packages into a DAS asset library. Export packages can contain CAS resources, DAS resources, or migration sets.

    During an import process, it is possible to encounter conflicts between the resources being imported and the resources that already exist in the destination DAS asset library. A resource conflict occurs when an incoming resource has the same resource ID and version as an existing resource, but those resources have different content. You can handle resource conflicts using a default resolution policy for import to automatically resolve conflicts during the import process.

    Note

    If the export package you are importing contains a resource object that exists in another domain in the same tenant, the system does not re-import the resource; rather, it simply associates the resource with the domain you are attempting to import it into. However, changes made to the resource in one domain do not affect the resource in other domains.

    To import an export package:

    1. In the OpenText Communications web client, click the Import button in the application toolbar.

    2. In the Import dialog box, click the File selection button to open the file browser and select the export package that you want to import, then click Open.

    3. If necessary, in the Package password field, enter the password for the export package ZIP file.

    4. In the Import format list, select one of the following:

      • DAS—Indicates that the export package contains DAS resources. This is the default selection.

      • CAS—Indicates that the export package contains CAS resources.

      • Rationalization—Indicates that the export package contains a migration set. For more information, see section 8.3.2.2.2 "Creating a migration set" in OpenText Communications - Web Client User Help (CCWC250100-H-UGD)section 8.3.2.2.2 "Creating a migration set" in OpenText Communications - Web Client User Help (CCWC250100-H-UGD).

    5. In the Default resolution policy list, select from the following options to set the default resolution policy to use to resolve any resource conflicts that might occur during the import:

      • Auto rename—Automatically rename and import conflicted resources. This resolution policy assigns a new name and a randomly generated resource ID to the imported resource. The import process also updates references to the imported resource with references to the renamed resource. The naming convention for the renamed object is <originalName>_<importTimeStamp>.

        Note

        Auto rename is not supported for system variables. The conflict resolution policy automatically changes from Auto rename to Skip for system variables.

      • Error—Cancel the import and issue an error message. Keep in mind that using this policy will cancel the entire import process if any conflicts are encountered. No resources from the export package will be imported.

      • Replace—Overwrite resources that exist in the asset library. If the existing resource is in an approved state, the imported resource is stored as a new version of the existing resource. However, if the existing resource is in draft or rejected state, then the content of the draft or rejected version of the existing resource is replaced with the content of the imported resource. Keep in mind that if you have existing resources in review state, these resource cannot be replaced and the import process will fail. You must first either return the resources to a draft state or approve them and then retry the import.

        For example, if versions 1, 2, and 3 of a page resource already exist, but then a conflicting page version 3 is imported, with Replace selected, the incoming resource will become version 4.

      • Skip—Do not import resources that have conflicts. The import process updates references to skipped resources with references to the existing resources in the asset library. You cannot skip required references.

    6. Click Next. The import process evaluates the selected resource package, and identifies any resource conflicts.

      • If no conflicts are found, then the import is completed and Import dialog box displays the list of imported resources. Review this list and click Close to close the dialog box.

      • If resource conflicts are found, then the Import dialog box displays the list of resources to be imported. By default, all conflicted resources are set to be resolved using the default resolution policy. To continue the import process, go to the next step.

    7. Review the displayed list and change the resolution policies for individual resources, if needed. Optionally, use the following lists to filter the displayed resources:

      • Resource type—Lets you filter the displayed list for a specific resource type

      • Status—Lets you filter the displayed list based the conflict

    8. Click Import. The import is completed with all conflicts resolved as specified, and the Import dialog box displays the list of imported resources. Review this list and click Close to close the dialog box.
