    If you manually manage your database schemas, then you will need to set up tenant schemas directly in the database for each of the OpenText Communications services, and then use System management view to associate the new tenant with the schema for each service in the OpenText Communications deployment.

    Note

    This step is required only for manually managed databases. If you are using automatic schema management for your OpenText Communications databases, see Creating tenants in System management view or Creating tenants with a bootstrap configuration.

    After you create the tenant schemas in your database, you must also use Liquibase to initialize the schemas. You can either allow Liquibase to handle schema initialization automatically or you can use it to generate the SQL file that can be applied to the database. The OpenText Communications Liquibase scripts are packaged in the exstreamce-database-setup-version.zip file that is available for download on My Support.

    Notes

    • Regardless of the schema management method that you use for your databases, OpenText recommends that you use either the tenant creation wizard in System management view or a bootstrap configuration to create new OpenText Communications tenants in OTDS. Doing so saves a significant number of manual steps. For information about creating tenants manually in OTDS, see OpenText Directory Services Web Client Help (OTDS-H-AWC).

    • For information about OpenText product compatibility, see the OpenText compatibly information provided in the release notes for the product. Release notes are available on My Support.

    To create database schemas for a new tenant:

    1. Extract the contents of the exstreamce-database-setup-version.zip file, and locate the exstreamce-database-setup-version folder.

    2. In the exstreamce-database-setup-version folder, extract the contents of the liquibase-core-<version>-bin.zip file into a Liquibase folder.

    3. In the Liquibase folder, copy the JDBC driver for your databases into the lib folder.

    4. Create database schemas for the new tenant in each OpenText Communications service in your deployment. If you're using separate database instances for each service, then create the appropriate schema in each database.

      For example, if you're creating a tenant called sandbox, you might create the following schemas in the database:

      • For Design, exd_sandbox

      • For Orchestration, exo_sandbox

      • For Interactive, exe_sandbox

      • For Journey, exi_sandbox

      Important

      To ensure security in a multi-tenant deployment, you must use a separate schema and schema user for each tenant.

    5. Assign a schema owner for each schema. At a minimum, the schema owner must have CRUD permissions (Create, Read, Update, and Delete).

    6. For each schema, complete one of the following steps to initialize the schema in your database:

      Step Description
      Use Liquibase to generate SQL statements and manually initialize the schema in the database.

      1. Use the following command with the liquibase.bat batch file or the liquibase bash script file to generate the SQL statement that is needed to initialize the schema:

        <liquibaseExecutable> --driver="<dbDriverClassName>" --url="<dbURL>" --username="<SchemaUser>" --password="<schemaUserPassword>" --changeLogFile="<dbDirectory>\<serviceName>\changelog.xml" updateSQL

        The resulting output contains the SQL statements that you need, along with some other information. By default, the output is written to the console. To write the output directly to a SQL file, add the following parameter to the command:

        --outputFile=<fileName>

      2. If the output was written to the console, create a SQL file that contains the SQL statements.

      3. Use the SQL file to initialize the schema in the database.

      Use Liquibase to automatically initialize the schema in the database.

      Use the following command with the liquibase.bat batch file or the liquibase bash script file to generate the SQL statement that is needed to initialize the schema:

      <liquibaseExecutable> --driver="<dbDriverClassName>" --url="<dbURL>" --username="<schemaUser>" --password="<schemaUserPassword>" --changeLogFile="<dbDirectory>\<serviceName>\changelog.xml" update

      Example 2.1: Generate SQL statements to initialize a Design tenant schema

      The following command uses the liquibase.bat batch file on Windows to generate the SQL statement to initialize a tenant schema for the Design service in a PostgreSQL database:

      liquibase.bat --driver="org.postgresql.Driver" --url="jdbc:postgresql://sampleserver.opentext.com:5432/ExstreamDatabase?raiseExceptionOnSilentRollback=false" --username="exd_sandbox" --password="vExstream1!" --changeLogFile=".\postgresql\das\changelog.xml" updateSQL


      Example 2.2: Automatically initialize an Orchestration tenant schema

      The following command uses the liquibase bash script on Linux to automatically initialize a tenant schema for the Orchestration service in an Oracle database:

      liquibase --driver="oracle.jdbc.driver.OracleDriver" --url="jdbc:oracle:thin://sampleserver.opentext.com:1521/ExstreamDatabase" --username="exo_sandbox" --password="Exstream1!" --changeLogFile=".\oracle\orchestration\changelog.xml" update


      Example 2.3: Generate SQL statements to initialize an Interactive tenant schema

      The following command uses the liquibase bash script on Linux to generate a SQL file containing the SQL statements needed to initialize a tenant schema for the Interactive service in a SQL Server database:

      liquibase --driver="com.microsoft.sqlserver.jdbc.SQLServerDriver" --url="jdbc:sqlserver://localhost:1433;databaseName=ExstreamDatabase" --username="exe_sandbox" --password="Exstream1!" --changeLogFile=".\sqlserver\empower\tenant\db.changelog-master.xml" --outputFile=.\exe_sandboxTenant.sql updateSQL
