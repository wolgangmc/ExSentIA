2.2.3.2   Running a Helm upgrade with the bootstrap configuration

    When you use a bootstrap configuration to create a new tenant environment, it will name and configure tenants for each of the OpenText Communications services that you use and then configure the roles and domains in each tenant environment. After the bootstrap configuration is complete, you will need to finish installing the product key in each new tenant to make the tenant environment available to users.

    To create tenants with the bootstrap configuration:

    1. Create your custom bootstrap configuration.

    2. In the Helm chart, set bootstrap.enabled to true. You can set this value in the following ways:

      • Modify the bootstrap.enabled value in the values.yaml file that you used to initially deploy OpenText Communications.

      • Modify the bootstrap.enabled value in a custom bootstrap file.

      • Include --set bootstrap.enabled=true in your Helm command.

    3. Run the helm upgrade command with the custom bootstrap parameters.

      Note

      To retain your previous deployment configuration, make sure that you also use all of the other parameters (including the values.yaml file) that you used in your initial Helm install command. For example, if you used a custom YAML file with specific configuration parameters, you must include that file in the helm upgrade command.

      Example 2.12: 

      helm upgrade exstream exstream-<version>.tgz --wait --timeout 30m -f values.yaml

      In this example, the custom bootstrap configuration has been added to the values.yaml file that was used to initially deploy OpenText Communications and the Helm command references that edited file.


      Example 2.13: 

      helm upgrade exstream exstream-<version>.tgz --wait --timeout 30m --set bootstrap.enabled=true -f values.yaml -f bootstrap.yaml

      In this example, the bootstrap.enabled parameter is set inline, the bootstrap.config parameters have been added to a new bootstrap.yaml, and the Helm command references both the new YAML file and the values.yaml file that was used to initially deploy OpenText Communications.


      Example 2.14: 

      helm upgrade exstream exstream-<version>.tgz --wait --timeout 30m -f values.yaml -f tenant_testing.yaml -f tenant_development.yaml

      In this example, the custom bootstrap configuration for two new tenants—testing and development—has been added to the separate tenant_testing.yaml and tenant_development.yaml files, the OpenText Communications OTDS product key file for each tenant is specified inline, and the Helm command references both the custom files and the values.yaml file that was used to initially deploy OpenText Communications.
