2.6.3   Setting a default engine

  • Communication asset library view
  • Design asset library view
  • Orchestration asset library view
  • Monitor view
  • Rationalization asset library view
  • Dashboards view

    Setting the default engine for your tenant lets you specify the single-byte character set (SBCS) or double -byte character set (DBCS) engine version that applications will use if an engine is not otherwise specified. For example, you might configure the latest engine to be the default engine for your tenant so that new applications that use new functionality automatically use the latest engine version. Additionally, setting a default engine can save time when importing applications that already have a default engine specified. In that case, you don't need to configure the engine settings on those applications because they will automatically pick up the default engine for your tenant.

    If you are working with a new deployment and have not yet uploaded any engines, the first SBCS and DBCS engines that you upload will automatically be marked as “Default” in System management view. So, in that scenario, the SBCS engine would be the default engine for SBCS applications, and the DBCS engine would be the default engine for DBCS applications.

    Note

    Whenever DocGen is installed or upgraded, the engine in the DocGen image is automatically set as the default. You can set another engine as the default after installation.

    Remember that the default engine shows up as “Default” when users select between different engines in your tenant. So if you have an engine named “Development” and it is the default engine for your tenant, then that engine will show up as both “Default” and as “Development” in any lists where users select between engines.

    Important

    If you have applications that use a default engine, and you then select a different engine to be the default, behavior and output can change across those applications.

    To set a default engine:

    1. Sign in to the OpenText Communications web client as an OTDS administrator user at the following URL:

      https://<host>:<optionalPort>/design?tenant=system

    2. In the Engines panel, select an engine that is not currently marked as Default, and then click Open.

    3. Depending on the engine, select either the Default DBCS engine or Default SBCS engine check box, and then click Save.
