    Portable Data File 417 (PDF417) barcodes are two-dimensional, stacked barcodes that can encode over 1KB of data per label. They are mostly used when it is impractical to store item information in a database or the database is not accessible when and where the barcode is read. Because a PDF417 barcode can store so much data, item data such as the content of a shipping manifest or equipment maintenance history can be carried on the item, without requiring access to a remote database.

    Figure 5.6: PDF417 barcode example

    Each PDF417 barcode is a stack of vertically aligned rows. The barcode can contain between three and 90 rows. Each row includes between one and 30 symbol characters, excluding start, stop, and row indicator columns. There is a quiet zone on all four sides.

    A PDF417 barcode is created as a bitmap that is drawn to the page during engine run time. PDF417 barcodes require a 2D scanner. They cannot be read using a linear barcode scanner.
