    At least one engine must be available in the system tenant to generate previews, run simulations, and produce output in your OpenText Communications environment. You can also upload additional engines as needed for specific applications. Keep in mind that OpenText Communications engine runs are initiated from specific domains within each tenant, so the behavior of an engine run depends on the domain type and the product key used to license the tenant.

    As you upload engines, keep the following considerations in mind:

    • OpenText Communications engine ZIP files are available for download on My Support.

    • OpenText Communications deployments support only 64-bit Linux engines, version 20.2 or later.

    Note

    To upload engines, you must be a member of the otadmins group in the OTDS system tenant.

    To upload an engine:

    1. From My Support, download the ZIP file for the OpenText Communications engine that you want to upload.

    2. Sign in to the OpenText Communications web client as an OTDS administrator user at the following URL:

      https://<host>:<optionalPort>/design?tenant=system

    3. In the Engines panel, click Upload engine.

    4. In the file browser, select the downloaded engine ZIP file, and then click Open to upload the engine to the system tenant.
