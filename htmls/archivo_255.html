    For migrations sets that contain StreamServe templates, if the original content contains rules, those rules are also rationalized.

    To work with rules in Rationalization:

    1. In the Fragments panel, open a migration set.

    2. Click the Rules button .

    3. Select a rule from the list. The rule content is highlighted in the Preview panel.

    4. In the Properties panel for the rule, adjust the properties as needed:

      • On the General tab, the following information is available:

        • Name—Update the name to help differentiate the rule when Designer and Content Author users add it to content.

          Names do not support the following special characters: & / : * ! ? > < | + # %

        • Description—Optionally, add a description for the rule.

        • Fragment—A sample value for the rule appears in blue and is shown with the content that immediately surrounds it for context. Because the rule value that appears is sample data that changes for each customer, you cannot make changes to content in this field.

      • On the Occurrences tab, you can see all of the different instances of the rule that the Rationalization tool was able to identify across all of the templates in the migration set and DAS. You can click the up and down arrow buttons to the right of each instance to see what the fragment looks like in each case.

    5. Click Save to save your changes, or click Ignore to hide the rule from Designer and Content Author users.
