2.2.2.2   Understanding the color indicators for tenants

    Use the color indicators to quickly see general information about each tenant in your OpenText Communications deployment. Keep in mind that the indicators do not show the current status of the tenant; instead, they are meant to tell you at a glance whether a tenant has been created for a particular service or is currently enabled in OTDS. You can point to any of the indicators to see a tooltip that provides more information.

    The following table shows the indicators that are available in the Tenants and services panel, and what each indicator means.

    Table 2.2: Tenant indicators

    Indicator Description
    The tenant has been created for the indicated service.
    An error occurred with the tenant during the current browser session. If you resolve the error, then the indicator will change to green; however, if you do not resolve the error, then the indicator will change to gray during subsequent browser sessions.
    An error occurred with the tenant during a previous browser session. If you resolve the error, then the indicator will change to green; otherwise, it will continue to be gray.

    You’ll see this indicator in the following scenarios:

    • The tenant has been disabled in OTDS. For information about enabling and disabling tenants in OTDS, see OpenText Directory Services - Installation and Administration Guide (OTDS-IWC)OpenText Directory Services - Installation and Administration Guide (OTDS-IWC).

    • The tenant has been created in an OpenText Communications service, but has not been created in OTDS. You must create a corresponding tenant in OTDS before the tenant environment can be used in your OpenText Communications deployment.
