3.2   Asset library views at a glance

  • Dashboards view

    The asset library views in the OpenText Communications web client let you add and manage resources that are used to create communications and generate customer output. Depending on the type of resource that you want manage, you use one of following asset library views.

    Communication asset library view

    You use the Communication asset library view to add and manage the resources that are used to configure OpenText Communications communications. To access this view, click the Communication asset library button on the application toolbar in the OpenText Communications web client.

    The Communication asset library view contains the following panels:

    • Communication sets—Lets you combine other communication assets into one collection

    • Data—Lets you create and manage variables and data sources for use in communications.

      The Data panel also lets you edit rule and event resources, except for their logic and code, which you can edit in Designer.

    • Delivery—Lets you create and manage output drivers and queues

    • Region—Lets you create and manage languages and locales

    When you select a resource, a corresponding Properties panel lets you view and modify the properties for that selected resource. The Properties panel is a dynamic panel that is always located on the right side of the view.

    For information about adding resources in this view, see Adding communication resources.

    Design asset library view

    You use the Design asset library view to add and manage the design resources that are used to create and design Designer and Content Author communications. To access this view, click the Design asset library button on the application toolbar in the OpenText Communications web client.

    The Design asset library view contains the following panels:

    • Common—Lets you add and manage design resources that are not specific to Designer or Content Author

    • Designer—Lets you manage resources that are related to Designer communications

    • Author—Lets you manage resources that are related to Content Author communications, communication containers and documents.

    When you select a resource, a corresponding Properties panel lets you view and modify the properties for that selected resource. The Properties panel is a dynamic panel that is always located on the right side of the view.

    If a Designer or Content Author resource uses variations, you will see the number of alternate variations associated with the resource in the Variations column. Click the number of variations in the Variations column to expand the list of alternate variations.

    Important

    When the standard variation of a resource is in draft state, users can delete any version of an approved alternate variation from Design view in Designer and from Authoring view in Content Author. In Design asset library view, you do not have the option to delete only alternate variations. When you delete a standard variation in Design asset library view, all alternate variations are also deleted.

    For information about creating variations in Designer and Content Author, see OpenText Communications - Designer User Help (CCCD250100-H-UGD)OpenText Communications - Designer User Help (CCCD250100-H-UGD) and OpenText Communications - Content Author User Help (CCCA250100-H-UGD)OpenText Communications - Content Author User Help (CCCA250100-H-UGD), respectively.

    For information about adding resources in this view, see Adding design resources.

    Orchestration asset library view

    You use Orchestration asset library view to add and manage the resources that are used preview communications and create customer output using the OpenText Communications engine. To access this view, click the Orchestration asset library button on the application toolbar in the OpenText Communications web client.

    The Orchestration asset library view contains the following panels:

    • Flow—Lets you add and manage flow models and flow contexts for engine orchestration jobs that are defined in Orchestrator. It also lets you view the list of published flow contexts and flow models. For information about publishing flows, see section 9.2 "Publishing orchestration flows" in OpenText Communications - Web Client User Help (CCWC250100-H-UGD)section 9.2 "Publishing orchestration flows" in OpenText Communications - Web Client User Help (CCWC250100-H-UGD).

    • Context items—Lets you view the context items for the latest versions of flow contexts.

    • Templates—Lets you manage templates that are generated when application packages and subpackages are uploaded to DAS from Design Manager. Application templates can be used to create Designer communications, from which you can optionally create templates for Content Author communications.

      Note

      In DAS, communication sets can also be used to create Communication Designer communications.

      In addition, Design Manager application packages that allow split communications in Content Author, along with their linked single-document subpackages, can generate the following Content Author templates:

      • Communication container templates: These templates can be used to create Content Author communication containers.

      • Document templates: These templates can be used to create Content Author documents in an associated communication container.

      For more information about packaging Design Manager applications and single-documents for use as Content Author container templates and document templates, see Section 8.2.1.1 “Generating templates for Content Author communication containers and documents”.

      For more information about using templates to create new Content Author communication containers and documents, see section 8.2.4 "Starting a Content Author communication container" in OpenText Communications - Web Client User Help (CCWC250100-H-UGD)section 8.2.4 "Starting a Content Author communication container" in OpenText Communications - Web Client User Help (CCWC250100-H-UGD).

    • Design packs—Lets you manage design packs that are uploaded along with Design Manager application packages

    • Packages—Lets you manage application packages and subpackages that were created in Design Manager and uploaded to DAS from Design Manager.

      In the Properties panel for an application package, the Linked subpackages list on the General tab shows subpackages that are linked to the application package, if any.

      For information about creating subpackages and uploading them to DAS, see section 2.3.3 "Creating subpackages for campaigns, documents, and objects" in OpenText Exstream Design and Production - Output, Production, and Delivery Help (CCDPOPD250100-H-UGD)section 2.3.3 "Creating subpackages for campaigns, documents, and objects" in OpenText Exstream Design and Production - Output, Production, and Delivery Help (CCDPOPD250100-H-UGD) and section 2.3.4 "Uploading a subpackage file to DAS" in OpenText Exstream Design and Production - Output, Production, and Delivery Help (CCDPOPD250100-H-UGD)section 2.3.4 "Uploading a subpackage file to DAS" in OpenText Exstream Design and Production - Output, Production, and Delivery Help (CCDPOPD250100-H-UGD).

      In addition, see Managing application packages and subpackages for information about the following topics:

      • Rules that apply to application package and subpackage resources.

      • Linking subpackages to application packages.

      • Using application package templates and linked subpackage templates to create Content Author communication containers and Content Author documents.

    • Run definitions—Lets you manage run definitions for engine orchestration jobs that uploaded along with Design Manager application packages.

    When you select a resource, a corresponding Properties panel lets you view and modify the properties for that selected resource. The Properties panel is a dynamic panel that is always located on the right side of the view.

    For information about adding resources in this view, see Adding orchestration flow resources.

    Monitor view

    You use Monitor view to see the flow instances that correspond to your engine orchestration jobs and the status of each job. To access this view, click the Monitoring button on the application toolbar in the OpenText Communications web client.

    Note

    If you are using the Cloud Platform, Monitor view is only available if the OpenText Communications subscription is set to Standard (not Base).

    The Monitor view contains the following panels:

    • Jobs—Lets you manage engine orchestration jobs

    • Flows—Lets you manage the flow instances associated with a specific orchestration job

    When you select a resource, a corresponding Properties panel lets you view the properties for that selected resource. The Properties panel is a dynamic panel on the right side of the view.

    For information about monitoring orchestration jobs, see Monitoring orchestration jobs.

    Rationalization asset library view

    You use Rationalization asset library view to add and manage the migration sets that are used in Designer communications. To access this view, click the Rationalization asset library button on the application toolbar in the OpenText Communications web client.

    The Rationalization asset library view contains the Migration sets panel. When you select a resource in the Migration sets panel, a corresponding Properties panel lets you view and modify the properties for that selected resource. The Properties panel is a dynamic panel that is always located on the right side of the view.

    For specific information about adding resources in this view, see Rationalizing external content.

    Dashboards view

    You use Dashboards view to create and manage dashboards and their widgets, which can display information such as orchestration job status and licensing information. To access this view, click the Dashboards button on the application toolbar in the OpenText Communications web client.

    For information about dashboards, see Using Dashboards.
