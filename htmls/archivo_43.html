    If your initial deployment process did not set up a sample tenant environment, you can use a bootstrap configuration to manually configure a sample tenant with the default domains, roles, and user permissions.

    Step 1: Setting up tenant schemas

    If you are using manual schema management for your OpenText Communications databases, then before you can create the sample tenant in your deployment, you will need to first create the tenant schemas for each OpenText Communications service as described in the following table.

    Note

    The schema name and schema owner credentials in the following table are the default values that are the same as those used if a sample tenant is automatically configured during deployment. You can use custom user-configured values instead of these default values. If you choose to use custom values, make sure that you also update the default values in the sample bootstrap configuration.

    Service Schema name Schema owner

    Design

    exd_test

    • User name: exd_test

    • Password: Exstream1!

    Step 2: Creating the sample bootstrap configuration

    The sample bootstrap configuration contains the values that are used to create OTDS users and groups, and configure roles and domains for the sample tenant. The sample bootstrap configuration is specified in the same manner as any other custom bootstrap configuration that is used to create additional tenants in an OpenText Communications deployment, that is, you can either add the bootstrap parameters to the values.yaml that you used to initially deploy OpenText Communications, or you can create a separate bootstrap file with those values. Regardless of the method, the sample bootstrap configuration is provided in the following parameter block:

    Example 2.15: Bootstrap parameter block

    bootstrap:
	enabled: true
	config: 
		<Enter sample bootstrap configuration parameters>

    Step 3: Creating the sample tenant with a bootstrap configuration

    To create the sample tenant, run a helm upgrade command with the sample bootstrap parameters. The Helm command that you use to create the sample tenant is similar to the Helm command that you would use to create any other additional tenants in an OpenText Communications deployment.

    For example, if you added the bootstrap configuration for the sample tenant, including the product key contents, to a bootstrap_sample.yaml file, then you can use the following Helm command:

    helm upgrade exstream exstream-<version>.tgz --wait --timeout 30m -f values.yaml -f bootstrap_sample.yaml
