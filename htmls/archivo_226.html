8.2.4   Starting a Content Author communication container

    If your deployment has the required templates, you can create a new communication container with multiple new documents. You can also add documents to a communication container or remove documents.

    For more information about creating containers and adding documents to containers, see section 1.1.2.1 "Creating a communication container with documents" in OpenText Communications - Content Author User Help (CCCA250100-H-UGD)section 1.1.2.1 "Creating a communication container with documents" in OpenText Communications - Content Author User Help (CCCA250100-H-UGD).

    Creating a new communication container with new documents:

    You can create Content Author communication containers and Content Author documents from the following templates, which are listed in the Templates panel in Orchestration asset library view:

    • Communication container templates

    • Document templates

    For more information, see Orchestration asset library view and Managing application packages and subpackages .

    To create a new communication container with new documents:

    1. In Communication launcher view, click the Communications panel, and then click the New button to see a list of available templates.

    2. In the Template name column, select a template for a communication container. A stacked icon indicates a container template.

      The Template name column shows approved templates only.

    3. Click Next.

    4. To create the new communication container, complete the following steps:

      1. Enter a name and, optionally, a description for the new communication container.

        Names do not support the following special characters: & / : * ! ? > < | + # %

      2. Click the Create communication container option.

      3. Click Next.

    5. To create a new document in the communication container, complete the following steps:

      Note

      This step is required because a communication container must have at least one document.

      1. Enter a name and, optionally, a description for the new document.

        Names do not support the following special characters: & / : * ! ? > < | + # %

      2. In the Document template list, choose a document template.

        Only document templates that are linked to the template for the selected communication container appear in the list.

      3. Click Create.

      The new communication container opens in Communication view with the new document showing in the preview area.

    Adding a new document to a communication container

    If you want to add a new document to a communication container, you need to select a document template. A document template is available to choose if it meets the following requirements:

    • No document in the container was created from the document template.

    • The subpackage template corresponding to the document template was generated from a subpackage that is linked to the application template corresponding to the container template. You can access the Linked subpackages list for an application package in Orchestration asset library view.

    To add a new document to a communication container:

    1. In Communication launcher view, click the Communications panel, and then click the New button to see a list of available templates.

    2. In the Template name column, select the container template that was used to create the communication container to which you will add the new document. A stacked icon indicates a container template.

      The Template name column shows approved templates only.

    3. Click Next.

    4. To add a new document to a communication container, complete the following steps:

      1. Enter a name and, optionally, a description for the new document.

        Names do not support the following special characters: & / : * ! ? > < | + # %

      2. Click the Create document option.

      3. In the Communication container list, choose the communication container in which to add the new document.

        The list includes only communication containers that are in the draft workflow state and not locked by another user.

      4. In the Document template list, choose the template for the document that you want to add. The list includes only document templates that are associated with the container template that you selected in step 2.

        Note

        The Document template list excludes document templates that were already used to create a document in the selected container.

      5. Click Create to add the new document to the communication container.

        The communication container opens in Communication view with the new document showing in the preview area.

    Removing a document from a communication container

    You can remove a document from a communication container if the document is not used in an earlier version of the container. Additionally, the communication container must be in the draft workflow state and not locked by another user.

    To remove a document from a communication container:

    1. In Communication launcher view, click the Communications panel.

    2. In the Communication name column, click the name of the communication container that includes the document that you want to remove. A stacked icon indicates a communication container.

      Tip

      Clicking a communication container name shows or hides a tree view of the documents that it includes.

    3. In the document tree for communication container, click the name of the document that you want to remove.

    4. In the document tree, click document to remove, and then click Delete.

    5. In the Remove content dialog box, click Remove to remove the document.

    Reordering documents in a communication container

    If you want to change the order in which documents appear in the output, then follow this procedure to change the order of the documents in the communication container.

    Note

    Reordering documents in a container does not create a new version of the container.

    To reorder documents in a communication container:

    1. In Design asset library view, click the Author panel.

    2. Optional To show only Content Author communications, communication containers, and documents, perform the following steps:

      1. In the Author panel, click the Filter assets button.

      2. In the asset list, select Communication (CA) and clear any other asset selections.

    3. To find the name of the container with the documents that you want to remove or reorder, perform the following steps:

      1. In the Author panel, click the row for a document that you want to remove.

      2. In the Properties panel, click the References tab, and then click Used by.

      3. In the Name column, note the name of the container that uses the document.

    4. In the Author panel, click the row for the container that you found in step 3.

    5. In the Properties panel for the selected container, on the General tab, perform either of the following tasks:

      Task

      Action

      To reorder documents in the selected container

      1. For the Documents list, click Reorder

      2. In the Documents dialog box, click the Reorder icon for a document, and then drag the document to its new position in the list.

      3. Repeat step 2 for each document that you want to reorder.

      4. Optionally, do one or both of the following:

        • Click Reset to revert to the original list order.

        • Click Remove beside a document to remove it from the list.

      5. Click Save.

      To remove documents from the selected container

      For each document that you want to remove: In the Documents list, click the document name, and then click Remove.
