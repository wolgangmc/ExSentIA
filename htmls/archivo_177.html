4.12   Setting up search keys

  • Communication asset library view
  • Design asset library view
  • Orchestration asset library view
  • Monitor view
  • Rationalization asset library view
  • Dashboards view

    Search keys let you add data to the print stream to both electronic and print output. Data you can add includes comments, searchable tags, instructions to print devices, and more.

    Since you can customize the output driver, value, timing, and other properties, search keys can be used to define unique methods of counting or as a way of inserting output device or post-processing commands (for example, specifying the stapling options).

    You can create the following types of search keys:

    • Library search keys—If you know you are creating a search key that will be used in multiple locations, you can create a search key that is stored as a resource in the asset library. Library search keys are placed in the print stream around content.

    • Local search keys—If your search key is for only one communication, you can create a local search key and place it directly in the content of the design. Local search keys are placed in the print stream within content.

    Creating a search key

    1. Do any of the following to create a search key:

      Task Action
      Create a library search key from the asset library

      1. In the Communication asset library, click the Delivery tab.

      2. Click Add and then click Search key.

      Create a local search key from the Communication Composer, in Designer

      1. From Designer, in the Communication Composer, select a communication.

      2. Click the Search Keys tab.

      3. Click the Add a search key button .

      4. From the Location list, select Local.

      Create an library search key from the Communication Composer, in Designer

      1. From Designer, in the Communication Composer, select a communication.

      2. Click the Search Keys tab.

      3. Click the Add a search key button .

      4. From the Location list, select Library.

    2. In the Name box, enter a name for the search key.

    3. Optional In the Description box, enter a description for the search key.

    4. In the Key box, enter the key text to be output.

    5. From the Timing list, select the timing of the search key if used in sorting and bundling. For more information about sorting and bundling, see section 9.7 "Sorting and bundling output" in OpenText Communications - Web Client User Help (CCWC250100-H-UGD)section 9.7 "Sorting and bundling output" in OpenText Communications - Web Client User Help (CCWC250100-H-UGD).

    6. From the Location in file list, select the location where you want to place the search key in the print stream.

      If you select Before named page or After named page, complete the following additional steps:

      1. To specify the named page, click in the Named page box, select the named page, and click OK.

      2. In the Pages to include list, select the pages on which the search key is placed:

        • All pages – Apply the search key to all instances of the named page.

        • First page – Apply the search key only to the first instance of the named page.

        • Last page – Apply the search key only if the named page is the last page in the output for a customer.

    7. From the Source list, complete the following steps to specify the content or value of the search key:

      Task

      Action

      Use a variable to set the value of the key

      1. In the Source list, select Variable.

      2. To select the variable that sets the value of the key, click in the Variable box, select a variable, and click OK.

      3. If you do not want to add a search key if the variable has no value, select Do not create if variable is empty.

      Use text to set the value of the key

      1. In the Source list, select Text.

      2. In the Text Value box, enter the text you want to use.

      Use the name of the object where you place the key, such as a page name or document name, as the value of the key

      In the Source list, select Name of object.

      Note

      For AFP output, a colon and space are added, followed by the data from the variable or text.

    8. From the Output type list, select which outputs receive search keys.

      • All outputs – Applies search keys to all outputs that support search keys. Search keys are supported only by AFP, PDF, and PostScript outputs.

      • Specified output type – Applies search keys only to the selected output type. To select the output type you want to use, choose an output from the PDL list.

      • Specified output queue – Applies search keys only to outputs included in a specified output queue. To select the output queue you want to use, click in the Queue box, and then select an output queue.

    9. From the Search key type list, select one of the following types of search keys to use in the output:

      • Default – Uses the default search key setting from the properties of the output object.

      • As is – Adds a record with no comment or NOP characters. This method can be used to add search data such as special indexing or Table of Contents tags. Use this method to enter custom printer commands and secondary device commands (for example, for staplers).

      • NOP comments – Prefixes the search key with the appropriate No Operation (NOP) comment value for the driver (such as %). This method is useful for adding information that can be used for indexing, archiving, future machine instructions, or a very large instruction word (VLIW).

      • TLEs – Supported only with AFP output. Adds Tagged Logical Elements (TLEs). TLEs serve as begin and end records in AFP output. These can be array variables so that each element is included as a separate entry in the print stream.

      • NOP comments and TLEs – Supported only with AFP output. Allows the engine to write search keys to both TLEs and NOP comments in AFP output.

    10. Click Create.

    11. Click Save.

    If you create a library search key, you can also define additional common settings. For information about these settings see the following locations:

    • Domains – For information about defining the domains to which a resource is assigned, see Section 3.13 “Associating resources with domains”.

    • Workflow – For information about managing an approval workflow, see Section 3.6 “Managing approval workflows”.

    • References – For information about viewing the linked resources connected to a selected resource, see Section 3.3 “Viewing resources in asset library views”.
