4.2   Configuring data sources

    To enable the OpenText Communications engine to connect customer data to your communications, you must map a data source using a sample data file. When you map a data source, you associate a variable with each data area in the sample file, which creates data maps that the engine uses to find customer data in your production data file. The engine then uses that data to populate variables and customize your communications.

    Designer uses the data source to identify which sample file to use for a particular data map. The same sample file can be used in multiple data sources, and be mapped differently for each data source with which is it associated. The data source defines several aspects of the file to be mapped. For example:

    • The name of the data source

    • The sample data file to be used for the data map

    • An associated design pack

    • The type of data source that it represents

    • The data source format

    • The production data source name

    In Communication asset library view, you use the Data panel to set up a data source for use in your communications, and associate a sample file and design pack with a data source. You then open the data source in Mapping view to map the data source. You can create variables and update variable properties in both Mapping view and the Data panel of the Communication asset library view.

    In Mapping view, you can map the data source using variables from the following sources:

    • Asset library—Variables created in either Mapping view or Communication asset library view are stored in the asset library and are available to all Designer data sources. Library variables that are mapped in a data source are also available to any communication that uses that data source.

    • Design pack—When you create a Design Manager application for use with Designer, you can include variables in the design pack. When you select a design pack to set up a data source in Designer, the variables in that design pack are available for use in that data source. For more information about configuring a design pack, see section 2.2.1 "Configuring a design pack for Communications Designer" in OpenText Exstream Design and Production - Design and Import Help (CCDPDI250100-H-UGD)section 2.2.1 "Configuring a design pack for Communications Designer" in OpenText Exstream Design and Production - Design and Import Help (CCDPDI250100-H-UGD).
