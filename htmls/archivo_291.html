    For a containerized OpenText Communications deployment, logging occurs in Kubernetes. Viewing these logs can be helpful if you're trying to troubleshoot issues in your environment.

    To view a log, use the kubectl logs command in the Kubernetes pod that you want to inspect. For more information about using this command for logging and debugging in Kubernetes, see the Kubernetes documentation.

    Note

    You must have access to your Kubernetes cluster to view logs for OpenText Communications deployments.

    The following table describes where to find the logs for each OpenText Communications component, and the information you can see in each log:

    Component Kubernetes pods Logged information
    Design exstream-design Information related to DAS resource storage
    Orchestration exstream-orchestration

    Information related to orchestration flow initiation, flow processing, and output channel delivery

    Interactive exstream-empower

    Information related to the Interactive service. These logs also include Interactive audit log entries, which start with audit: AuditLog: to differentiate them from other entries.

    Batch engine exstream-batch

    Information related to asynchronous engine processing for orchestration jobs

    On-demand engine exstream-ondemand

    Information related to synchronous engine processing for previews, simulations, and orchestration jobs
