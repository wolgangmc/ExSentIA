4.11   Setting up recipient profiles

  • Communication asset library view
  • Design asset library view
  • Orchestration asset library view
  • Monitor view
  • Rationalization asset library view
  • Dashboards view

    If you want to make copies of documents in a communication to send to recipients or destinations in addition to the customer, you can create these additional copies (known as recipient copies) using a data file to identify the data for the recipients of the copies.

    For example, suppose you are designing an insurance claim letter to be sent to a claimant (the customer). You might need to create the following copies:

    • A copy for an attorney, with the same documents as the claimant

    • A copy for the claimant's agent, without terms and conditions

    • A copy for long-term retention in an archive, with only the legally required information

    To create a recipient profile:

    1. In the Communication asset library, click the Delivery tab.

    2. Click Add and then click Recipient profile.

    3. In the Properties area, add a name and optionally a description.

      Names do not support the following special characters: & / : * ! ? > < | + # %

    4. Click Create.

    Defining recipient profile properties

    After you create the recipient profile object, you can use the Properties panel to view and modify its properties.

    Option Description
    Variable values can be updated Make variables editable so that end users can update recipient information—or even add new recipients—as needed.
    Included documents Select which documents to send to the recipient. If you do not specify any documents, the recipient copy includes all the documents in the customer set.
    Cover page Include a cover page for each copy that uses this recipient profile. Click the Cover page list to select a design page to use as the cover page.
    Recipient variables to include in Interactive documents Configure recipient variables so that end users can edit them in Interactive. Because you can control editability at the variable level, you can configure a recipient profile so that end users can edit only certain variables.

    If end users in Interactive add a new recipient or edit an existing recipient, the new or updated recipient information is saved only in that Interactive document and does not alter the recipient information in the database that is used to create the Interactive document.

    To include Interactive recipient variables:

    1. Click Add.

    2. Select a recipient variable for inclusion in an Interactive document. You cannot use system variables as recipient variables.

    3. Optional Add a label to identify the type of data that is used for the variable in the Recipient Viewer in Interactive. For example, “Phone number” for a variable that contains the recipient’s phone number.

    4. Optional Select this option to allow end users to make changes to the variable in Interactive.

    5. Optional To select a variable as the main identifier for the recipient in the Recipient Viewer in Interactive, click the More actions button at the end of the row, and then select Recipient name.

    If you want to remove a variable from this list, click the More actions button at the end of the row, and then select Remove.
