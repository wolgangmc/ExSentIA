4.3   Defining events

    Events are snippets of code that are executed for a communication at a specific point during the engine run. Each event can be defined to be executed before, during, or after customer processing, or at the end of the engine run. Unlike complex rules, events do not return a value and can be used to modify both global and local variables and to perform complex calculations. For example, you could use events to set a variable value that differs from the value in the customer data, to calculate running totals in a table, or to create custom page counts.

    Library events can optionally be stored in the DAS asset library for reuse across multiple locations. To avoid unexpected output changes, the settings on the General tab and Logic tab are view only. However, you can modify the following settings as needed:

    • Domains – For information about defining the domains to which a resource is assigned, see Section 3.13 “Associating resources with domains”.

    • Workflow – For information about managing an approval workflow, see Section 3.6 “Managing approval workflows”.

    • References – For information about viewing the linked resources connected to a selected resource, see Section 3.3 “Viewing resources in asset library views”.
