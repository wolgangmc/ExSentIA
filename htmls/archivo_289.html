10.5   Output review widget

    The Output review widget lets you preview the output for any flow models that are currently in a hold state. After reviewing the output, you can proceed with delivery or reject the output.

    To add an output review widget:

    1. Click the Dashboards button.

    2. Click Add widget.

    3. Enter a name and, optionally, a description for the widget.

      Names do not support the following special characters: & / : * ! ? > < | + # %

    4. In the Widget type list, select Output review.

    5. In the Flow model list, select the flow models that you would like to make available for preview. The resulting widget will only show the flow instances and steps that are in a hold state.

    6. In the Timeline list, select one of the following options: Last day, Last week, Last month (default), All time, or Custom.

      If you select Custom, use the From and To boxes to enter a range of dates. You can also click the calendar button to select a date.

    7. Click Create.

      After the widget is added, the dashboard changes to edit mode.

    8. Click the Dashboards button.

    9. Click Save.

    To preview output:

    1. In the widget list, expand the flow instance that you would like to preview.

      The individual flow steps that are in a hold (delayed) state are listed below the flow instance.

    2. Click the Preview icon for the flow step output that you would like to preview.

    3. In the Output file dialog box, use the tabs to review the contents for a selected output type.

      Note

      HTML-based output that includes embedded images, CSS, or other files may not preview accurately due to browser limitations.

    4. Click Close.

    To release or cancel a flow step or instance that is on hold:

    Depending on the results of the preview, you can use this widget to release or cancel individual flow steps or a flow instance. This is the same functionality provided in Monitor view.

    1. Select the flow step or flow instance that you want to release or cancel.

    2. In the row for the flow step or instance, click the More actions button .

    3. Click either Release or Cancel.

    Keep the following in mind about releasing and canceling:

    • Releasing or canceling a flow instance is recommended over releasing or canceling multiple flow steps.

    • Only one flow instance, or multiple flow steps within a single flow instance, can be released or canceled at a time.

    • A maximum of 1000 flow steps can be displayed per flow instance.

    • If all the flow steps in a flow instance are selected, the flow instance is also selected by default. Note that if a flow instance contains more than 1000 flow steps, and if the first 1000 are selected, then the entire flow instance is still selected.

    • If you select multiple flow steps, allow time for the loading icon to disappear; the release or cancel process is working in the background.
