2.5.2   Integrating OpenText Core Signature for secure document signing

    OpenText Core Signature is an application that allows trusted recipients to review and sign documents within a secure workflow. Designer integrates with OpenText Core Signature to allow you to add special placeholders known as signature objects to PDF page-based designs. Signature objects can be used as placeholders for document recipients to include signatures, dates, text, or check boxes.

    When a document is delivered to OpenText Core Signature, a signature request email is automatically generated and delivered to designated recipients. The email contains a link that opens the full output file in OpenText Core Signature and allows signers to locate the specific signature objects that they are required to complete. For more information about signing a document in the OpenText Core Signature app, see OpenText Core Signature User Help (CDSIGPK-H-UGD).

    Note

    For information about OpenText product compatibility, see the OpenText compatibly information provided in the release notes for the product. Release notes are available on My Support.

    Related information

    Define the OpenText Core Signature connection profile

    1. From the home page, click the Tenant Management button .

    2. In Tenant management view, click the Domains tab.

    3. Select the domain where you want to fulfill PDF output to OpenText Core Signature.

    4. In the Properties panel for the domain, click the OpenText Core Signature tab.

    5. Define the following parameters:

      Table 2.14:

      Parameter Description
      OpenText Core Signature URL

      The OpenText Core Signature service URL. For example, https://sign.core.opentext.com.

      OpenText Core Signature Token URL

      The OpenText Core Signature Token URL. For example, https://otdsauth.OCP.opentext.com.

      Tenant Id

      The ID of the tenant to which you want to post OpenText Core Signature data. This ID is located in OpenText™ Admin Center.

      Subscription

      The subscription name for OpenText Core Signature. This subscription name is located in Admin Center.

      Client ID

      The ID of the client that is used to connect to OpenText Core Signature.

      For more information about locating the equivalent setting in your OpenText Core Signature service, see OpenText Core Signature Service Developer guide.

      Secret

      The secret that is used to connect to OpenText Core Signature.

      For more information about locating the equivalent setting in your OpenText Core Signature service, see OpenText Core Signature Service Developer guide.

      Username and Password The OpenText Core Signature user name and password.

    6. Click Save.

    7. If you manage multiple domains, repeat step 2 through step 6 for each domain that you want to connect to OpenText Core Signature.

    Enable OpenText Core Signature designs for Designer

    1. From the home page, click the Tenant Management button .

    2. In the Tenant management view, click the Configurations tab.

    3. Click Designer.

    4. In the Properties panel, in the General tab, select the Enable OpenText Core Signature designs check box.

    5. Click Save.

    For information about adding the signature object in Designer, see  section 13.17 "Create signature objects" in OpenText Communications - Designer User Help (CCCD250100-H-UGD)section 13.17 "Create signature objects" in OpenText Communications - Designer User Help (CCCD250100-H-UGD).

    Create the signature request email

    1. From the home page, click the Design asset library button .

    2. In Design asset library view, click the Designer tab.

    3. Select the communication where you want to create a signature request email to use with OpenText Core Signature.

    4. In the Properties panel, click the Signatures tab.

    5. In the Subject box, enter the subject line for the email.

    6. In the Body box, enter the content of the email that will be delivered to signers by OpenText Core Signature.

    7. To specify the email addresses of signers, do one of the following:

      Table 2.15:

      Task Action

      Use an array variable to specify the email addresses of all signers.

      1. Select the Use array for signers' email addresses check box.

      2. In the Primary Signer’s email address box, select an array variable.

      When an array is used to specify the email addresses of signers, the first email address in the array is used for the primary signer, the second for the secondary signer, and so on.

      Use separate variables to specify the email addresses of different signers.

      1. In the Primary signer's email address box, select a variable to define the email addresses of primary signers.

      2. For each additional signer, click Add additional signer, and in the Additional signer's email address box, select a variable to define the email address of the signer.

      The order in which the variables are defined determines whether the signers will be primary, secondary, and so on.


    8. Click Save.
