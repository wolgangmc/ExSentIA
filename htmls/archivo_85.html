    Experience Customer Data Platform (CDP) enables access to rich customer data for advanced targeting and personalization across customer touch points. By capturing a customer’s online behaviors and associating those behaviors to an audience, OpenText Customer Data can build a profile of the user for targeting and deeper personalization. This deeper personalization can increase conversion rates by tailoring content and communications based on user preferences and interests. OpenText Customer Data is intended as a proxy to audience services to capture customer activity on a website, define audiences based on characteristics, and record audience membership in each customer’s profile. OpenText Customer Data can also integrate with Google Analytics or Google Big Query to allow for additional audience providers.

    For more information about the OpenText Customer Data platform, see OpenText Customer Data Cloud Deployment Guide (CDPEXPCDP-UAG).

    Note

    For information about OpenText product compatibility, see the OpenText compatibly information provided in the release notes for the product. Release notes are available on My Support.

    When you use OpenText Customer Data, you can use the collected audience data in your designs to better target and communicate with your customers. From the web client, you can activate OpenText Customer Data so that you can leverage this collected audience data using variables in your Designer and Content Author designs.

    To integrate OpenText Customer Data with Designer and Content Author designs, you must complete the following high level steps:

    Task Information

    1 In your OpenText Customer Data hub, define any necessary permissions or authentications.

    See OpenText Customer Data Administration Site Guide (CDPEXPCDP).

    Note

    For information about OpenText product compatibility, see the OpenText compatibly information provided in the release notes for the product. Release notes are available on My Support.

    2 In web client, activate OpenText Customer Data so that it can be used in Designer and Content Author designs.

    See Activating OpenText Customer Data for use in Designer and Content Author designs .

    3 In web client, create variables that can leverage OpenText Customer Data audience data.

    See Creating variables that leverage OpenText Customer Data audience data.

    4 In Designer and Content Author, apply OpenText Customer Data variables to the design.

    In Designer and Content Author you can apply OpenText Customer Data variables anywhere that variables are used in a design, such as inserting variables in line with text or using variables with custom logic.

    5 In Designer and Content Author, target content based on audience data.

    In a few specific scenarios, when you use OpenText Customer Data variables, you are prompted to select the OpenText Customer Data audience data so that you can target designs and design content based on OpenText Customer Data audiences.

    See Targeting content based on audience data.

    Activating OpenText Customer Data for use in Designer and Content Author designs

    This task assumes that you have set up and defined your OpenText Customer Data hub with any necessary permissions or authentications. For more information about completing this setup, see OpenText Customer Data - Cloud Deployment Guide (CDPEXPCDP-UAG)OpenText Customer Data - Cloud Deployment Guide (CDPEXPCDP-UAG).

    To activate OpenText Customer Data for use in Designer and Content Author designs:

    1. In the web client, click the Tenant Management button .

    2. On the Tenant management page, click the Configurations tab.

    3. In the Name area, select Designer.

    4. In the Properties panel, on the General tab, select the Data Platform check box.

    5. In the Data Platform URL box, enter the URL of the hub service.

    6. To test the URL, click Validate. The URL is validated to confirm that you have access to the OpenText Customer Data service. If you receive a failure, contact your OpenText Customer Data administrator.

    Creating variables that leverage OpenText Customer Data audience data

    Before you create variables that leverage OpenText Customer Data audience data you must first meet the following prerequisite: The Data Platform settings must be active on the Tenant. For information about activating OpenText Customer Data for use in Designer and Content Author designs, see Activating OpenText Customer Data for use in Designer and Content Author designs

    To create variables the leverage OpenText Customer Data audience data:

    1. Create a new variable with any configurations needed.

      For more information about creating a variable, see Section 4.6.2 “Creating variables”.

    2. In the web client, from the home page, click the Communication asset library button .

    3. Click the Data tab.

    4. In the Library, select the variable that you created to leverage OpenText Customer Data data.

    5. In the Properties panel, on the Value tab, complete the following settings:

      1. Select the Enable selection list check box.

      2. In the Selection list area, select OpenText Customer Data.

      3. If you want to filter the tags that are allowed when this variable is selected, select the Enable filters check box and then select the tags that you want to allow.

    6. Map the variable to the OpenText Customer Data customer driver file that was provided. This customer driver file contains the collected audience data.

      If you have questions about what customer driver file to use, contact your OpenText Customer Data administrator.

      OpenText Customer Data variables are mapped exactly the same as any other variable. For more information about mapping variables, see Section 4.2.3 “Mapping data sources”.

    Targeting content based on audience data

    If your Designer or Content Author designs includes OpenText Customer Data variables, you can target designs and design content based on OpenText Customer Data audiences. in the following scenarios:

    • Targeting design variations based on audience data

    • Targeting custom rules based on audience data

    • Targeting simulation previews based on audience data

    Targeting design variations based on audience data

    Design variations allow you to create different designs that vary based on customer data. You can use variations as a simpler alternative to targeting rules that determine which content goes to which customers. With variations, you can use a single design as your base (the standard variation) and then customize communications for specific groups of customers by creating as many variations of that design (the alternate variations) as you need. Best of all, you can easily see all of the variations that are associated with a design as you work in Design view. Variations essentially take the place of targeting rules, so targeting rules that you apply to an alternate variation will not be honored by the engine; you can, however, apply targeting rules to a standard variation, and those rules will be honored by the engine.

    When creating a variation in a design that uses OpenText Customer Data, you are prompted to identify the audience data that is unique to each variation. The available audience data available is determined by the available OpenText Customer Data variables and OpenText Customer Data data.

    If you have questions about the available audience data, contact your OpenText Customer Data administrator.

    To target a variation based on audience data:

    1. Create a variation and customize the settings as needed.

      For more information creating variations for use in Designer , see section 25 "Design variations" in OpenText Communications - Designer User Help (CCCD250100-H-UGD)section 25 "Design variations" in OpenText Communications - Designer User Help (CCCD250100-H-UGD).

      For more information creating variations for use in Content Author, see section 2.22.1 "Creating variations of text messages, graphic messages, and clauses" in OpenText Communications - Content Author User Help (CCCA250100-H-UGD)section 2.22.1 "Creating variations of text messages, graphic messages, and clauses" in OpenText Communications - Content Author User Help (CCCA250100-H-UGD).

    2. In the Audiences area, select the check box for the audience data that you want to use.

      Tip

      To filter the audience data so that you can find the data that you want to use, you can click the Tag filter button and in the Available tags area, you can select the tags that you want to view in the Audiences dialog box.

    3. Click Create.

    Targeting custom rules based on audience data

    This task is specific to target custom rules based on audience data. If you create a custom rule based on an OpenText Customer Data variable, you are then prompted to select the audience data that you want to use to compare against the rule logic.

    For more information about creating rules for use in Designer, see section 22 "Rules" in OpenText Communications - Designer User Help (CCCD250100-H-UGD)section 22 "Rules" in OpenText Communications - Designer User Help (CCCD250100-H-UGD).

    For more information about creating rules for use in Content Author, see section 2.19 "Adding rules to content and communications" in OpenText Communications - Content Author User Help (CCCA250100-H-UGD)section 2.19 "Adding rules to content and communications" in OpenText Communications - Content Author User Help (CCCA250100-H-UGD).

    To target custom rules based on audience data:

    1. In the Rule Composer, click the Custom rule tab.

    2. To create a condition for your rule, select whether to include or exclude content based on the rule.

    3. Complete the following to specify how an OpenText Customer Data variable is used in a custom rule:

      1. In the Rule subject list, select the OpenText Customer Data variable that you want to use as a base for the rule.

      2. In the Comparison list, select the type of comparison the rule will make.

      3. In the Select an Audience box, click the Select an Audience button .

        Tip

        To filter the audience data so that you can find the data that you want to use, you can click the Tag filter button and in the Available tags area, you can select the tags that you want to view in the Audiences dialog box.

      4. In the Audiences dialog box, select the check box for the audience data that you want to use.

      5. Click OK.

    4. Optionally, click the Add component button to add another condition to your rule, or click the Remove values button to remove a condition.

    5. Click Next.

    6. If the communication is based on a communication set, use the Location list to select whether the rule should be saved locally or as a library rule.

      • You can switch the Local or Library option after a rule is created.

      • Creating a library rule stores the rule in DAS. It can be added to a communication set and has an approval workflow.

    7. Specify a name and, optionally, a description for the rule.

      Names do not support the following special characters: & / : * ! ? > < | + # %

    8. Click Save and Insert.

    After you have applied the rule, the targeted content is included or excluded in customer communications based on whether or not the customer data meets the conditions of the rule. You can use Simulation view to see how the rule affects your content in different data scenarios.

    For more information about previewing OpenText Customer Data audience data in different data scenarios, see Targeting simulation previews based on audience data

    Targeting simulation previews based on audience data

    Before you generate production output, you can use Simulation view to run the engine in preview mode to see your communication as it would appear in the final output. In Simulation view, the engine runs in preview mode to create output that is displayed in the browser window. If you are using OpenText Customer Data, you can preview the alternate appearances for variations and audience data in the simulation by selecting the audience data you want to preview.

    This task discusses targeting Simulation preview based on audience data; however, there are multiple ways to configure the appearance of the simulation preview.

    For general information about simulation preview and other configuration options in Designer, see section 26 "Previewing communications" in OpenText Communications - Designer User Help (CCCD250100-H-UGD)section 26 "Previewing communications" in OpenText Communications - Designer User Help (CCCD250100-H-UGD).

    For general information about simulation preview and other configuration options in Content Author, see section 3 "Viewing a communication in different data scenarios" in OpenText Communications - Content Author User Help (CCCA250100-H-UGD)section 3 "Viewing a communication in different data scenarios" in OpenText Communications - Content Author User Help (CCCA250100-H-UGD).

    To target simulation previews based on audience data:

    1. In the Variations panel, click the Edit button for the OpenText Customer Data variable that controls the design variation you want to preview.

    2. In the Audiences dialog box, select the audience data that you want to preview, and click OK.

    3. Click Run.

    To reset the variables to their original values, click Reset. Variable values are also reset to the original values if you select a different sample file.
