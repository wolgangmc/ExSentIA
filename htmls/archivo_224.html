8.2.2   Prerequisites for using Content Author

    Content Author lets business users add targeted content to communication designs that are created in Design and Production or in Designer. To use Content Author, these communications must be set up correctly and available to Content Author users.

    The following table provides an overview of the minimum requirement to enable business users to edit Content Author communications:

    Table 8.2: Prerequisites for using Content Author

    Requirement

    Explanation

    A communication design that contains Content Author frames, either in a communication or in a document in a communication container

    In Content Author , users can add content only to those frames that are designated for use with Content Author. The communication designs that are used to create Content Author templates must include at least one Content Author frame. Communication designs can be created in the following ways:

    • Using Designer—Create design layouts in a Designer communication.

    • Using Design and Production—Create a Design Manager application and use Designer for Design and Production to create the design layout. The Design Manager application must contain the following objects:

      Design pack

      Design packs contain resources for customizing a communication, such as fonts, style sheets, colors, rules, and variables. The Design Manager application must contain at least one design pack.

      Design views

      Design views determine which preview options are available in Content Author. Each design view corresponds to a specific preview device. By default, applications are automatically configured to use the standard design view for print preview.

      Output queues

      Output queues determine the output types that can be generated when the communication is processed using the OpenText Communications engine.

      Data files

      At least one customer driver file is required to produce output from the communication.

    For more information, see section 2.3.3 "Creating a design for Content Author" in OpenText Exstream Design and Production - Design and Import Help (CCDPDI250100-H-UGD)section 2.3.3 "Creating a design for Content Author" in OpenText Exstream Design and Production - Design and Import Help (CCDPDI250100-H-UGD) and section 18 "Designing for Content Author" in OpenText Communications - Designer User Help (CCCD250100-H-UGD)section 18 "Designing for Content Author" in OpenText Communications - Designer User Help (CCCD250100-H-UGD)

    One or both of the following templates:

    • An approved Content Author communication template

    • An approved Content Author communication container template with at least one document template

    To create a new Content Author communication, communication container, or document, users select an approved template in Communication launcher view.

    Content Author communication templates can be created in either of the following ways:

    • Using Designer—Create a Content Author template in Designer.

    • Using Design and Production—Package an appropriate Design Manager application and upload it to DAS to create a corresponding template and engine run definition for that application. An engine run definition is required for communication previews.

    For more information, see Section 8.2.3 “Starting a Content Author communication”.

    Content Author communication container templates and document templates are created by uploading the following packages from Design Manager to DAS:

    • An application package for an application that was packaged with the check box Enable split communication option in Content Author selected. The application package template can function as a communication container template.

    • One or more Single Document subpackages. After you link the subpackages to the application package above, the templates for the linked subpackages can function as document templates for documents in a communication container.

    For more information, see section 8.2.1.1 "Generating templates for Content Author communication containers and documents" in OpenText Communications - Web Client User Help (CCWC250100-H-UGD)section 8.2.1.1 "Generating templates for Content Author communication containers and documents" in OpenText Communications - Web Client User Help (CCWC250100-H-UGD) and section 8.2.4 "Starting a Content Author communication container" in OpenText Communications - Web Client User Help (CCWC250100-H-UGD)section 8.2.4 "Starting a Content Author communication container" in OpenText Communications - Web Client User Help (CCWC250100-H-UGD).

    Note

    In Design asset library view, you can filter assets on the Templates panel by Communication (CA) to list only the templates for Content Author communications, communication containers, and documents.

    User permissions to access and work in Content Author

    Content Author users must be members of at least one OTDS group that is assigned a role with at least the Access and Creator web client access permissions enabled for Content Author. For reviewers, the Approver option must also be enabled.

    For more information, see Configuring roles and permissions
