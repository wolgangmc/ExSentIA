    Understanding engine processing and timing helps you to improve the overall production of communications by showing you how the layout and order of your design can ultimately impact your output. Additionally, you can better identify and resolve unexpected results if you understand how the engine processes customer data.

    Engine processing can differ depending on the format and layout of your customer data, and depending on whether you are using a two-pass engine process to complete the output sorting and bundling process. Depending on your communication setup, use the following content for additional information about the engine process and how the engine applies timing to your communication:

    Task Information

    You are using a data file with a general layout or with data segments

    		Section 9.8.1 “Engine processing with general data and data segments”

    You are using a data file with a general layout or with data segments, and you are using two-pass processing for output sorting and bundling

    • For pre-sort processing, see Section 9.8.1 “Engine processing with general data and data segments”

    • For post-sort processing, see Section 9.8.2 “Post-Sort engine processing with general data and data segments”
