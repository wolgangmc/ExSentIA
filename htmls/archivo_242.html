8.3.2.2.1   Understanding the status indicators for migration sets

    You can use the status indicators to quickly check the status for a migration set. The status column uses color indicators to identify the status of migration sets, and you can point to any of the status indicators to see a tooltip that provides more information.

    Note

    Although a migration set can be associated with multiple domains, the status indicator that is displayed applies only to the current domain.

    The following list shows the status indicator colors that are available for migration sets, and what each color means:

    • (gray)—The migration set was created (or is in the process of being created), but it has not yet been rationalized.

    • (yellow)—The migration set was rationalized but is not yet associated with a communication or data source.

    • (green)—The migration set was rationalized and is associated with a communication or data source.

    • (red and yellow)—New content has been added to a migration set that was previously in the yellow state, but it has not yet been rationalized.

    • (red and green)—New content has been added to a migration set that was previously in the green state, but it has not yet been rationalized.

    Tip

    While rationalization is in progress, you can view the percent completed by positioning the pointer over the status indicator.
