    You can use data segments to group records that are related to one particular type of information, such as all of the debits or all of the deposits for an account.

    While you can use data segment data in a variety of ways, the following are some common use cases:

    Task Benefit

    Combine data segments with tables

    Since data segments help identify groups of similar data, a data segment is especially useful when the order of the data is different for each customer, or when the number of data types varies for each customer. Data segments are also useful in communications that are similar in design but that contain different types of data, and for transactional communications. A data segment lets you design a table that can be used with multiple sets of data for a customer, particularly in transactional communications or in household mailings.

    For more information, see section 13.6.4 "Using table segments to organize table content" in OpenText Communications - Designer User Help (CCCD250100-H-UGD)section 13.6.4 "Using table segments to organize table content" in OpenText Communications - Designer User Help (CCCD250100-H-UGD).

    Drive document inclusion with data segments

    When creating new documents, you can use the presence of data segments to determine when to include specific documents, which can also repeat depending on the data. For example, each bank account in a bank statement represents a different data segment. If a customer has multiple checking accounts, then those accounts represent multiple instances of a particular type of data segment, and the document would repeat in customer output according to the number of checking accounts that the customer has.

    For more information, see section 7.11 "Using data segments to drive document inclusion" in OpenText Communications - Designer User Help (CCCD250100-H-UGD)section 7.11 "Using data segments to drive document inclusion" in OpenText Communications - Designer User Help (CCCD250100-H-UGD).

    To mark the start of a new data segment:

    1. Select the data area that you want to use as the start of a new data segment. Typically this would be the starting tag of a group of data.

    2. In the toolbar, click the Start data segment button .

      Note

      Clicking the Start data segment button again removes the Start of data segment designation.

    3. In the Data segment dialog box, specify a name and select one of the following occurrence options:

      • Always starts a new data segment—Starts a new data segment at each equivalent data area in the file.

      • First occurrence only—Starts a new data segment only the first time it appears in the customer.

    4. Click Insert.

    You can use the Data segment properties panel to make changes as needed.
