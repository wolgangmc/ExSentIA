8.2.1.1   Generating templates for Content Author communication containers and documents

    If an uploaded Design Manager application package has the option to split communications in Content Author, you can generate a communication container template and its document templates in DAS by linking the application package to its single-document subpackages in DAS.

    In addition to allowing communications to split in Content Author, the application package must come from a “base” application that includes design packs but no designs, such as pages and emails. Also, the single-document subpackages that you link to the base application package in DAS must include the designs, and they must have been packaged from the base application in Design Manager.

    The following procedures summarize how to create a base application package with single-document subpackages in Design Manager, and how to link them in DAS to generate a container template with associated document templates.

    Note

    For detailed instructions on creating and packaging applications and subpackages in Design Manager, see OpenText Exstream Design and Production - Output, Production, and Delivery Help (CCDPOPD250100-H-UGD)OpenText Exstream Design and Production - Output, Production, and Delivery Help (CCDPOPD250100-H-UGD).

    To create a base application with single-document subpackages in Design Manager:

    1. Create a base Design Manager application with design packs, but no designs.

    2. Create one or more Design Manager single documents that include designs.

    3. Package the base application with the check box Enable split communication option in Content Author selected in the DAS Package Details dialog box.

      Note

      To open the DAS Package Details dialog box, click the Edit button following the Upload package file to DAS domain check box.

    4. Package each single document as a subpackage of the base application.

    5. Upload the base application package and single-document subpackages to DAS.

    The template for the base application package and the templates for the single-document subpackages are generated in DAS on upload. After the base application package and its single-document subpackages are linked in DAS (as described next), their templates will function as a container and document templates, respectively.

    To link single-document subpackages to their base application package in DAS:

    1. In Orchestration asset library view, click the Packages panel.

    2. Click the base application package that was packaged in Design Manager with Enable split communication option in Content Author selected.

      Note

      The selected application package must be in the draft state and not reserved by another user.

    3. In the Properties panel, on the General tab, add one or more single-document subpackages of the base application to the Linked subpackages list.

      For more information on linking subpackages to an application package, see Managing application packages and subpackages .
