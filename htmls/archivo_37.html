    To manage and use individual tenant environments in OpenText Communications, you must set up the appropriate user groups in OTDS and add at least one user to each group. The following table lists the minimum OTDS groups that are required in each new tenant.

    Table 2.3: Required OTDS groups

    OTDS group Purpose
    tenantadmins

    Tenant administrator users must be members of the tenantadmins group in the corresponding OTDS tenant.

    tenantusers

    Tenant users that are not part of another required group should be included as members of the tenantusers group in the corresponding OTDS tenant.

    empoweradmins

    Interactive administrators in an Interactive tenant must be members of the empoweradmins group in the corresponding OTDS tenant.

    empowerintegrators

    Interactive system integrators in an Interactive tenant must be members of the empowerintegrators group in the corresponding OTDS tenant.

    empowereditors

    Interactive users for an Interactive tenant must be members of the empowereditors group in the corresponding OTDS tenant.


    In addition to these groups, you might need to create additional OTDS groups for various users depending on your organization’s needs. See the default OTDS groups in the sample OTDS tenant for an example of a typical deployment.

    For detailed information about creating users and groups in OTDS, see OpenText Directory Services Web Client Help (OTDS-H-AWC).

    Note

    For information about OpenText product compatibility, see the OpenText compatibly information provided in the release notes for the product. Release notes are available on My Support.
