5.5   Adding fonts

  • Communication asset library view

  • Orchestration asset library view
  • Monitor view
  • Rationalization asset library view
  • Dashboards view

    You can use the Common panel in the Design asset library view to add fonts to the asset library for use in a communication. These fonts supplement any fonts that are included in an associated design pack, and will be available for all Designer users. You can also enable fonts to be used with Content Author designs. Fonts must be in an approved state in order to be used in a design.

    Note

    When using a design pack as the basis for a communication, keep the following in mind:

    • If the design pack includes a style sheet, Designer will honor the font information from the style sheet over fonts from DAS.

    • If you are using Content Author, only fonts included in the design pack will be available within the editor (that is, if you add any DAS fonts to the design in Designer, they will not be available while editing or adding content directly within Content Author).

    When you add a font file to Designer, you are technically adding a font family, rather than just an individual font. Some font families are made up of two or more subfamilies. This means that if you add a font, and then later add another font file that is part of a subfamily in that font (such as a bold serif version of the font), it will not appear as a separate font in the asset library list. Instead, it will be added to the properties of the original font family. The distinction between family and subfamily can vary based on the font and how it was created.

    If you add a variable font in the form of an OpenType Font Variations (TTF) file, you will see only the default font face in the Design view font list (for example, “Regular”). Any subfamily font faces that are incorporated in the font file—such as a bold or italic version—will not be available for use.

    Each user needs to have a local installation of any fonts that are used in the communication that they are editing. If a user's workstation does not have a font that is used in the design, then the default font for the communication will be used instead. Because fonts are generally distributed in the same manner as software—that is, they are licensed rather than sold outright—many font licenses place restrictions on how fonts can be used. Before installing and using any fonts in your communication, be sure that you are complying with any font licensing restrictions. For example, while some fonts cannot be embedded as part of a file, others can be embedded but not edited—instead, they are used only for the purposes of printing or previewing a file. Conversely, fonts without licensing restrictions can be edited, printed, and added to end users' systems without limitations.

    Note

    If you have questions about font installation or licensing restrictions, talk to your system administrator. For information about the licensing restrictions of a particular font, contact the vendor that supplied the font.

    You can also designate any approved font to be used as the default font. Default fonts are set in Tenant management view, and are used for all applicable designs in the tenant.

    To add a font:

    1. In the Common panel, click Add, then click Font in the selection list.

    2. In the file browser, select the font file that you want to add, then click Open.

      Note

      Only OTF and TTF formats are supported. If the font that you want to add already exists in the asset library, the existing font information is updated and a new font resource is not added.

    3. In the Properties panel, review the properties of the selected font and optionally, enter a description for the font resource.

    4. Click Create.

    If you want to make the font available for use in Content Author, once the font has been created, select Enable for use in Content Author in the Properties panel.
