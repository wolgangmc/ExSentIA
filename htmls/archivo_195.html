    You can create reusable HTML code as a resource that designers can apply to web designs using one of the following methods:

    • Create an HTML placeholder variable – OpenText Communications supports importing HTML content at run time by using an HTML placeholder variable. This feature is intended to be used with snippets of HTML content rather than complex web communications. For example, suppose that your marketing organization produces promotional content in HTML for use on the company website. You can import this HTML content at run time so that the promotions appear inline along with content that was created in Designer. For more information about using HTML passthrough to import external content, see Section 8.3.1 “Considerations for importing HTML content”.

    • Create an HTML code object – OpenText Communications supports creating a reusable resource that is stored in the Design asset library view that can be created and managed by administrators. Designers can then select from a predefined library of HTML objects for specific HTML code to include in a web design. For more information about creating an HTML code object, see Creating an HTML code object.

    Security considerations when adding HTML code

    Before you use any object that leverages external HTML content, it is important to understand the risks and security considerations of using HTML code. Make sure that you have assessed and completed a security review of the following considerations before using any HTML code:

    Important

    The following security guidelines apply regardless of the method you choose to use when adding HTML code to your application.

    Consideration Description
    Adhere to your company’s security policy Your company might have guidelines about security measures that should be taken before you ship HTML content to customers. For example, you might complete the following analysis as a security best practice:

    • Complete a static analysis of the source code to identify poor security practices.

    • Complete an analysis of dependencies to identify known third-party vulnerabilities.

    Use trusted content delivery networks and libraries

    HTML can reference existing external libraries, such as jQuery, on a content delivery network. If these networks are not trusted, they can become compromised and the libraries can be modified to use malicious code.

    Make sure you are using a trusted content delivery network. If a trusted content delivery network is not available, you should scan the source and bundle external libraries so that they cannot be changed without the use of the OpenText Communications workflow.

    Minimize disclosure of sensitive information to third parties

    HTML code can include resources, such as JavaScript, that might send data to a third party.

    To mitigate this concern while designing and testing your application within the OpenText Communications simulation environment, you can identify the allowed URLs for external resources that can be accessed, such as scripts. You can identify both a global list and a local list of allowed URLs. This action blocks any request unless you specify the resource in the Allowed script and HTTP URLs for simulation list. For more information about specifying allowed URLs and scripts for simulation, see Specifying the allowed URLs for simulation with HTML code.

    Important

    This process only limits allowed URLs for external resources during the OpenText Communications simulation process. This limitation is not extended to the final output. It is recommended that you complete any output specific code protections before generating the final output.

    Specifying the allowed URLs for simulation with HTML code

    Specifying whether to allow a URL for simulation globally or locally might depend on how broadly you want to allow access to that URL. For example, for a global list, you might add sites that have a single uses, such as Google Maps. Alternatively, for a local list, you might restrict access to a specific URL if you make calls to a site that is hosted by a service.

    To specify a global list of allowed URLs for simulation

    To identify a global list of URL sources that are permitted during simulation for HTML code that contains external requests, such as scripts or endpoints:

    1. Click Tenant management .

    2. On the Configurations tab, click Designer.

    3. In the Properties panel, on the General tab, in the Allowed script and HTTP URLs for simulation box, enter the URL.

    To specify a local list of allowed URLs for simulation

    To identify a local list of URL sources that are permitted during simulation for HTML code that contains external requests, such as scripts or endpoints:

    1. Click Design asset library view .

    2. In the Common panel, create or select an HTML object in the selection list.

    3. In the Properties panel, on the General tab, in the Allowed script and HTTP URLs for simulation box, enter the URL.

    Creating an HTML code object

    1. Click Design asset library view .

    2. In the Common panel, click Add, then click HTML in the selection list.

    3. In the Properties panel, enter a name and optionally a description.

      Names do not support the following special characters: & / : * ! ? > < | + # %

    4. To add code to the object, complete the following steps:

      1. Click HTML composer.

      2. In the HTML Composer window, enter or upload your HTML code.

      3. Click Save.

    5. Click Create.
