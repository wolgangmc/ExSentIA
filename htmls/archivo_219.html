8.1.3   Creating a Designer design layout

  • Communication asset library view

  • Orchestration asset library view

  • Rationalization asset library view

    Organizations often use design layouts when they have corporate standards they want to enforce. Design layouts simplify document creation by pre-setting components within a new document. Design layouts use paper types and additional settings to ensure that pages have a consistent design, even if multiple people create the designs. Design layouts are designed like a page, but because they have additional features, you can set defaults that cannot be overridden in associated designs.

    Design layouts are created in the OpenText Communications web client and then, in Designer, you can apply design layouts to one or more page designs. Approved design layouts are available for selection in the Properties panel when creating a a page design.

    For more information about applying a design layout to a page design, see OpenText Communications - Designer User Help (CCCD250100-H-UGD)OpenText Communications - Designer User Help (CCCD250100-H-UGD).

    Some potential advantages of using a design layout include the following:

    • Allows you to create a reusable template for page designs, which can provide a uniform look across multiple communications

    • Facilitates adherence to company design standards

    • Reduces repetitive design creation

    • Provides the ability to update the layout of a page design without modifying each instance that requires the change

    For example, suppose you want to create a template for correspondence using a design layout. When a user creates a new page using the design layout, the letterhead, date, watermark, and body text box are already set up for the page. The design user only has to create the content.

    When a design layout is in use for a page design, the selected layout controls the paper type, style sheet, and duplex options for that design.

    Note

    Before you can create a design layout, you must have already created a paper type. Paper types define the properties of the sheet of paper that corresponds to the page.

    Creating a design layout

    1. In the OpenText Communications web client, click the Communication launcher button on the application toolbar.

    2. In the Design layouts panel, click the New button.

    3. Specify a name and, optionally, a description for the design layout that you want to create.

      Names do not support the following special characters: & / : * ! ? > < | + # %

    4. Optional If you want to use resources from a design pack, click the Use design pack check box, and then select the design pack you would like to use.

    5. Select a Paper type to use as the basis for the design layout.

    6. Optional In the Style sheet list, select a style sheet. Design pack-based style sheets are identified with a (design pack) label. Communication set based style sheets only appear in the list if they are in an approved state. If the communication did not include a style sheet and if no style sheets are included in the client, the only option is None. For more information about style sheets, see Section 5.14 “Using style sheets to enforce corporate styles”.

    7. To specify how single page (simplex) design pages should be printed when the output queue is set to duplex, in the How to print simplex page on duplex outputs list, select one of the following options:

      • Starts on front or back, must match paper type—The page can be placed on the front or back of any sheet of paper, as long as the paper types match.

      • Starts on front, matching paper type allowed on back—The page is always on the front of a sheet of paper. Anything, as long as it uses the same paper type, can be placed on the back.

      • Starts on front, nothing on back—The page is always on the front of a sheet of paper. Nothing can be placed on the back of the sheet.

      • Starts on front or back, different paper types allowed—The page can be placed on the front or back of a sheet of paper, and can use any available paper type.

      • Starts on front, different paper types allowed—The page can be placed only on the front of a sheet of paper and can use any available paper type.

    8. Optional If available, in the Category tags applied area, select any tags you want to apply.

    9. Click Create.

    Opening a design layout

    1. In the Design layouts panel, click one of the following buttons:

      • Library—View all of the existing design layouts

      • Recent—View a list of design layouts that you have recently opened in Designer

      • Favorites—View a list of design layouts that have been bookmarked

        Tip

        If you work frequently with a specific design layout, you can bookmark it so that it appears in the Favorites list. To bookmark a design layout, point to it in the Library or Recent tab, and click the Bookmark icon in the Favorites column.

    2. Select the design layout that you want to open, and click Open.

      The design layout opens in Designer Design view, where you can edit the layout as you would for a standard page design.

      Note

      The following objects are not supported for design layouts: frames, flow objects and properties, table of contents, and variations.
