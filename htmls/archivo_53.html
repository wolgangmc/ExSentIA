2.3.1.2   Configuring permissions for a role

  • Communication asset library view
  • Design asset library view
  • Orchestration asset library view
  • Monitor view
  • Rationalization asset library view
  • Dashboards view

    1. In the Roles panel in Tenant management view, select the role.

    2. In the Properties panel, click the Permissions tab.

      1. In the Web client access area, select the permission check boxes for each item in the Type list as needed. Note that if you want to provide Creator or Approver permissions to a user, you must also provide Access permission to them.

        The following table describes the permissions that are granted in the web client for each permission type, and whether a Named User Seat is required. If a Named User Seat is required, you must allocate either a Designer or Content Author counter to the user in the OTDS tenant.

        Note

        When you assign the Creator permission for some of the asset library types, the Approver permission for auto-approved resources in that library type is automatically assigned.

        • When you assign the Creator permission for some of the asset library types, the Approver permission for auto-approved resources in that library type is automatically assigned.

        • When you select a check box, permissions for the corresponding resources are automatically selected in the Resource access area. A partial selection icon indicates that some of the corresponding resources in the list are checked, but not all of them.

        The following table describes the default permissions that are assigned to a role. Default permissions can be modified by an administrator to add or remove access to features for a user role.

        Type Access Creator Approver
        Designer Enables the Designer launcher panel in Communication launcher view and the Design tab in the Design asset library view, and allows users to open communications in Designer Allows users to create, update, and delete Designer resources Allows users to approve Designer resources
        Content Author Enables the Content Author launcher panel in Communication launcher view and the Author tab in the Design asset library view, and allows users to open communications in Content Author Allows users to create, update, and delete Content Author resources Allows users to approve Content Author resources
        Communication asset library Allows users to access Communication asset library view and Mapping view Allows users to create, update, and delete resources in Communication asset library view and to map data sources in Mapping view Allows users to approve resources in Communication asset library view and Mapping view
        Design asset library Allows users to access Design asset library view Allows users to create, update, and delete resources on the Common tab in Design asset library view Allows users to approve resources on the Common tab in Design asset library view
        Orchestration asset library Allows users to access Orchestration asset library view Allows users to create, update, and delete resources in Orchestration asset library view Allows users to approve resources in Orchestration asset library view
        Rationalization asset library 1 Allows Designer or Content Author users to access Rationalization asset library view Allows a Designer or Content Author user to create, update, and delete resources in Rationalization asset library view. Allows a Designer or Content Author user to approve resources in Rationalization asset library view.
        Monitor Allows users to access Monitor view Not available Not available
        Monitor job files Allows users to access orchestration job input and output files

        Note

        This option protects sensitive data—such as Personal Identifiable Information (PII)—by only allowing specific roles access to engine input files, delivery reports, and output files.

        		Not available Not available
        Categories Allows users to create new category tags outside of Tenant management view Not available Not available
        Dashboard Allows users to access Dashboards view Not available Not available

        1 Rationalization must be enabled in your product key. In order to create migration sets and work with content fragments, you must have a Named User Seat for Designer or Content Author.

    3. Optional To set specific resource permissions, expand the Advanced section in the Resource access area, then select Create, Write, Delete, and Approve permissions for individual resource types as needed.

      Tip

      To apply a selected permission option to all resources in the DAS asset library, select the All check box in the Resource type list.

      Keep in mind that you do not need to explicitly set permissions to read resources in a domain. A user that is assigned a role that has any permission in a domain, for example, Access permission for Design asset library—or even no permission at all—will also have read access to all resources in that domain.

    4. Click Save.
