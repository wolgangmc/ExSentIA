8.2.3   Starting a Content Author communication

    You begin in Communication launcher view in the OpenText Communications web client by selecting from a list of existing communications or by creating a new communication from a list of templates.

    To access Communication launcher view, click the Communication launcher button on the application toolbar in the OpenText Communications web client.

    If you have questions about the templates that are available, talk to the person or people who designs communications in your OpenText Communications environment.

    Creating a new communication

    1. In the Communications panel, click the New button to see a list of available templates.

      Note

      If you want to create a new communication container with documents, see Section 8.2.4 “Starting a Content Author communication container”.

    2. In the Template name column, select a template. Only templates that have been approved are visible in this column.

    3. If you are not sure which template to select, click Preview to see a preview of a selected template. The Template preview dialog box displays print and HTML previews, as available.

    4. Click Next.

    5. Specify a name and, optionally, a description for the communication that you want to create.

      Names do not support the following special characters: & / : * ! ? > < | + # %

    6. In the Category tags applied area, click on the tags that you want to apply. To apply all tags, click Add all.

      You can also use the Search box to search for a specific tag by name.

      Note

      This option might not be available in your environment. If you have questions, speak to your administrator.

    7. Click Create.

      The new communication opens in Communication view in Content Author.

    Opening an existing communication or communication container

    1. In Communication launcher view, click the Communications panel, and then click one of the following buttons:

      • Library—Displays all of the existing communications and communication containers

      • Recent—Displays a list of communications and communication containers that you have recently opened in Content Author

      • Favorites—Displays a list of communications and communication containers that were marked as favorites.

        Tip

        To add a communication or container to the Favorites list, point to the communication or container in the Library or Recent list, and then click the Favorite icon in the Favorites column.

    2. In the Communications panel, in the Communication name list, perform one of the following tasks to open an existing communication or communication container in Communication view:

      Task

      Action

      To open a communication

      Click the communication name and click Open.

      To open a communication container

      1. Click the communication container name beside a stacked icon , and then click Open.

        In Communication view, the first document in the container opens automatically.

      2. Optionally, in the Applied document list, choose a different document to preview, if available.

      To open a specific document in a communication container

      1. Click the communication container name beside a stacked icon .

      2. In the document tree for the container, click the document that you want to open, and then click Open.

        In Communication view, the selected document in the container opens.

      3. Optionally, in the Applied document list, choose a different document to preview, if available.

      Note

      Reviewers who can only approve communications initially see only communications in the Submitted for approval workflow state when they open the Communications panel. If those users want to see communications in other workflow states, they must apply an advanced search filter that has the other states selected. For more information, see Filtering content resources using advanced search options.
