    OpenText Communications deployments can use the following types of product keys:

    Non-production product licenses

    A non-production, or development, license is used to design communications and create output for testing. When you use a non-production license to run the OpenText Communications engine, a Demonstration Powered by Communications watermark is placed at the top of each page in the generated output. The transactions are not counted, because customers have unlimited transactions when they run the OpenText Communications engine using a non-production product license.

    Contents of a non-production product license

    When an OpenText Communications Non-Production Kit is ordered by OpenText Sales for local on-premise installation (that is, not in the OpenText cloud), it delivers a non-production product license which contains the following features:

    • The feature to use the transactional OpenText Communications software and generate watermarked, non-production output

    • One non-production Designer Standard Named User Seat for an administrator user

    • One non-production Design and Production Design Tools Standard Concurrent Seat

    In order for users to be able to log in, they must have a seat in the non-production license. Therefore, the following seats must be purchased separately as non-production seats in order to be available in your non-production license:

    • Content Author Standard Named User Seats

    • Interactive Standard Named User Seats

    • Additional Designer Standard Named User Seats

    • Additional Design and Production Design Tools Standard Concurrent Seats

    • Design PDF Standard Concurrent Design Seats

    • Design Compare Standard Concurrent Design Seats

    • PDF Converter Standard Concurrent Design Seats

    Production product licenses

    A production license is required to create production output. Output that is produced using a production license does not contain an OpenText Communications watermark.

    Contents of a production product license

    When an OpenText Communications Production Kit is ordered by OpenText Sales, it delivers a production product license which contains the following features:

    • The feature to use the transactional OpenText Communications software and generate production output

    • One production Designer Standard Named User Seat for an administrator user

    The following seats must be purchased separately in order to be available in your production license:

    • A quantity of annual transactions for production use

    • A quantity of Designer Standard User Seats for use by additional administrators and Orchestration and Monitoring users.

    Content Author and Interactive Standard Named User Seats are optional purchases that can be included in the production license.

    The production license maintains the available annual transactions that have been purchased, and OTDS maintains the transaction counts that have been used from the available quantity in the production license. A transaction is defined as any event in which an input record, or portion of an input record, is processed and measured by the software. This means that each customer record that is processed during a single production engine run is considered a transaction if final output is generated for that customer input record. If recipient processing is enabled, then the transaction count is also incremented for each recipient copy that is produced.

    Keep in mind that the following items are not considered final output:

    • Previews (including Interactive previews)

    • Orchestration simulations run using a non–production engine

    • Design preview simulations

    • Application reports

    • Interactive output generated for Interactive output queues

    • Output queues for pre-sort processing where only a sort-index and sort-data output are generated during the pre-sort processing

    Note

    Queue reports, unlike application reports, do increment the transaction usage count, since they are only generated when output is created for a customer record.

    Additional transactions can be ordered and added to your production license, if needed.

    Important

    After a production license is used for an OpenText Communications engine run that increments the transaction count, you can no longer remove that license from OTDS. Make sure that production licenses are installed correctly before initiating any production engine runs.

    Message transactions

    OpenText Communications message transactions are designed and priced to accommodate the creation and delivery of smaller transactions, which are smaller in byte size than regular transactions. These micro-transactions in OpenText Communications, such as high-volume mobile messages, are referred to as message transactions.

    Message transactions, if purchased, are incremented when the Messaging output driver is used to deliver SMS messages, and no other output driver is included in the production engine run.

    Message transactions, if purchased, are added to the OpenText Communications production license as an annual quantity.

    Note

    If you are producing message output using the Messaging output driver in OpenText Communications, and have either exceeded the transaction count for message transactions or have not purchased message transactions, then the engine will produce the message output and count it as a regular transaction. Message transactions cannot be purchased unless transactions have already been purchased or both are purchased together.

    Temporary product licenses

    A temporary license is used strictly for evaluation purposes and needs to be installed on a separate tenant from any production and non-production licenses. Temporary licenses allow only for non-production output. If you’re using a temporary license, you should associate it with a tenant that is not already using a product license—such as the sample tenant that is included, but not automatically activated, in each OpenText Communications deployment.

    Note

    • A temporary license can be installed in a system tenant provided that there are no non-production licenses installed in OTDS in either the system tenant or any of the OTDS tenants.

    • If a temporary license and a non-production license are installed in the same OTDS tenant, or if a non-production shared license is installed in a system tenant and used by OTDS tenants, the temporary license is ignored.

      If a shared non-production license is being used in a system tenant, in order to use a temporary license to evaluate non-purchased features, a separate OTDS tenant must be used in a new OpenText Communications deployment.

    Optional product licenses

    There is also an optional product license that is used to . This license can be used in either production or non-production environments, or both if they are managed from the same OTDS tenant. The license provides access to the Google Cloud Platform Generative AI language model. Experience Aviator gives users the ability to generate intelligent automated text for designs and content. For more information, see section 2.7 "Experience Aviator product licenses" in OpenText Communications - Containerized Deployment Licensing Guide (CCDPCD250100-ULI)section 2.7 "Experience Aviator product licenses" in OpenText Communications - Containerized Deployment Licensing Guide (CCDPCD250100-ULI).

    After you have created a tenant for use in your OpenText Communications deployment, you must install OpenText Communications OTDS product keys in the tenant before you can use it. For specific information about using and configuring OTDS, see OpenText Directory Services Web Client Help (OTDS-H-AWC).

    Note

    For information about OpenText product compatibility, see the OpenText compatibly information provided in the release notes for the product. Release notes are available on My Support.
