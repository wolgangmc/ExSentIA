2.4.4   Setting up email notifications for workflow changes

  • Communication asset library view
  • Design asset library view
  • Orchestration asset library view
  • Monitor view
  • Rationalization asset library view
  • Dashboards view

    You can use Tenant management view to configure your OpenText Communications deployment to send out notifications to specified email distribution lists when specific design resources are submitted for approval. For example, when a user submits a Content Author communication for approval, an automated email notification is sent to approvers on a distribution list to prompt them to review that communication.

    Notification emails contain content and use settings that you specify when you add or edit a notification on the Notifications panel. Each notification email also automatically includes a redirect link for the submitted resource so that approvers can access and review the resource directly. From the redirect link, resources open in the context that makes the most sense for each type—for example, Content Author communications open in Communication view in Content Author; Designer pages open in Design view in Designer; and so on.

    To prevent an excessive number of emails being sent to a distribution list, when a resource is approved or rejected, an email is sent only to the user who originally submitted the resource for approval. In that case, OpenText Communications checks OTDS for the email address that is associated with the user who submitted the resource for approval. If no email address is available in OTDS, then OpenText Communications does not send a notification. You should ensure that approvers on a distribution list do not have the permissions required to change the workflow state for a submitted resource back to draft. If an approver returns a submitted resource to draft state, then no email notifications will be sent to any user about the change.

    Important

    If more than one notification exists for a resource type, then the user who submits a resource of that type for approval will receive more than one email when the resource is approved or rejected. For example, suppose that a notification is configured for only Content Author resource types, but another is configured for both Content Author and Designer resource types. In that scenario, when a user submits a Content Author resource for approval, then that user will receive two email notifications for the same resource.

    There are three basic steps involved in setting up a new notification:

    Step 1: Specify parameters and activate the notification

    1. In the Notifications panel in Tenant management view, click Add.

    2. In the Approver distribution list box, enter the email address for the distribution list that will receive notifications. The address for the distribution list is not validated or maintained in DAS; it must exist in your mail server. If the distribution list is incorrect or does not exist, then the notification will not be sent.

      Note

      • You can enter one email address only in the Approver distribution list box. If you need to add recipients for a notification, add their email addresses to the specified distribution list on the SMTP server.

      • The address that you enter in the Approver distribution list box also serves as the notification name, so it must be unique.

    3. Optional In the Domains list, select one or more domains. After that, the notification is sent only when an object that is associated with any of the specified domains changes state.

      The All domains default setting sends the notification when any object changes state, regardless of its associated domains.

    4. In the Resource types section, select the types of resources that will be included in the notifications:

      • All—Select this option to send notifications whenever any of the resources in the list are submitted for approval. Note that selecting no options in the list is the same as selecting All.

      • Designer—Select this option to send notifications whenever a Designer asset is submitted for approval. Designer assets are listed on the Designer panel in Design asset library view.

      • Content Author—Select this option to send notifications whenever a Content Author asset is submitted for approval. Content Author assets are listed on the Author panel in Design asset library view.

      • Communication Asset—Select this option to send notifications whenever a communication asset is submitted for approval. Communication assets are listed on the panels found in Communication asset library view.

      • Design Asset—Select this option to send notifications whenever a design asset is submitted for approval. Design assets are listed on the Common panel in Design asset library view.

      • Orchestration Asset—Select this option to send notifications whenever an orchestration asset is submitted for approval. Orchestration assets are listed on the panels in Orchestration asset library view.

    Step 2: Configure global SMTP settings for notifications

    The Properties panel for a notification includes a Global tab with settings that connect the OpenText Communications deployment to an SMTP email server. The SMTP server forwards notifications sent from the OpenText Communications deployment to the email address specified in the Approver distribution list for the notification.

    Each notification is registered to the single, unique email address in the Approver distribution list box on the General tab, but all notifications use the same SMTP settings on the Global tab.

    Important

    Changing SMTP settings on the Global tab changes the SMTP settings for all notifications in the tenant.

    To configure the global SMTP settings:

    1. In the Properties panel, click the Global tab.

    2. Configure the following settings to connect to the SMTP email server that DAS uses to send email notifications:

      Setting Description

      Host

      The host name of the SMTP server

      Port

      The SMTP listener port

      Authentication type

      The method to use for authenticating to the SMTP server

      This setting includes the following options:

      • Basic Authentication—Exposes the Username and Password settings for connecting to the SMTP server

      • No authentication—Allows the notification configuration to be saved without a username and password

      From

      The email address for the notification sender

    3. Click Save.

    Note

    The global SMTP settings persist in DAS even after all notifications are deleted. To access the global SMTP settings if no notifications exist, add a notification to the tenant and then click the Global tab on the notification Properties panel. Any changes you make to the global SMTP settings are saved when you save the new notification properties.

    Step 3: Configure the email layout

    1. In the Properties panel, click the Layout panel.

    2. Enter the subject line and content of the email notification in the Email subject box and Email body box, respectively. These are required fields and cannot be blank.

      When you place your cursor in the Email subject or the Email body box, you can also enter resource properties from the Resource property variables list. These properties are treated as resource-specific variables where they appear in the email content. When each notification is sent, the variable values are populated with the current information for the resource that triggered the notification. The following table lists the resource properties that are available as variables. No other variables can be used in email notifications.

      Note

      The variables that appear in the Resource property variables list have user-friendly names that don't necessarily match the information inside the brackets after you place a variable. For example, the Updated by variable in the list appears as {{LASTMODIFIEDBY}} when placed in the Email subject or Email body fields. The names in the brackets represent actual field names in the DAS database.

      Resource property Description

      Created By

      The user that created the resource

      Description

      The description of the resource, if any

      Domain

      The domain in which the resource was created or updated

      Effective date

      The start date on which the resource is available to be included in customer output. This option is available in the list only if effectivity is enabled for the current tenant.

      Expiration date

      The date on which the resource is no longer available to be included in customer output. This option is available in the list only if effectivity is enabled for the current tenant.

      ID

      The DAS resource identifier string for the resource

      Name

      The name of the resource

      Updated

      The date and time when the resource was last updated

      Updated by

      The user that last modified the resource

      Version

      The current version of the resource

      Workflow comment

      The comment, if any, in the workflow state change that triggered the notification

    3. Click Create.

      The notification is created. On the General tab, the Active indicator in the Notification status section turns green to indicate that the notification has been activated. To deactivate the notification in the future, simply click the Inactive indicator.

    With the notification in place, whenever a specified resource type is submitted for approval, an email will be sent to the distribution list. The email body will contain the text that you specified, including any populated variables. Additionally, the email will contain a link to the object that was submitted so that it can be accessed directly from the notification email.
