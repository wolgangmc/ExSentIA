    If a tenant has the default workflow enabled for Content Author, you can configure each domain in the tenant to use any one of the following workflow types:

    Note

    In any workflow type, users can submit drafts for approval if their OTDS group’s role has Creator permission. If their group’s role has Approver permission but not Creator permission, they can approve drafts but they cannot submit them. For more information about configuring role-based permissions OTDS groups, see Section 2.3.1 “Configuring roles and permissions”.

    • Standard workflow: (Default) A user submits a draft resource for approval. Any approver can approve or reject the draft. Each draft requires only one approval. Optionally, you can require a separate approver so that the person who submits the draft cannot approve it.

    • Simple workflow: Drafts are not submitted for approval. Any approver who also has Creator permission can approve a draft, including their own. Each draft requires only one approval.

    • Advanced workflow: A user submits a draft resource for approval. A member of each approver group in an approval role set reviews the draft. The approver groups have role-based permissions that allow their members to approve the draft's resource type. Approvals progress in series, parallel, or both. The process depends on the number of approval levels in the set and the number of groups at each level. Optionally, the groups at each level are notified when they can review the draft.

      After a member from every group on a level approves, the draft moves to the next level. After the highest level approves, the draft moves to the approved state. At any time before the advanced workflow completes, users who are not active approvers can check the approval status for the draft.
