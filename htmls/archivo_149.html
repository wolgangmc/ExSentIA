4.6.1   Variable and data types

    In the OpenText Communications web client, you can create user-defined variables that are then stored in DAS. You can also utilize variables that are included with a design pack.

    • User-defined variables are variables that you create yourself to specifically take data from a data file and then use the data in a design or in a rule.

    • System variables let you use system information, such as the information stored in the design environment or the information generated as part of production, to complete tasks such as monitoring and controlling an engine run, formatting designs, or organizing a customer mailing.

    When you create a variable, you must select the type of data that the variable represents. For system variables, the data type is preselected. This data type defines properties such as the available formatting of the variable when it is included in the output, how designers can use the variable in a design, and how the engine calculates the value of the variable.

    Table 4.1: Available variable data types

    Type Icon Data referenced

    Boolean

    Boolean variables represent data that must be one of only two possible values. For example, the value of a Boolean variable is always true or false and can have no other values.

    Currency

    Currency variables represent data that is a monetary amount. Formatting can be language-dependent.

    Date

    Date variables represent data that is a date, a time, or both. Formatting can be language-dependent.

    Note

    The date 02/29 is a valid value for the date variable only when the associated year is a leap year.

    Floating

    Floating variables represent data that is a numeric value that includes decimals. Floating variables are IEEE 754 single-precision values. Formatting can be language-dependent.

    Integer

    Integer variables represent data that is a whole number, positive or negative.

    Placeholder

    Placeholder variables represent data that is content from an external source and is accessed during an engine run.

    String

    String variables represent data that contains text. String variables can be language-dependent.


    The number of values a variable can represent varies. A variable can always represent a single value for each customer, or the variable can represent multiple values for each customer.

    If a variable represents only one value per customer (for example, the customer name), it is called a scalar variable. If a variable represents multiple values per customer, it is called an array variable. If the number of values in an array varies for each customer (for example, the number of transactions in a month), it is called a growing array.
