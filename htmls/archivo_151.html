4.6.3   Updating variable properties

    After you create a variable, you can use the Properties panel to view and modify the variable properties. You can modify only those variables that were created in Mapping view or directly in the Communication asset library view. To modify variables from your design pack, make your changes in Design Manager.

    You can edit variable properties in the following locations:

    • Mapping view— Either select the variable in the Variables panel or select the variable chip in the Data Source Editor area. You can use the following cards in the Variable properties panel to view and modify the variable properties: General , Value , and Used by .

    • Communication asset library view—In the Data panel, select the variable. You can use the General, Value, Domains, and Reference tabs in the Properties panel to view and modify the variable properties.

      Note

      Unlike other resources, variables do not require workflow steps and cannot be locked by a user.

    Modifying general properties

    The following table lists the properties available on the General tab:

    Table 4.2: General properties

    Property Description

    Name

    The name of the variable. Variable names must be alphanumeric and cannot exclusively use numerical characters.

    Description (Optional) A description of the variable

    User-friendly name

    		 (Optional) An alternate name for the variable that users will see in the design environment
    Type

    The type of data that is represented by the variable

    Note

    Use caution when changing the variable type after a variable has been created. If a variable is in use, changing the type may prevent the objects it is associated with from working properly.

    For example, if changing the variable type of a variable that is used in a rule causes the rule to be invalid, the engine will not finish its set up process and no output will be produced. The invalid rule must either be fixed or removed from the design.

    In the Properties panel, you can use the References tab to see where the selected variable is used in the current domain. However, changing the variable type will also affect anywhere it is used in the tenant, including other domains.

    Array

    Indicates that the variable can represent more than one value per customer.

    For placeholder variables, select the Array check box if the variable references more than one external file.

    Placeholder type

    The type of file that you want to import at run time.

    The Placeholder type list is active only when the variable type is Placeholder.

    You can select from the following options:

    • EPS—For EPS files

    • HTML—For HTML pass-through files

    • Image resource—For images from a printer resource (such as PostScript TIFF images or Metacode IMGs)

    • Interactive—For Interactive documents

    • JPEG—For JPEG files

    • PDF—For PDF files

    • PNG—For PNG files

    • TIFF (B&W G4 or uncomp)—For Group 4 Tiff Standard or 1-bit uncompressed black and white TIFF images

    • TIFF (Color)—For full-color TIFF files

    • Word (*.docx)—For DOCX files.

      Caution

      Not all DOCX design settings have an equivalent in OpenText Communications. Because of this limitation, before you select the DOCX file that you want to import, see the list of DOCX design considerations and supported features. See xxx.

    Note

    If you want to import images that include transparency, the images must be in PNG format.

    Pages to import

    Specifies which pages to import at run time.

    The Pages to import list is active only when the variable type is Placeholder and the placeholder type is PDF.

    You can select from the following options:

    • All—Import all the pages in the file

    • Range—Specify a range of pages to import

      In the Range box, enter the starting and ending pages.

      To indicate a range, separate the numbers with a hyphen (for example: 1-5). To indicate separate pages or identify multiple ranges, separate the numbers with a comma (for example: 3-17,22-28,31).

    • Variable—Use a variable to identify which pages to import

      In the Variable list, select a variable to control the imported pages. Only non-system scalar string variables can be used, and the content of the variable should use the same format as the Range option.

    Each customer file is unique

    Indicates that the file to be imported is different for each customer.

    The Each customer file is unique check box is active only when the variable type is Placeholder, with the exception of HTML placeholders.

    Do not convert HTML

    Specifies that the imported HTML is displayed in the browser according to the styles applied inline within the HTML.

    The Do not convert HTML check box is active only when the variable type is Placeholder and the placeholder type is HTML.

    Design sample

    (Optional) The text string that you want to appear when the variable is placed in a design

    Output format

    The formatting method to use for the variable in final output.

    The Output format list is not active when the variable type is Placeholder.

    Depending on the variable type, you can select from the following options:

    String

    • Custom—Applies a custom format string that you create

    • Keep blanks—Preserves spaces as they appear in the string

    • Trim blanks—Removes all of the spaces found before the first character and after the last character of the string

    • Lower, keep blanks—Converts the string to all lowercase characters and preserve all of the spaces as they appear in the string

    • Lower + exceptions, keep blanks—Converts the string to all lowercase characters, except those words found in an exceptions file, and preserve all of the spaces as they appear in the string

    • Upper, keep blanks—Converts the string to all uppercase characters and preserve all of the spaces as they appear in the string

    • Upper + exceptions, keep blanks—Converts the string to all uppercase characters, except those words found in an exceptions file, and preserve all of the spaces as they appear in the string

    • InitCap, keep, blanks—Converts the string so that each word begins with an uppercase character and preserve all of the spaces as they appear in the string

    • InitCap + exceptions, keep blanks—Converts the string so that each word begins with an uppercase character, except those words found in an exceptions file, and preserve all of the spaces as they appear in the string

    • Sentence, keep blanks—Converts the string so that only the first word of a sentence begins with an uppercase character and preserve all of the spaces as they appear in the string

    • Sentence + exceptions, keep blanks—Converts the string so that only the first word of a sentence begins with an uppercase character, except those words found in an exceptions file, and preserve all of the spaces as they appear in the string

    • Exceptions only, keep blanks—Converts the words found in the exception file to match their format in the file and preserve all of the spaces as they appear in the string

    • Phone number (999) 999-9999—Converts string values to a telephone number with parentheses around the area code and with a hyphen between the first three and last four digits of the number

    • Phone number (999) 999 9999—Converts string values to a telephone number with parentheses around the area code and with a space between the first three and last four digits of the number

    • Phone number 999 999-9999—Converts string values to a telephone number with no parentheses around the area code and with a hyphen between the first three and last four digits of the number

    • Phone number 999 999 9999—Converts string values to a telephone number with no parentheses around the area code and with a space between the first three and last four digits of the number

    Integer

    • Custom—Applies a custom format string that you create

    • General number—Uses simple numbers

    • Percentage—Adds a percent (%) symbol to the end of the number

    • Text upper (ONE, TWO)—Converts integer values to uppercase words

    • Text mixed (One, Two)—Converts integer values to mixed-case words

    • Text lower (one two)—Converts integer values to lowercase words

    • Alpha upper (A, B, C)—Converts integer values to uppercase alpha characters

    • Alpha lower (a b, c)—Converts integer values to lowercase alpha characters

    • Roman upper (I, II, III)—Converts integer values to uppercase Roman numerals

    • Roman lower (i, ii, iii)—Converts integer values to lowercase Roman numerals

    • Fixed decimal with currency—Inserts a fixed decimal point and the proper currency symbol for the locale

    • Fixed or integer with currency—Inserts a whole number if the number does not have a fraction and inserts the proper currency symbol for the locale. If the number has a fraction, the engine formats it as a floating number and inserts the proper currency symbol for the locale.

    • Absolute value—Converts the values to the absolute value. This option lets you produce a negative number without a negative sign. For example, both 1 and –1 appear as 1.

    Boolean

    • T/F—Returns T when the condition is true or F when the condition is false

    • Y/N—Returns Y when the condition is true or N when the condition is false

    • 1/0—Returns 1 when the condition is true or 0 when the condition is false

    • TRUE/FALSE—Returns True when the condition is true or False when the condition is false

    • YES/NO—Returns Yes when the condition is true or No when the condition is false

    Float

    • Custom—Applies a custom format string that you create

    • Use locale specification—Uses the format indicated in the 'SYS_LocaleCustomer' variable

    • General number—Uses simple numbers

    • Percentage—Adds a percent (%) symbol to the end of the number

    • Percentage x 100—Adds a percent (%) symbol at the end of the number and multiply by 100. This option lets you format fractions as percents. For example, .75 appears as 75%.

    • Fixed decimal—Adds zeros to pad the number of digits after the decimal point

    • Fixed decimal with currency—Inserts a fixed decimal point and the proper currency symbol for the locale

    • Fixed or integer with currency—Inserts a whole number if the number does not have a fraction and inserts the proper currency symbol for the locale. If the number has a fraction, the engine formats it as a floating number and inserts the proper currency symbol for the locale.

    • Significant decimal—Inserts the least number of decimal values possible after the whole number

    • Fixed or integer—Inserts a whole number if the number does not have a fraction. If the number has a fraction, the engine formats it as a floating number.

    • Absolute value—Converts the values to the absolute value. This option lets you produce a negative number without a negative sign. For example, both 1 and -1 appear as 1.

    Date

    Designer provides multiple options for formatting date values. For example: mm-dd-yy, dd/mm/yyyy, mm/yyyy.

    If you select Custom, you must create a customized formatting.

    If you select Use locale specification, output uses the format indicated in the 'SYS_LocaleCustomer' variable.

    Currency

    • Custom—Applies a custom format string that you create

    • Use locale specification—Uses the format indicated in the 'SYS_LocaleCustomer' variable

    • General number—Uses simple numbers

    • Fixed decimal—Adds zeros to pad the number of digits after the decimal point

    • Fixed decimal with currency—Inserts a fixed decimal point and the proper currency symbol for the locale

    • Fixed or integer with currency—Inserts a whole number if the number does not have a fraction and inserts the proper currency symbol for the locale. If the number has a fraction, the engine formats it as a floating number and inserts the proper currency symbol for the locale.

    • Significant decimal—Inserts the least number of decimal values possible after the whole number

    • Fixed or integer—Inserts a whole number if the number does not have a fraction. If the number has a fraction, the engine formats it as a floating number.


    Modifying value properties

    The following table lists the properties available on the Value tab:

    Table 4.3: Value properties

    Property Description

    Initial value

    The default or initial value of the variable. If the variable type is Boolean, the Initial value can be either false or true.

    Source

    The method used to access or calculate the variable. You can select from the following options:

    • User value—The variable value is specified in the variable properties

    • File only—The variable value is read from a data file

    • Counter—The variable value counts events

    • Crossref page number—The variable value is the page number of the target that references this variable. This option is active only when the variable type is Integer.

    Reset timing

    Specifies when the variable value is reinitialized (reset to zero). You can select from the following options:

    • Automatically—Resets before each customer

    • Before each customer—Resets only before each customer is read

    • Never reset—Resets only at the beginning of each engine run

    • Named data segment—Resets before the engine reads the section that you choose in the Data segment list that appears

    • All data segments—Resets before all data segments

    Enable selection list

    Associates a static list of options with the variable. These list items are used as attributes when users in Designer and Content Author create variations. They are also available when the variable is used to build custom rules or is used in Simulation view.

    The Enable selection list option is active only when the variable type is String, Currency, Float, or Integer.

    To associate a static list with the selected variable:

    1. Select the Enable selection list check box.

    2. Under the List items box, click Edit.

    3. In the Add new static selection list item box, enter the first entry for the static list.

      Repeat this step to add as many items as needed to the list.

    4. Click Save twice.

    Watch level

    Specifies when the engine should write information to a debug file. You can select from the following options:

    • None—The engine disables watch level activities

    • Changed—The engine writes information each time the variable changes

    • Set—The engine writes information when the variable is set

    Validation method

    The method for automatically validating the value of the variable for both the initial value and values that are mapped in the data source. This option is not active when the variable type is Placeholder or Boolean. You can select from the following options:

    • None—The variable value is not validated when the value changes

    • Equal to—Specify a precise valid value

    • Not equal to—Specify a precise value that is not valid

    • Greater than—Specify a value that the variable value must be greater than

    • Greater than or equal to—Specify a value that the variable value must be greater than or equal to

    • Less than—Specify a value that the variable value must be less than

    • Less than or equal to—Specify a value that the variable value must be less than or equal to

    • Value range—Specify the range of valid values

    • Length range—Specify the minimum and maximum lengths of the variable. This option is active only if the variable type is String.

    Validation value

    		 The validation value to check against. This box is active only if the Validation method is set, and is not a range.

    From

    		 The lower value in a validation range

    To

    		 The upper value in a validation range

    Action if invalid

    The type of error processing to perform if the value is invalid. This option is active only if the Validation method is set.

    • Message, continue—You receive a message and the engine continues processing

    • Message, set to default—You receive a message and the engine resets the variable to its default value and continues processing

    • Message, stop—You receive a message and the engine stops processing

    • Message, skip document—The engine skips the document if the variable is not valid or is not found in the data


    Inserting images using a placeholder variable

    You can use a placeholder variable to insert DAS images into a Designer or Content Author communication at run time. To insert images, the placeholder variable value needs to use the syntax ot:das/<Image_ID>, where <Image_ID> is the DAS resource identifier string that is found in the advanced resource details.

    Example: ot:das/168cacd3-19d5-4eca-b3ed-ad36bed0cdc5

    Note

    Placeholder variables only import the type of file that is specified in the variable type, such as JPG or PNG. If you use ot:das/<Image_ID> to pull images from DAS, all the DAS images that you want to use must use the same format as the placeholder variable type; images that are different from the placeholder variable type do not appear in the output.

    Viewing references for a selected variable

    The Used by card in Mapping view and the References tab in the Communication asset library view show the data sources and Designer communications that reference the selected variable.

    The information displayed here is not editable. This information can provide you with some impact control, in that it is useful in determining how a change to any of the variable properties might affect other referenced objects, such as data sources and communication objects.

    Specifying how array variable elements are displayed

    If your design includes an array variable, you can specify how the elements in the array are displayed. This can be useful for situations like controlling how the engine selects the correct external file when using a placeholder variable.

    To specify how array elements are displayed:

    1. Open the design in Design view.

    2. Within the design area, select the array variable.

    3. In the Variables panel, open the General properties card.

    4. In the Array element to use list, select one of the following options:

      Option

      Description

      Automatically display an array element for each occurrence of the variable

      Select Automatic.

      Display a specific element in the array

      1. Select Specified.

      2. In the adjacent box, select the specific element number you want to use for this variable.

      Display all instances of the array in the data file for each occurrence of the variable, including blank spaces

      Select All, blanks.

      Display all instances of the array in the data file for each occurrence of the variable, and separate each element in content using a line break (soft return).

      Select All, line breaks.

      Display all instances of the array in the data file for each occurrence of the variable, and separate each element in the content using a paragraph break (hard return).

      Select All, paragraphs.

      Display all instances of the array in the data file for each occurrence of the variable, and list them in order. Select All, print as list.
