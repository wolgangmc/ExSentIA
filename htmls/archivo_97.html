2.6.2   Naming engines

    By default, when you upload an engine to your tenant, OpenText Communications automatically assigns a name to the engine that matches its serialization. As an alternative, you can add user-friendly names—such as “Production” or “Development”—for each of the engines in your tenant. Then, when users need to select an engine for an engine run definition or a flow context, they can select from more intuitive options, instead of from a series of filenames.

    Naming engines can also be useful if you need to replace an engine that is currently available in your tenant. For example, if you are in the process of phasing out an engine in your tenant, you can change the name of that engine to “Deprecated v20.2” so that users know not to select it in the future for their applications. After you have migrated all applications to other engines, you can delete that engine.

    To name an engine:

    1. Sign in to the OpenText Communications web client as an OTDS administrator user at the following URL:

      https://<host>:<optionalPort>/design?tenant=system

    2. In the Engines panel, select an engine from the list, and then click Open.

    3. Specify a name for the engine, as well as an optional description, and then click Save.

      Names do not support the following special characters: & / : * ! ? > < | + # %
