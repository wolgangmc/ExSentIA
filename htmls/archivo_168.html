    The HTML output driver produces HTML files that use Cascading Style Sheets (CSS) and JPEG images. These files are used for web-based communications.

    OpenText Communications provides HTML output that supports text and images as output for presentation on the Internet, intranets, or other electronic networks. With HTML output, you can produce web pages and other browser-based communications. You can deploy OpenText Communications-generated HTML pages on the web, in eLearning sessions, for online help programs, and in other communications that use standard web browsers.

    The HTML content that you produce from Designer is HTML5. This lets you produce output to which you can, using external systems, add any of the new features that are specified by the HTML5 recommendation. You can also include accessibility tags so that your HTML output can be read by a screen reader or text-to-speech converter.

    HTML5 uses Cascading Style Sheets, level 3 (CSS3) for styles and object positioning. With HTML5 output, all styles are defined in an external CSS file or within the style element in the HTML itself, depending on how the HTML output object is configured. Individual elements use class selectors to apply styles. The separation of styles in HTML5 output lets you more easily change styles to adjust the look and feel of your existing output, without producing new output.

    Note

    Starting in 23.4.0 and later, in HTML5 output generated from Designer, container cells are produced using a DIV structure. This DIV structure allows for easier post processing to create more dynamic output. For backward compatibility, you can use the USE_TABLES_FOR_GRID_CELL_CONTAINERS engine switch to produce HTML5 output within a table structure.

    For more information about the USE_TABLES_FOR_GRID_CELL_CONTAINERS engine switch, see section 7.7.33 "USE_TABLES_FOR_GRID_CELL_CONTAINERS" in OpenText Exstream - Switch Reference Help (CCDPSR250100-H-UGD)section 7.7.33 "USE_TABLES_FOR_GRID_CELL_CONTAINERS" in OpenText Exstream - Switch Reference Help (CCDPSR250100-H-UGD).

    If you create an HTML output object for your communication, when the engine runs, OpenText Communications produces the following set of files:

    • HTML output file (.html) – The source HTML file

    • JPEG (.jpg) – These are the images that are imported into the design in their original format (such as JPEG or PNG). The engine converts the original files into the format specified in the output object. These files contain the graphic design objects (such as charts, lines, and shapes) that you added to your design using the drawing tools in Designer for Design and Production. The engine converts these design objects into separate image files in the format specified in the output object properties, and then references those image files in the HTML output, along with any images you imported at run time in their original format (such as JPEG or PNG).

    • External Cascading Style Sheets files (.css) (Optional) – These files contain the styles that are used in the output. You receive multiple CSS files if you are delivering container designs in the output and you are using additional cascading style sheets to control the final appearance of your container design.

    • External JavaScript files (Optional) – These files contain any JavaScript code used within the design.

    After you create the output object, you can use the Properties panel to view and modify the following output properties. After making changes, click Save or the changes will be lost.

    General output settings

    The following sections provide information about the general output options.

    External style sheets

    When you create HTML output, you can either use an external Cascading Style Sheet (CSS) file for styles used in the HTML, or you can include style information at the top of each HTML file, within the <head> element.

    Using an external style sheet makes it very easy to change the appearance of your HTML output outside of OpenText Communications, but selecting this option also means that you must manage multiple files as part of your workflow, since all of the data defining your communication is not included in one self-contained package file. Consider the pros and cons of both CSS placement options, and then choose the one that best suits your needs.

    If you are delivering container designs in the output, and you are using custom CSS style sheets in addition to the CSS style sheets that OpenText Communications automatically creates to manage the styles that you applied in Designer, the CSS placement option you select applies to the custom CSS style sheets as well as the automatically generated ones.

    To include style information in an external file, select the Create external cascading style sheet option.

    External JavaScript files

    If you want to create separate files for JavaScript output objects, instead of inserting JavaScript into the HTML output, select the Create external JavaScript option.

    Accessibility standard

    If you want to create output that can be read by a screen reader or text-to-speech converter, you can enable accessibility for the output objects. In the Accessibility standard list, select the accessibility standard that you want to use.

    Additional settings

    This output includes additional common settings. For more information about any of these settings, see the following locations:

    • Domains – For information about defining the domains to which a resource is assigned, see Section 3.13 “Associating resources with domains”.

    • Workflow – For information about managing an approval workflow, see Section 3.6 “Managing approval workflows”.

    • References – For information about viewing the linked resources connected to a selected resource, see Section 3.3 “Viewing resources in asset library views”.
