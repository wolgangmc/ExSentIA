4.14   Creating locale objects

    Locale objects are resources that allow you to differentiate variable data not only by language, but also by geography.

    If your organization has a multi-regional or an international customer base, your customers might not only speak various languages, but they might also use different currencies, date formats, and numeric notation systems. For example, in the United States, dates are expressed in “month/day/year” format, whereas in other countries/regions, dates are expressed in “day/month/year” format. In OpenText Communications, you can create locale objects to specify how monetary amounts, numbers, and dates must be presented in communications intended for specific geographical areas.

    Note

    If the format setting for an individual variable differs from the locale object setting, the variable format is used in output.

    • Date format—Specifies how dates are represented in a locale.

    • Currency format—Specifies the characters and appearance a locale uses to represent currency.

    • Default floating format—Specifies the characters and appearance a locale uses to represent floating point numbers. Floating point numbers are numbers that are not whole numbers (for example, 3.173).

    Keep in mind that locale objects reference language objects; therefore, you must create language objects for each language that your organization supports before you can create locale objects.

    Multiple locale objects can reference the same language object. However, you must create a separate locale object for a country/region when you need to specify at least one unique property for that country/region. For example, if you have customers in Mexico and Spain, you must create two separate locale objects for these countries—even though the customers in both countries speak Spanish—because these countries use different currencies.

    When using locales with a communication that is based on a communication set, OpenText recommends the following:

    • If you plan to include more than one language variation in a design, you can specify a default locale for the communication. This is done from Designer, in Communication Composer, on the General properties panel of the communication.

    • For any locales you plan to use in a communication, include the appropriate locale resource objects in the communication set. This ensures that they are available to the engine during production runs.

    Creating a new locale object

    1. In the Region panel, click Add, then click Locale in the selection list.

    2. In the Properties panel, enter a name and, optionally, a description for the locale object resource.

      Names do not support the following special characters: & / : * ! ? > < | + # %

    3. In the Language list, select the language object for this locale from the list of available languages.

    4. Click Create.

      The locale object resource is created and the Properties panel displays the locale properties.

    5. Use the Properties panel to define the following attributes:

      Property Description
      Language The primarily spoken language of the region.
      Date format The preferred date notation for the region.
      Currency format Symbol

      The shorthand notation for the region’s basic unit of currency. For example, “$” or “£”.

      Thousands

      The character symbol used as the thousands separator in monetary amounts and numbers. For example, “,” (comma) or “.” (period). That is, the number “one thousand” is expressed numerically as “1,000” in some countries/regions and as “1.000” in other countries/regions.

      Decimal

      The character symbol used as the decimal separator in decimal numbers and monetary amounts. For example, “,” (comma) or “.” (period). That is, the amount “one dollar” (or other unit of currency) is expressed numerically as “1.00” in some countries/regions and as “1,00” in other countries/regions.

      Negative

      The format used to express negative numbers and monetary amounts. For example, “($1)” or “-£1”.

      Positive

      The format used to express positive currency amounts. For example, “$1” or “£1”.

      Digits

      The number of digits shown after the decimal point in decimal numbers. This number can range from 0 to 4.

      Set zero

      If you want to specify text to display when the value is zero (0), select the Set zero check box and enter the character or string that you want to use in place of 0 in the text box.

      Default floating format Digits

      The number of digits to show after the decimal point when calculating numbers. This number can range from 0 to 4.

      Thousands

      The character to use for the thousands separator in floating point numbers.

      Decimal

      The character to use as the decimal separator in floating point numbers.

      Negative

      The formatting method to indicate negative floating point numbers.

      Set zero

      If you want to specify text to display when the value is zero (0), select the Set zero check box and enter the character or string that you want to use in place of 0 in the text box.

      Show leading zero

      Select to show a leading zero in amounts less than one (1). For example, “0.03”.

      Array variable list format Delimiter

      The separator character used for displaying lists of values from array variables. For example, “, ” or “; ”.

      Note

      Although not required, if you want the separator to include a space, as seen above, it needs to be included in this field.

      Conjunction

      The final conjunction used for displaying lists of values from array variables. For example, “, and ” for locales where the primary language is English or “, y ” for locales where the primary language is Spanish.

      Note

      Although not required, if you want the conjunction to include spaces, as seen above, they need to be included in this field.

    Using locale objects for design variations and data mapping

    When you create a locale object, its name is automatically added as a static selection list item in the SYS_LocaleCustomer system variable, which controls which locale object to use at engine run time. If the SYS_LocaleCustomer variable is then added as a variation to a communication set, any Designer designs or Content Author templates based on that communication set will include those locale objects as variation options.

    Note

    In order to use SYS_LocaleCustomer in Content Author, the Include for Content Author option needs to be enabled for the variable in the communication set.

    Keep the following recommendations in mind when creating variations using the SYS_LocaleCustomer variable:

    • If you create a design variation using a locale as an attribute, the language property of that locale object dictates the spelling dictionary used during editing. If multiple locales are used in a variation, then the default locale spelling dictionary is used.

    • If you are using both the SYS_LocaleCustomer and SYS_LanguageCustomer variables, and you want to create a variation using only one of them, OpenText recommends that you use the SYS_LocaleCustomer variable; otherwise you might see incorrect formatting in your output.

    • If you create a variation using both the SYS_LocaleCustomer and SYS_LanguageCustomer variables, the language setting for both should match. For example, both should be set to English or both to French.

    • In order to ensure you get the correct formatting and language in your output, either map the SYS_LocaleCustomer variable in your data source or use a targeting rule to set the correct values for the variable.

    • If you include a language variation in a design, OpenText strongly recommends that you specify a default locale for the communication. A default locale can be designated from Designer, in Communication Composer, on the General properties panel of the communication.

    For more information on using variations in a communication, see OpenText Communications - Designer User Help (CCCD250100-H-UGD)OpenText Communications - Designer User Help (CCCD250100-H-UGD).

    If you map Date, Currency, or Float type variables whose output format is set to Use locale specification, and then use those variables in a communication design that is associated with one of the SYS_LocaleCustomer locale variations, then the engine will use the settings in the locale object properties to properly display the data in the output.

    Duplicating a locale object

    If you want to create a copy of a locale object, you can duplicate it. This might be useful if you want a similar locale object with only minor modifications.

    To duplicate a locale object, in the Region panel, select the locale object that you want to copy and click Duplicate. You can then update the settings for the new locale object.
