    You can use the Configurations panel in Tenant management view to specify tenant-wide defaults for design assets in OpenText Communications.

    On the General tab:

    OpenText Digital Asset Management Services

    Select the Use Media Management Services check box and then specify the OpenText Digital Asset Management URL to connect to the service, and the username and password to connect to OpenText Digital Asset Management during fulfillment. You can optionally specify the name of an OpenText Digital Asset Management AMD profile from those that currently exist in the OpenText Digital Asset Management service. The automatic changes to the image prescribed by the profile are applied when the image is inserted in Design view or Authoring view. You can use the Run validation button to verify that OpenText Communications is properly connecting to the service.

    Before they can access images stored in OpenText Digital Asset Management, Designer and Content Author users must log in to OpenText Digital Asset Management in OpenText Communications. The log in prompt appears when they insert an image in Design view or Authoring view.

    For more information about deploying OpenText Digital Asset Management, see OpenText Digital Asset Management Cloud Deployment Guide (CDPEXPCDP-UAG).

    For information about managing permissions in OTDS, see OpenText Directory Services Installation and Administration Guide (OTDS-IWC).

    Note

    For information about OpenText product compatibility, see the OpenText compatibly information provided in the release notes for the product. Release notes are available on My Support.

    For information about specific configurations required to integrate OpenText Digital Asset Management with your OpenText Communications deployment, see OpenText Communications - Cloud Deployment Guide (CCMCE250100-IGD)OpenText Communications - Cloud Deployment Guide (CCMCE250100-IGD).

    Default color space

    In the Default color space list, choose the color model that OpenText Communications uses by default to represent colors in digital and print outputs:

    • RGB (Red, Green, and Blue): A color model for digital displays, such as web and email outputs. You specify an RGB color using its separate red, green, and blue values.

    • HEX: A color model for digital displays that represents each RGB color with a six-digit hexadecimal code. The code is of the form #RRGGBB, where RR is the red value, GG is the green value, and BB is the blue value. The hex digits include 0 through 9 and A through F. For example, the hex color code for RGB white is #FFFFFF.

    • CMYK (Cyan, Magenta, Yellow, and Key, also known as Black): A color model for printing ink on paper that represents colors using varying amounts of cyan, magenta, yellow, and black ink.

    Color settings in the properties of communications or design resources can override the Default color space setting.

    Prevent duplicate names within domains

    Some resources, such as variables, require duplicate name prevention by default. If you want to avoid identical names within the same domain for any new or updated resource that does not already require duplicate name prevention, select the Prevent duplicate names within domains check box. When selected, this setting is applied to all domains within the tenant.

    Enforce category selection

    If one or more categories are configured in the tenant, then selecting or clearing the Enforce category selection on design and authoring objects check box has the following effects:

    State

    Effect

    Selected

    Enforces category selection. Users must apply one or more categories when creating a new resource in the tenant. Users can create new resources only if category tags are available in the tenant.

    Cleared (default)

    Does not enforce category selection. Users can optionally apply categories when creating a new resource in the tenant.

    If you want to enforce category selection, configure at least one category tag on the Categories panel in Tenant management view. If no category tags are configured, then the Enforce category selection on design and authoring objects check box is unavailable. For more information about creating category tags, see Section 2.4.6 “Creating category tags”.

    Control default editing access to resources in associated domains

    In an asset library view, when a user associates a resource with a different domain in the tenant, a dialog box opens to show the the Grant edit permissions with association check box. Selecting or clearing this check box makes the resource editable or read-only (respectively) in the associated domain.

    You can set the default state of the Grant edit permissions with association check box using the Automatically set domain associated content object access to read only check box. Selecting or clearing this option in Tenant management view has the following effects:

    State

    Effect

    Selected

    Clears the Grant edit permissions with association check box by default so that resources in associated domains are read-only by default.

    Cleared (default)

    Selects the Grant edit permissions with association check box by default so that resources in associated domains are editable by default.

    For more information about associating a resource with a domain, see Section 3.13 “Associating resources with domains”.

    On the Intelligence tab:

    OpenText Communications Experience Aviator

    OpenText Communications Experience Aviator provides Designer and Content Author users the ability to generate intelligent automated text for use in their designs and content. This feature can be used to create new text content or to modify existing text content. Experience Aviator has the ability to modify text in several ways; for example, you can use it to alter tone, improve readability, or change the voice of a piece of text.

    Note

    This functionality is available only if an Experience Aviator product key has been licensed and allocated in OTDS.

    To provide the capability to create generative AI content, select the Enable Experience Aviator check box.

    Experience AI Services for Assisted Authoring and translation

    If you deployed OpenText Communications Experience AI Services (EAIS) in your Kubernetes cluster, then you can enable EAIS tenant-wide. If you enable EAIS, then designers and authors in the tenant can do the following while editing text content in Designer and Content Author:

    • Analyze text in languages that support Assisted Authoring

    • Translate text

    For more information about analyzing and translating text in Designer and Content Author, see OpenText Communications - Designer User Help (CCCD250100-H-UGD)OpenText Communications - Designer User Help (CCCD250100-H-UGD) and OpenText Communications - Content Author User Help (CCCA250100-H-UGD)OpenText Communications - Content Author User Help (CCCA250100-H-UGD).

    To enable Experience AI Services for an OpenText Communications tenant:

    1. In the OpenText Communications web client, sign into the following URL as a tenant administrator user:

      https://<host>:<optionalPort>/design?tenant=<tenantName>

    2. In Tenant management view, click Configurations > Design asset library.

    3. On the Properties panel, click the Intelligence tab, and then select the Use Experience AI Services check box.

    4. In the Connection URL box, enter the URL for EAIS. Your OpenText Communications administrator can provide the URL.

    5. Click Run validation.

    6. Check the validation status for the connection:

      • If the green Validated status icon appears, the tenant can connect to EAIS.

      • If the red Not validated status icon appears, the tenant cannot connect to EAIS. Check the URL and your internet connection and then select Run validation again. If Not validated still appears, ask your OpenText Communications administrator for assistance.

        Note

        If the tenant cannot connect to EAIS, you must clear the Use Experience AI Services check box before you can save the configuration.

    For more information about deploying and configuring Experience AI Services, see OpenText Experience AI Services Configuration Guide (CDPEAIS-PGD).

    Note

    For information about OpenText product compatibility, see the OpenText compatibly information provided in the release notes for the product. Release notes are available on My Support.

    Configuring tenant-wide settings for Assisted Authoring

    Assisted Authoring provides language-specific text analysis tools to users in Designer and Content Author. Assisted Authoring tools can measure readability, sentiment and tone, and grammar. If Assisted Authoring is configured for a language in a communication, a designer or author can apply the Assisted Authoring tools while editing text in that language.

    The following languages support Assisted Authoring: Dutch, English (American, Australian, and British), French, German, Portuguese, Portuguese (Brazilian), and Spanish. Not all text analysis types are available for every supported language.

    Important

    OpenText Communications does not include a spelling dictionary for all languages, such as German. A dictionary must be installed in the tenant to use Assisted Authoring with those languages. If you add a dictionary, specify its language code (for example, de_DE for German) to ensure that Assisted Authoring works with that language.

    Note

    OpenText Communications includes spelling dictionaries for the following languages, but these languages do not support Assisted Authoring: Hungarian, Korean, Polish, Russian, Swedish, and Thai.

    In Tenant management view, you can configure a default list of languages for Assisted Authoring that applies to every communication in the tenant. You can also configure the Assisted Authoring settings for each language in the default list.

    The tenant-wide default list of languages for Assisted Authoring initially includes only English (American). If you add another language to the default list, its initial Assisted Authoring settings are preconfigured and ready to use. You can optionally change the initial settings for any language in the default list.

    Tip

    In addition to editing the tenant-wide default Assisted Authoring settings, you can also customize the Assisted Authoring settings for any communication that you select in Design asset library view.

    To edit the tenant-wide Content languages list for Assisted Authoring:

    1. In Tenant management view, click Configurations > Design asset library.

    2. In the Properties panel, click the Intelligence tab.

    3. Select the Experience AI Services (EAIS) check box. If the red Not validated status icon appears, validate the EAIS URL.

    4. On the Intelligence tab, click the Edit list button following the Content languages list.

    5. In the Manage Languages dialog box, you can add or remove content languages as follows:

      • To add a language: In the Available languages list, click the New list item icon for the language.

      • To remove a language: In the Added languages list, click the Remove list item icon for the language.

    6. Click Apply.

    Important

    If you add a language to the Content languages list, ensure that a dictionary with that language code is installed (for example, de_DE for German). Otherwise, Assisted Authoring will not work for that language.

    To edit the tenant-wide Assisted Authoring settings for a content language:

    1. Perform step 1 through step 3 in the previous procedure.

    2. On the Intelligence tab, choose the language that you want to edit in the Content languages list, and then click Modify details.

    3. In the Assisted Authoring dialog box, select the Enable check box for each Analysis type that you want to make available for the selected language.

      The licensed level of service for each analysis type appears after its check box. For information about licensing analysis types, see OpenText Experience AI Services Configuration Guide (CDPEAIS-PGD).

      Note

      For information about OpenText product compatibility, see the OpenText compatibly information provided in the release notes for the product. Release notes are available on My Support.

    4. Configure the settings for each enabled Analysis type, and then click Save. For more information about configuring analysis types, see Configuring analysis types for Assisted Authoring.

      Note

      If you have questions about the available text analysis types for a supported language, contact your Experience AI Services administrator.

    5. Click Save.

    Configure tenant-wide text translation settings

    If EAIS is enabled for the tenant, you can select the translation service provider used by EAIS. You can also edit the list of active languages for translation. The list of target languages for translation in Content Author includes only the active languages.

    To select the translation service and active languages for translation:

    1. In Tenant management view, click Configurations > Design asset library.

    2. In the Properties panel, click the Intelligence tab.

    3. Select the Experience AI Services (EAIS) check box. If the red Not validated status icon appears, validate the EAIS URL.

    4. On the Intelligence tab, choose a translation service provider from the Translation service provider list.

    5. To add or remove active languages, click Edit list following the Active languages list.

    6. In the Edit active languages dialog box, you can add and remove active languages as follows:

      • To add a language: In the Inactive list, click the Add to active list icon for the language.

      • To remove a language: In the Active list, click the Remove from active list icon for a language.

    7. Optional Click Reset to undo your list changes and continue editing.

    8. Click Apply.
