5.2   Creating colors and color families

  • Dashboards view

    You can use the Common panel in the Design asset library view to create color and color family resources in the asset library. Custom colors and color families are often used as a way to create an approved color palette to ensure consistency in all designs across an enterprise.

    You can create the following color resources in the asset library:

    Method Description
    Colors

    When you create a new color resource, you use standard colors to define the color resource. Pick a color from a color palette or enter a color code. See Creating a new color.

    Alternate colors

    For any color, you can also define color variations, called alternate colors, which can change based on customer data. When a color with alternate colors is applied in the design, then that color can change automatically in the output based on the presence of customer data values. Alternate colors are especially useful if your organization wants to manage color variations for marketing, campaigns, or branding. See Defining alternate colors.

    Note

    Alternate colors are not the same as conditional colors. Conditional colors are color variations specified for individual content such as inline text, text box backgrounds, and table cell backgrounds. After a default color is applied to content, the conditional color can change based on customer data. Conditional colors are especially useful if you want to emphasize certain values. For example, if a statement balance is less than or equal to zero dollars, you can use a conditional color to make the balance appear in red, and a different conditional color to make balances above zero dollars appear in blue. See section 13.3 "Applying color to designs" in OpenText Communications - Designer User Help (CCCD250100-H-UGD)section 13.3 "Applying color to designs" in OpenText Communications - Designer User Help (CCCD250100-H-UGD).

    Color families Color families are groups of colors. Colors within approved color families are available to select on the Custom colors tab of any color selection dialog box in the domain.

    Note

    The only exception is the color selection dialog box for the default bullet and numbering styles in Tenant management view. Custom colors are not available for those items.

    The OpenText Communications web client supports custom colors in the RGB, CMYK, and hexadecimal code color models. At run time, the engine converts colors as needed so that they are compatible with the selected output type. If the output device you are using does not support the color selected for an object, the color is remapped to the closest available color.

    Creating a new color

    1. In the Design asset library, on the Common tab, click Add, and then select Color.

    2. In the Properties panel, do the following:

      • Enter a name and, optionally, a description for the color.

        Names do not support the following special characters: & / : * ! ? > < | + # %

      • Select a color.

        1. Click the Color selection box.

        2. Select a color using one of the following methods:

          • Basic selector—Pick a color from the provided color palette.

          • Advanced selector—Use the palette to select a color or enter the RGB values, hex code, or CMYK values for a color.

        3. Click Apply.

    3. Click Create.

    Defining alternate colors

    1. In the Design asset library, on the Common tab, create or select a color object.

    2. In the Properties panel, click the General tab.

    3. Select Allow alternate colors.

    4. To control alternate color selection, complete the following steps:

      1. Click the Alternate color variable box.

      2. In the Variable dialog box, select a scalar string variable that includes a static selection list .

      3. Click OK.

    5. To specify the alternate colors you want to apply, complete the following steps in the Alternate colors area: :

      1. In the Name box, enter a name for the alternate color.

      2. From the Value list, select the value that will trigger the application of the color.

      3. In the Color box, define the color that you want to apply.

      4. To add another alternate color, click Add and repeat step 5.a through step 5.c.

        Tip

        To remove an alternate color, select the alternate color and click Remove.

    6. Click Save.

    Creating a new color family

    1. In the Design asset library, on the Common tab, click Add, and then select Color family.

    2. In the Properties panel, enter a name and, optionally, a description for the color family.

      Names do not support the following special characters: & / : * ! ? > < | + # %

    3. Click the Edit button.

    4. In the Color family dialog box, select the colors that you want to add to the color family, then click Apply.

      Note

      Only approved colors can be added to a color family. Also, only the latest approved color within a color family is available for use in designs or other resources.

    5. Click Create.

    Note

    If a color is hidden after being added to a color family, the color name appears in gray italics in the color family properties. As with other hidden resources, if a color is in use and then subsequently hidden, it will still be visible where used, but will not be available for use in a new instance.
