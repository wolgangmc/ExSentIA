8.3.1   Considerations for importing HTML content

  • Communication asset library view
  • Design asset library view
  • Orchestration asset library view
  • Monitor view
  • Rationalization asset library view
  • Dashboards view

    OpenText Communications supports importing HTML content at run time by using an HTML placeholder variable. This feature is intended to be used with snippets of HTML content rather than complex web communications. For example, suppose that your marketing organization produces promotional content in HTML for use on the company website. You can import this HTML content at run time so that the promotions appear inline along with content that was created in Designer.

    OpenText Communications supports HTML placeholder variables as text only. You cannot use an HTML placeholder variable with a placeholder document or an empty image object.

    Figure 8.1: Example of how placeholder variables work with HTML import

    When importing HTML content, you can set up your placeholder variable so that it provides instructions to the engine using one of the following methods:

    To use this method Do this
    Define a path to the HTML content

    1. In Communication asset library view, create an HTML placeholder variable and specify a static user value for the file path of the HTML content to import.

    2. In Designer, insert the placeholder variable inline in a design object that can contain text.

    Include the HTML content in a data file

    1. In Communication asset library view, create an HTML placeholder variable with File only selected from the Source list in the variable properties.

    2. Create a data file and map the placeholder variable to the data area that contains the HTML content.

    3. In the Data area properties panel, select an option in the Encoding list:

      • None—Select if the content was not encoded.

      • Base-64—Select only if the content was encoded prior to the engine run.

    4. In Designer, insert the placeholder variable inline in a design object that can contain text.

    Include the HTML content in a communication event

    1. In Communication asset library view, create an HTML placeholder variable.

    2. Add the HTML placeholder variable to the communication set that the communication is based on.

    3. From Designer, in Communication Composer, specify the value of the variable by creating an event on the Logic tab of the communication properties.

    4. In Designer, insert the placeholder variable inline in a design object that can contain text.

    When the engine encounters imported HTML, it builds a set of default styles before parsing the HTML. These styles are based on the placeholder variable formatting in Designer. The engine parses each import only one time.

    For best results, create an HTML placeholder variable for each instance you want to import. If you insert the same HTML placeholder variable in a design multiple times, this set of default styles is created for the first instance of the HTML, and then used for all subsequent instances of the placeholder variable.

    For example, suppose the first instance of the HTML placeholder variable is in a paragraph formatted as 8-point font and a later instance of the variable is in a paragraph formatted as 15-point font. In the output, all instances of the variable will be formatted as 8-point font, regardless of the formatting applied to the placeholder variable in Designer. There are two exceptions to this behavior. If the imported HTML includes inline styles, those are honored because inline styles override formatting applied in Designer. Additionally, if you are producing unconverted HTML or HTML Email output, the imported HTML will be formatted as defined in the original file.

    To be sure that imported HTML appears in the output as expected, your imported HTML should be well-formed and consistent with the guidelines outlined in this section.
