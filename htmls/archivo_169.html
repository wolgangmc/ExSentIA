    The HTML Email output driver produces HTML files that are used for email-based communications.

    If you create an HTML Email output object for your communication, when the engine runs, OpenText Communications produces the following set of files:

    • HTML Email output file (.html) – The source HTML file

    • JPEG (.jpg) – These are the images that are imported into the design in their original format (such as JPEG or PNG).

    In HTML Email output, all of the text and paragraph styles that you applied in Design view are embedded in the *.html output files, either inline with the HTML elements or within the <head> element at the top of the files. Because all of the styles are embedded with the content in the HTML files, you do not receive a separate CSS file in the output.

    You can also include accessibility tags so that your HTML Email output can be read by a screen reader or text-to-speech converter.

    If you apply custom CSS styles to an email container design by linking a cascading style sheet object to the container design label, you can specify whether you want those custom CSS styles to be placed inline at the HTML element level or within the <head> element at the top of the *.html output files. You do not have the option to place the custom CSS in a separate, external file because email clients do not support external style sheets.

    After you create the output object, you can use the Properties panel to view and modify the following output properties. After making changes, click Save or the changes will be lost.

    General output settings

    The following sections provide information about the general output options.

    Accessibility standard

    If you want to create output that can be read by a screen reader or text-to-speech converter, you can enable accessibility for the output objects. In the Accessibility standard list, select the accessibility standard that you want to use.

    Alignment

    In the Alignment list, select the horizontal alignment that you want to apply to the primary container of the HTML Email output. Keep in mind that this setting does not affect the alignment of the content that is inside the primary container.

    Additional settings

    This output includes additional common settings. For more information about any of these settings, see the following locations:

    • Domains – For information about defining the domains to which a resource is assigned, see Section 3.13 “Associating resources with domains”.

    • Workflow – For information about managing an approval workflow, see Section 3.6 “Managing approval workflows”.

    • References – For information about viewing the linked resources connected to a selected resource, see Section 3.3 “Viewing resources in asset library views”.
