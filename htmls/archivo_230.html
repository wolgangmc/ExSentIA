    This set of topics provides an overview of the capabilities for leveraging external content, in formats such as HTML, PDF, or DOCX. It includes instructions for importing external content directly into a design or dynamically during a production run.

    You can import external content using the following methods:

    • HTML content can be imported at run time by using an HTML placeholder variable.

    • DOCX content can be imported as a larger set of DOCX files using Rationalization.

    • PDF content can be imported from a PDF DAS resource directly in Designer or as a larger set of PDF files using Rationalization.

      Content from a PDF resource object can be added and edited in Designer. Rationalized PDF content can be added and edited in both Designer and Content Author.

    • StreamServe template text content can be imported using Rationalization.

      This content can be added and edited in both Designer and Content Author.

    For more information about rationalization, see section 8.3.2 "Rationalizing external content" in OpenText Communications - Web Client User Help (CCWC250100-H-UGD)section 8.3.2 "Rationalizing external content" in OpenText Communications - Web Client User Help (CCWC250100-H-UGD).
