5.14.1   Creating communication set-based style sheets

    To use communication set-based style sheets, you first create a standard set of style objects that defines the formatting and appearance of paragraphs and text. Then you add those style objects to a style sheet. For example, you could create style objects such as Heading1, Footnote, Emphasis, TableText, and so on.

    When you produce HTML or HTML Email output from a communication that uses style sheets, the engine uses the paragraph and text styles that you apply as the class names for the elements, and wraps the text in the resulting HTML output.

    Example: For example, if you create a paragraph style named headline, and apply that style to a paragraph in Designer, the HTML output will include class="headline". Likewise, if you apply a text style named small_print, the HTML output will include class="small_print".

    Optionally, when you create a style sheet, you can also create alternate versions of that style sheet that are controlled by the customer's data. This set up allows the styles you use in the design to change based on the information in the customer data.

    Example: For example, you might want the colors in the output to be different depending on the type of account purchased by the customer, or you might want to change the font size based on a standard request for large print.

    To use style sheets to enforce corporate styles, you must complete the following high level tasks:

    1. Create text and paragraph styles.

      Important

      You must create at least one paragraph style before you can create a style sheet.

    2. Create a style sheet.

    3. Optional Create alternate style sheets.

    If you are creating style sheets for use with Content Author, you must complete additional tasks to associate the style sheet with your Content Author communication set and optionally define how styles are enforced in Content Author. For more information, see Using style sheets with Content Author.

    Creating text and paragraph styles

    Styles define text characteristics that affect the appearance of documents. Styles are implemented only if they are part of a style sheet. Prior to creating a style sheet, consider how styles work together.

    There are two types of styles:

    • Paragraph styles – Applies formatting and text characteristics, such as font type, size, color, and styling, to an entire paragraph. Applies paragraph-specific characteristics, such as layout, spacing, indent, and splitting to an entire paragraph.

    • Text styles – Applies formatting and text characteristics, such as font type, size, color, and styling, to selected text.

      Note

      You can apply text styles to selected text within a paragraph that has a paragraph style applied to it. However, text styles cannot overlap.

    Important

    If you modify and approve an existing style, the changes are applied everywhere that style is used. For Communication Designer, changes are visible the next time you run the engine or open any design in the editor that uses the modified style. For Content Author, changes are visible the next time you run the engine; however because the styles are included based on the communication, you must open individual objects such as messages and clauses from within a communication in order to view any style changes.

    To create a style:

    1. In the Design asset library, click the Common tab.

    2. Click Add > Style.

    3. In the Properties panel, add a name and optionally a description.

      Names do not support the following special characters: & / : * ! ? > < | + # %

      Important

      If you are producing HTML output, you must follow HTML naming conventions for style names, since they are used as class names in the HTML output. This means that each style name must be unique, begin with an alphabetic character, and contain only alphanumeric characters, hyphens (-), and underscores (_).

    4. In the Style type list, select whether you are creating a Paragraph or Text style.

    5. To define the appearance of the style in page-based layouts, click the Page tab and complete the following steps:

      1. To define the text properties of the style – In the Text area, specify the font properties, such as font type, color, point size, and styling. Text properties can be created for both text and paragraph style types. The options you select change the appearance in the Preview area.

        Note

        The preview area is intended as a sample of the selected style and is not an exact representation. For example, some font styling and line spacing options may not be shown accurately.

      2. To define the paragraph properties of the style – In the Paragraph area, specify the paragraph properties, such as layout, margins, spacing, justification, tab sizing, and splitting options. Paragraph properties can be created only for paragraph style types.

        • If you specify a Line spacing of Minimum, the lines must be at least the distance set in the adjacent Amount box. The space is increased if the font requires more room.

        • If you specify a Line spacing of Exactly, the lines are always spaced the distance set in the adjacent Amount box, even if the text overlaps. The spacing is applied after the line of text.

    6. If you want the style to appear different in HTML-based designs, click the HTML tab and complete the following steps:

      1. Clear the Use page styling check box.

      2. To define the text properties of the style – In the Text area, specify the font properties, such as font type, color, point size, and styling. Text properties can be created for both text and paragraph style types. The options you select change the appearance in the Preview area.

        Note

        The preview area is intended as a sample of the selected style and is not an exact representation. For example, some font styling and line spacing options may not be shown accurately.

      3. To define the paragraph properties of the style – In the Paragraph area, specify the paragraph properties, such as layout, margins, spacing, justification, tab sizing, and splitting options. Paragraph properties can be created only for paragraph style types.

        • If you specify a Line spacing of Minimum, the lines must be at least the distance set in the adjacent Amount box. The space is increased if the font requires more room.

        • If you specify a Line spacing of Exactly, the lines are always spaced the distance set in the adjacent Amount box, even if the text overlaps. The spacing is applied after the line of text.

    7. Click Create.

    Creating a style sheet

    A style sheet is a collection of styles. You must add a style to a style sheet before it can be applied to text in Designer. In Designer, you associate a style sheet with a communication design using the Properties panel in the Communication Composer.

    Before you create a style sheet you must have completed the following tasks: –

    • You must have created at least one paragraph style.

    • Any styles that you want to include in the style sheet must be approved.

    To create a style sheet:

    1. In the Design asset library, click the Common tab.

    2. Click Add > Style sheet.

    3. In the properties panel, add a name and optionally a description.

      Names do not support the following special characters: & / : * ! ? > < | + # %

    4. Optional To restrict Content Author users to only styles from this style sheet, select the Allow only styles from this style sheet when Content Author uses this style sheet. If this option is not selected, users in Content Author can apply custom formatting to designs.

    5. To add styles to the style sheet, complete the following steps:

      1. In the Included styles area, click Edit list.

      2. In the Styles dialog box, in the Available styles list, click + to add a style to the style sheet. Paragraph styles are identified with a paragraph icon ¶ and text styles are identified with an a icon . You must select at least one paragraph style.

        Tip

        You can preview a style by clicking the style name.

      3. You must identify a default paragraph style. To identify a default paragraph style, in the selected styles list, select the star icon adjacent to the paragraph style that you want to use as a default.

      4. Click Apply.

    6. Click Create.

    Note

    If a style is hidden after being added to a style sheet, the style name appears in gray italics in the style sheet properties. As with other hidden resources, if a style is in use and then subsequently hidden, it will still be visible where used, but will not be available for use in a new instance.

    Creating an alternate style sheet

    Optionally, when you create a style sheet, you can also create alternate versions of that style sheet that are controlled by the customer's data. Alternate style sheets use the same styles as the default style sheet, but allow you to apply different formatting properties. For example, your organization might use style sheets to reflect different appearances due to marketing, campaigns, or branding. Because default style sheets and alternate style sheets share styles, designers would only have to apply a heading once, and then that heading would appear with different fonts, colors, or alignment depending on the style sheet that was applied to the customer output.

    To create alternate style sheets:

    1. Create a style sheet. You must create a style sheet and add a default set of styles before you can create alternate style sheets.

    2. In the properties panel, select the Allow alternate style sheets check box.

    3. Adjacent to the Select an alternate style sheet box, click the Alternate style sheet selection button and select the variable that contains the static selection list that you want to use to control style sheet selection.

    4. To create a new alternate style sheet, complete the following steps:

      1. In the Alternate style sheets area, click Add.

      2. Enter a name for the alternate style sheet and select the static selection list value which specifies when this alternate style sheet should be used.

      3. Click Add.

    5. To modify the styles for an alternate style sheet complete the following steps:

      1. In the Alternate style sheet list, select the name of the style sheet that you want to modify.

      2. In the Included styles list, select the style that you want to modify.

      3. Click Modify style.

      4. In the Style editor dialog box, modify the style as needed.

      5. Click Update. A (modified) tag is appended to any styles that are changed for the alternate style sheet.

    6. Repeat step 4 through step 5 for each additional alternate style sheet you want to create.

    Using style sheets with Content Author

    With Content Author communications, you can use style sheets to enforce text branding in communication output. To specify how style sheets are applied in Content Author communications, you must complete the following tasks.

    To enforce style sheet formatting and prevent manual formatting in Content Author:

    1. Complete the basic creation of the style sheet.

    2. Add any alternate style sheets.

    3. Select Allow only styles from this style sheet when Content Author uses this style sheet.

    To make a style sheet available inontent Author:

    1. In the Communication asset library, click on the Communication sets tab.

    2. Double-click the communication set to which you want to add the style sheet.

    3. Click Add > Style sheet.

    4. In the Properties panel, select the style sheet you want to use.

    5. Click Insert. The style sheet you select is added to the Branding area.
