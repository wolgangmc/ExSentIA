2.5.4   Customizing Assisted Authoring for a communication

  • Communication asset library view

  • Orchestration asset library view
  • Monitor view
  • Rationalization asset library view
  • Dashboards view

    Assisted Authoring provides language-specific text analysis tools to users in Designer and Content Author. Assisted Authoring tools can measure readability, sentiment and tone, and grammar. If Assisted Authoring is configured for a language in a communication, a designer or author can apply the Assisted Authoring tools while editing text in that language.

    For more information about using Assisted Authoring in Designer and Content Author, see OpenText Communications - Designer User Help (CCCD250100-H-UGD)OpenText Communications - Designer User Help (CCCD250100-H-UGD) and OpenText Communications - Content Author User Help (CCCA250100-H-UGD)OpenText Communications - Content Author User Help (CCCA250100-H-UGD).

    If Experience AI Services (EAIS) are enabled for the OpenText Communications tenant, you can add or change the Assisted Authoring settings for any content language in any communication that you select in Design asset library view. Default Assisted Authoring settings can also be configured tenant-wide and then customized or used as-is in multiple communications.

    Important

    OpenText Communications does not include a German spelling dictionary, so a German dictionary must be installed in the tenant to use Assisted Authoring with German. If you add a German dictionary, specify the language code de_DE to ensure that Assisted Authoring works with German.

    Editing the Assisted Authoring settings in a communication

    If EAIS is enabled in the tenant, you can select a communication in Design asset library view and then edit its Assisted Authoring settings. For example, you can configure communication-specific versions of the following:

    • The list of content languages for Assisted Authoring.

    • The Assisted Authoring settings for each language in the list.

    Tip

    Initially, the Assisted Authoring settings for languages in a communication match the tenant-wide default settings. If a content language is configured for Assisted Authoring tenant-wide, then you can reset any communication-specific settings for that language to the tenant-wide default settings if necessary. For more information about resetting communication-specific settings to tenant-wide default settings, see Editing the Assisted Authoring settings for a language in a communication.

    Editing the Content languages list for Assisted Authoring in a communication

    You can add or remove languages from the list of content languages for Assisted Authoring in a communication. For example, you can add supported languages to a communication that were not added to the tenant configuration. Languages that were added in the tenant-wide Assisted Authoring settings intially appear in the list for the selected communication by default, but they can be removed.

    To edit the list of languages for Assisted Authoring in a communication:

    1. Enable Experience AI Services for the OpenText Communications tenant and then return to the home page.

    2. Click Design asset library view and then click Designer or Author, depending the type of communication that you want to edit, Communication (CD) or Communcation (CA).

    3. In the Designer or Author panel, click the row for a communication resource that is in the draft state and available for you to edit.

    4. In the Properties panel, click the Intelligence tab.

    5. On the Intelligence tab, click the Edit list button following the Content languages list.

    6. In the Manage Languages dialog box, you can add or remove content languages as follows:

      • To add a language: In the Available languages list, click the New list item icon for the language.

      • To remove a language: In the Added languages list, click the Remove list item icon for the language.

    7. Click Apply.

    Important

    For Assisted Authoring with German content: When adding a German dictionary, specify the language code de_DE. Otherwise, Assisted Authoring will not work for German content.

    Editing the Assisted Authoring settings for a language in a communication

    You can configure communication-specific language settings for any content language that supports Assisted Authoring.

    To edit the Assisted Authoring settings for a content language in a communication:

    1. Perform step 1 through step 4 in the previous procedure.

    2. On the Intelligence tab, choose the language that you want to edit in the Content languages list, and then click Modify details.

    3. In the Assisted Authoring dialog box, select the Enable check box for each Analysis type that you want to make available for the selected language.

      The licensed level of service for each analysis type appears after its check box. For information about licensing analysis types, see OpenText Experience AI Services Configuration Guide (CDPEAIS-PGD).

      Note

      For information about OpenText product compatibility, see the OpenText compatibly information provided in the release notes for the product. Release notes are available on My Support.

    4. Configure the settings for each enabled Analysis type, and then click Save. For detailed configuration instructions for each analysis type, see Configuring analysis types for Assisted Authoring

      Note

      If you have questions about the available text analysis types for a supported language, contact your Experience AI Services administrator.

    5. Optional If you want to reset all language settings in the Assisted Authoring dialog box to their tenant-wide default values, click Reset.

    6. Click Save.

    Configuring analysis types for Assisted Authoring

    In the Assisted Authoring dialog box, you can configure each analysis type for the selected content language.

    Configuring Readability analysis settings for Assisted Authoring

    The readability analysis tool gives business users statistics based on one of two different reading scales—the Flesch Reading Ease scale, and the Flesch-Kincaid Grade Level index. When you enable readability analysis, you can select a service provider as well as the readability scale that you want to use.

    To configure readability analysis:

    1. After you have selected the Readability analysis check box, in the Service provider list, select a provider:

      • OpenText—Uses OpenText Magellan to provide readability analysis.

      • Google—Uses Google Cloud Platform to provide readability analysis.

    2. In the Readability scale list, select a readability scale:

      • Flesch Reading Ease—The Flesch Reading Ease scale shows statistics based on the average number of words per sentence and the average number of syllables per 100 words.

      • Flesch-Kincaid Grade Level—The Flesch-Kincaid Grade Level readability index is also based on the number of syllables per word and the number of words per sentence. The resulting number indicates the approximate level of education, based on US guidelines, that is required to read and understand the text. This tool is available only when the spelling dictionary in Content Author is set to English (American).

    3. Based on the readability scale that you selected, set parameters in the Target range section and in the Acceptable range section:

      • Flesch Reading Ease scale:

        Value Meaning
        90-100 Very easy
        80-90 Easy
        70-80 Fairly easy
        60-70 Standard
        50-60 Fairly difficult
        30-50 Difficult
        0-30 Very difficult

      • Flesch-Kincaid Grade Level index: Select the approximate level of education, based on US guidelines, that is required to read and understand the text.

      When users in Content Author or Designer open the Assisted Authoring tools, they will receive an overall score that compares their content against the parameters that you set. Additionally, when they point to the numerical value for the overall readability of their content, they will see a tooltip that explains the scores for the scale that is being used.

    When users in Content Author or Designer open the Assisted Authoring tools, they will receive an overall score that compares their content against the parameters that you set. Additionally, when they point to the numerical value for the overall readability of their content, they will see a tooltip that explains the scores for the scale that is being used.

    Configuring Sentiment and tone analysis settings for Assisted Authoring

    Sentiment and tone analysis uses artificial intelligence provided by either OpenText Magellan or Google, depending on the option you select, to determine the overall attitude of content—positive, negative, or neutral—and is represented by a numerical score that ranges from -100 to 100. Keep in mind that the numerical values vary by provider, and they can be configured within Experience AI Services. For example, here are the recommended ranges for sentiment and tone when using OpenText Magellan as the provider:

    • Positive: A score greater than 33

    • Neutral: A score between -33 and 33

    • Negative: A score less than -33

    For information about configuring settings in Experience AI Services, see OpenText Experience AI Services Configuration Guide (CDPEAIS-PGD).

    Note

    For information about OpenText product compatibility, see the OpenText compatibly information provided in the release notes for the product. Release notes are available on My Support.

    Magellan also uses artificial intelligence to determine whether each sentence is fact or opinion. The number shown represents a blended score of all sentences that make up the content. Business users can use the navigation arrows on the toolbar to validate each sentence individually to ensure that sentiment and tone are consistent.

    Configuring Grammar analysis settings for Assisted Authoring

    The grammar analysis tool behaves much like traditional grammar checkers—it identifies issues and then makes individual suggestions to improve the content.

    To configure the Grammar analysis settings: – In the Grammar analysis section, in the Service provider list, select either LanguageTool or GrammarBot. For information about either of these service providers, consult their online documentation.
