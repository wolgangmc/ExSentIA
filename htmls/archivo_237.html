    OpenText Communications lets you retain accessibility information, such as alternate text, that is present in the tagging structure when you import the HTML content at run time and produce accessible HTML output.

    Specifying the reading language in accessible HTML

    If you're importing HTML content, and you want to specifically define the reading language that will be used by assistive technology tools for the imported content, then you must include the lang attribute in the imported HTML. If you do not include a lang attribute on the top-level tag, then OpenText Communications applies the language setting of the text paragraph that contains the placeholder variable as the default language.

    If you add a lang attribute on a <div>, <table>, or <p> tag, then OpenText Communications applies that language to all child elements contained within that tag, unless the child element includes a lang attribute.

    For example, suppose that you have the following imported HTML content:

    <div lang="es">
   <h2>¿Habla español?</h2>
   <p>Llame a uno de nuestros representantes de habla hispana.</p>
   <img src="phone.jpg" alt="Image of a phone" lang="en-US">
</div>

    In this example, the corresponding Designer element is an image embedded in a text box. The specified reading language for the text box is Spanish, and the reading language for the image's alternate text is English. Because the child element (in this case, the image), includes a lang attribute value, the alternate text of the embedded object would be read using English pronunciation rules.

    You can include the lang attribute for the following elements:

    HTML tag category Tags that support the lang attribute during HTML import
    Basic HTML structure

    • <html>

    • <body>

    • <h1> through <h6>

    • <div>

    • <p>

    Note

    With unconverted HTML imports, the engine removes the <html> and <body> tag from the resulting HTML output, and any language declarations made in those tags would be lost. For converted HTML, however, these tags are retained in the HTML output, as would the language declarations.

    Hyperlinks

    <a>
    Images

    <img>

    If an image is a hyperlink, and the lang attribute has different values on the <a> and <img> elements, then the behavior for imported HTML content differs from that of images placed in Designer. For images included in the imported HTML content, the lang attribute value on the <img> tag is honored in the output. For images placed in Designer, the lang attribute value on the <a> tag is honored in the output.

    For example, suppose that an image has French specified as the reading language for the alternate text and the hyperlink has English specified as the reading language. In typical HTML, it might appear like the following:

    <a href="www.url.com" title="Our website" lang="en">
    <img src="logo.jpg" alt="Notre logo d'entreprise" lang="fr">
</a>

    In accessible HTML and PDF output, French is specified as the reading language for both the alternate text and the hyperlink.

    For example, the imported HTML content would result in the following converted HTML output:

    <a href="www.url.com" title="Our website" lang="fr">
    <img src="logo.jpg" alt="Notre logo d'entreprise" lang="fr">
</a>

    Whereas the Designer content would result in the following HTML output:

    <a href="www.url.com" title="Our website" lang="en">
    <img src="logo.jpg" alt="Notre logo d'entreprise" lang="en">
</a>

    In the HTML output for either case, incorrect rules of pronunciation will be applied to either the image alternate text or the hyperlink. To avoid this issue, make sure that the lang attributes for the <a> and <img> elements contain the same value. If you are producing HTML output, then you can also choose to produce unconverted HTML output to maintain the language declaration for the image and alternate text. PDF output honors the language settings that you apply in Designer.

    Supported accessibility language codes

    If you want to use a variable as the Accessibility language control type for a language object, you need to create a scalar string variable that contains an ISO 639–1 or ISO 639–2 language code as its value.

    Important

    Keep in mind that accessibility languages are the languages that are available to be read by assistive technology tools. The list of accessibility languages does not represent the languages supported in OpenText Communications output.
