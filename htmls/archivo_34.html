    If you are using automatic schema management for your OpenText Communications databases, you can use the tenant creation wizard in System management view to create and set up a new tenant. If you are using manual schema management for your OpenText Communications databases, then you will need to first set up tenant schemas directly in the database for each of the OpenText Communications services, and then use System management view to associate the new tenant with the schema for each service in the OpenText Communications deployment.

    Alternatively, if you are using automatic schema management, you can specify the schema name and database credentials for each OpenText Communications service directly in the database. Then, instead of running Liquibase to create the database tables for the service resources, you can let the tenant creation wizard do the following:

    • Create the database tables in the schema.

    • Associate the schema and tables for each service with the tenant.

    Note

    Regardless of the schema management method that you use for your databases, you must use the OTDS REST API to create inherited tenants, which is the type of OTDS tenant that is used with OpenText Communications.

    To access the OTDS tenant REST API, use the following endpoint:

    {OTDS URL}/otdsws/api/index.html?tenants

    For each OpenText Communications tenant that you set up with the wizard, do the following:

    1. Install a product key for the new tenant. For information, see section 2.2.5 "Installing product keys for tenants" in OpenText Communications - Web Client User Help (CCWC250100-H-UGD)section 2.2.5 "Installing product keys for tenants" in OpenText Communications - Web Client User Help (CCWC250100-H-UGD).

    2. Sign in to the new tenant with your administrator credentials.

    3. Use Tenant management view to configure the roles and domains to make resources in the tenant available to users.
