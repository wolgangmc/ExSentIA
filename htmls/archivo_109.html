3.5   Applying metadata tags to resources

    Metadata provides the ability to easily identify resources in output files for use in downstream business processes, such as with OpenText Core Journey, and in output post-processing, such as applying additional CSS styles to specific design objects.

    You can add metadata to Designer communication resources, to individual design objects in these communications, and to DAS output queues and barcode resources. In email and web designs, you can also add metadata to containers. Output metadata is supported for the following output types:

    • HTML5

    • HTML (Email)

    • Messaging

    • PDF

    • Interactive

    Metadata tags are created and stored as DAS resources and can be managed in the same way as other DAS resource types. For information about managing resources in DAS, see Working with DAS resources.

    For information about applying metadata to resources in Designer, see OpenText Communications - Designer User Help (CCCD250100-H-UGD)OpenText Communications - Designer User Help (CCCD250100-H-UGD).

    Applying metadata to resources in the asset library

    To apply metadata to resources, you use the following views depending on the type of resource:

    • Communication asset library view—For output queues

    • Design asset library view—For Designer resources

    1. With the resource selected, click the Metadata (Class) tab in the Properties panel.

    2. Click the Assign metadata (class) button .

    3. In the Metadata (Class) dialog box, select the check box for each metadata (class) tag that you want to apply to the resource.

    4. Click OK to close the dialog box, and then click Save.

    Removing applied metadata

    To remove an applied metadata (class) tag, click the Remove metadata (class) button for that tag and then save your changes.
