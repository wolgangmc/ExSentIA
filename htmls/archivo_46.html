2.2.5.2   Step 2: Activate an OTDS resource for the tenant

    Each resource in OTDS represents an OpenText product. A product key that authorizes the use of the OpenText Communications deployment must be associated with a resource. You must have an active resource for your tenant so that OTDS can use it to authenticate users in your OpenText Communications deployment.

    Note

    If you are using a production product key and a non-production product key on the same tenant, then you must create separate resources for each product key.

    To activate a resource for the tenant:

    1. In the OpenText Communications web client, sign into the following URL as a tenant administrator user: 

      https://<host>:<optionalPort>/otdstenant/<tenant>/otds-admin/

    2. In the Setup menu, click Resources and then complete the following steps to create a new OTDS resource:

      Note

      If you used a bootstrap configuration to set up your OTDS resource, then you do not need to create a new resource. In the Actions menu for the existing resource, click Activation status, and then go to the next step.

      1. From the button bar, click Add.

      2. On the General tab, in the Resource Name field, specify a name for the resource and other general settings as needed, and then click Next.

        Tip

        Use a name that helps you easily identify whether the resource is associated with a production product key or a non-production product key.

      3. On the Synchronization tab, specify synchronization settings as needed, and then click Next.

      4. On the Principal Attribute tab, specify an authentication principal attribute, and then click Next.

      5. From the button bar, click Save.

    3. In the Resource Activation dialog box, copy the resource identifier string.

    4. Go to the Directory Services API for OTDS at the following URL:

      https://<host>:<optionalPort>/otdstenant/<tenant>/otdsws/rest

    5. In the Directory Services API, click Authorize near the top of the screen, and complete the following steps in the Available authorizations dialog:

      1. Specify scopes settings as needed, then click Authorize.

      2. If you are prompted to sign in, use OTDS administrator credentials.

      3. In the Available authorizations dialog, click Close.

    6. In the Directory Services API, complete the following steps to activate the resource: 

      1. In resources, click the /resources/{resource_id}/activate endpoint, then click Try it out.

      2. Paste the resource identifier value that you previously copied into the Resource ID box.

      3. Click Execute and scroll down to verify that you did not receive an error. The returned response should be something in the 200 range.

    7. Return to the OTDS web client, and in the Resource Activation dialog box, click Verify Activation.

    8. Click OK.
