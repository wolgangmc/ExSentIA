9.4   Monitoring orchestration jobs

    You can use the Monitor view to see the status of the jobs and flow instances that correspond to your engine orchestration jobs.

    When you send data to an input channel, it initiates new orchestration jobs for any published flow models that use that input channel. Each time a flow model is processed, the Orchestration service creates a corresponding flow instance.

    Jobs

    The Jobs panel displays the following information for orchestration jobs:

    Column Description
    ID

    ID of the orchestration job

    Trace ID

    Unique ID assigned for each run of the orchestration job. It can be used to follow the traces of a job through service log files.

    Channel

    Name of the input channel that initiated the orchestration job

    Start

    Processing start time for the job

    Finish

    Processing end time for the job

    Status

    Status of the job. Select the job to see additional information about the status on the General tab in the Properties panel.

    Triggered by The OpenText Communications role that initiated the orchestration job

    You can also view the associated flow instances for each job on the General tab in the Properties panel.

    In the Jobs panel, clicking the job ID opens the Flows panel and automatically selects the first flow instance for that job.

    Exporting job resources

    The export package information that is collected when you click the Export resource button for a job is only used to analyze job issues with support or Professional Services. The export package contains the job, the flow instances, the flow steps, the aggregated flow context snapshot, the flow model snapshot, the message file, and the standard delivery report. It does not contain any output files and cannot be re-imported.

    Tip

    When opening a support ticket related to a job, you should export the job resources and then attach the export package file to the ticket, along with the log files from all the services (design, batch, on-demand, orchestration, and so on). The Trace ID can be used to filter for all log entries associated with the exported job.

    Flows

    The Flows panel displays the following information for flow instances:

    Column Description
    ID

    ID of the flow instance that was initiated for a specific orchestration job

    Job ID

    ID of the orchestration job that was initiated by sending data to the input channel

    Flow model

    Name of the flow model that corresponds to the initiated flow instance

    Channel

    Name of the input channel that initiated the orchestration job

    Start

    Processing start time for the flow instance

    Finish

    Processing end time for the flow instance

    Status

    Status of the flow instance. Select the flow instance to see additional information about the status on the General tab in the Properties panel.

    On the General tab in the Properties panel, you can see the flow instance ID, job ID, a visualization of the flow instance, and the run history, which is a list of flow steps that were executed by this flow instance. You can click the More actions button at the end of the row of each flow step in the run history to select one of the following actions:

    • Show route—Changes the Run history list to show the flow steps that preceded the selected flow step.

      Once selected, this option is replaced by the Show all steps action, which, when selected, returns the Run history to the full list.

    • Show resources—Maps to the References tab with that flow step selected. This option is available only if job tracing has been enabled for the domain.

    • Reprocess flow step—Reprocesses the selected flow step and any subsequent steps in the flow model. This option is available only for failed flow steps.

    On the Files tab in the Properties panel, for each flow instance, you can also access the following files that are used or generated by the OpenText Communications engine:

    Type Files
    Input Input data file that was used to initiate the corresponding orchestration job
    Log files

    • Message file—The engine message file that was generated during the engine run for the flow instance

    • Delivery file—The output delivery manifest that contains a comprehensive list of the output composed during the engine run

    Output files The output generated from the engine run

    Note

    With the exception of message files, only user roles that have the Monitor job files permission have access to the contents on the Files tab. If you have questions about your access privileges, talk to your system administrator.

    If job tracing has been enabled for the domain, on the References tab in the Properties panel, for each flow instance, you can see which resources—and which version of those resources—were used, as well as who modified and approved them. You can also see which resources were used in each step of the flow. This information is available as long as the job is in Monitor view.

    Note

    Only resources that are required for fulfillment of a job are listed on this References tab. However, fonts are an exception to this; to see which fonts were used, check the References tab of the communication in Design asset library view.
