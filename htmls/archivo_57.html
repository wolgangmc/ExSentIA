2.3.2.1   Creating a new domain

  • Communication asset library view

  • Orchestration asset library view

  • Rationalization asset library view

    1. In the Domains panel in Tenant management view, click Add.

    2. In the Properties panel, enter a name for the new domain in the Name box, and optionally, enter a description.

      Names do not support the following special characters: & / : * ! ? > < | + # %

    3. If a tenant has both production and non-production keys associated with it, then choose one of the following options to specify which key the OpenText Communications engine uses for this domain:

      • To use an available production key, select the Production check box.

      • To use an available non-production key, clear the Production check box.

      Important

      Be sure to set this correctly when you create a new domain. You cannot modify this setting after you create the domain.

      Keep in mind that if the domain settings specify to use a key type that is not available in the tenant, the OpenText Communications engine uses whichever key is available. The following table describes how the behavior of the engine changes depending on the key that is used.

      Key type Engine behavior

      Production key

      • Output (including preview and simulation) does not contain the demo watermark.

      • Transaction count is incremented for qualified batch engine runs.

      Non-production key

      • Output (including preview and simulation) contains the demo watermark.

      • Transaction count is not incremented for qualified batch engine runs.

      For more information about OpenText Communications product keys, see section 2.2.5 "Installing product keys for tenants" in OpenText Communications - Web Client User Help (CCWC250100-H-UGD)section 2.2.5 "Installing product keys for tenants" in OpenText Communications - Web Client User Help (CCWC250100-H-UGD).

    4. If you want OpenText Communications to track the resources used for each job in the domain, select the Job tracing check box.

      When enabled, this option traces the flow instances that correspond to engine orchestration jobs. For each flow instance, it tracks which resources—and which version of those resources—were used, as well as who modified and approved them. For more information, see "Flows " in OpenText Communications - Web Client User Help (CCWC250100-H-UGD)"Flows " in OpenText Communications - Web Client User Help (CCWC250100-H-UGD).

      Note

      If you select Job tracing in a domain that has a communication that is using the High-throughput (realtime) option, realtime processing for the communication will be significantly degraded. For more information about the realtime option, see section 2 "Configuring flow contexts" in OpenText Communications - Orchestrator User Help (CCMCEORC-H-UGD)section 2 "Configuring flow contexts" in OpenText Communications - Orchestrator User Help (CCMCEORC-H-UGD).

    5. Click Create.
