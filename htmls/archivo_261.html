9.1   Adding orchestration flow resources

    In Orchestration asset library view, you can create the resources that are used to configure the orchestration flows that determine the manner in which engine runs are processed to produce customer output. Orchestration flows are defined using the following resources:

    • Flow model—Defines the data flow for engine orchestration jobs, starting from incoming data, continuing through document production and ending with document output.

    • Flow context—Contains the input and output channel definitions and the communication settings that are used in designing flow models. All channels and communications, defined in all flow contexts inside the same tenant and domain as a flow model, are available when designing the flow model.

    • Flow script—Lets you create reusable flow scripts in order to implement orchestration logic. These scripts can be assigned to flow context items as before and after events, and also to script processors. Flow scripts are composed using JavaScript.

    • Alert model—Lets you create an alert flow based on a job state change. When an orchestration job starts, all published flow models also trigger any existing published alert flows.

    • Notification model—Lets you create a notification flow based on a flow instance or a flow step event. A notification flow model links to one or more regular flow models, and runs in the regular flow model as a flow instance. If the notification flow fails, the job also fails.

    You use the Flow panel in the Orchestration asset library view to create flow resources, and then configure these resources in Orchestrator or the Flow script composer. For more information about defining orchestration flows or creating flow scripts, see OpenText Communications - Orchestrator User Help (CCMCEORC-H-UGD)OpenText Communications - Orchestrator User Help (CCMCEORC-H-UGD).

    Note

    After they are created you must publish your flow resources to make them available for processing by the Orchestration service.

    To work with flow resources, users must be members of at least one OTDS group that is assigned a role with at least the Access and Creator web client access permissions enabled for Orchestration asset library. For reviewers, the Approver option must also be enabled.

    Creating a new flow resource

    1. In the Flow panel, click Add, then click one of the following options in the selection list.

      • Flow model

      • Flow context

      • Flow script

      • Alert model

      • Notification model

    2. In the Properties panel, specify a name and, optionally, a description for the resource that you want to create.

      Names do not support the following special characters: & / : * ! ? > < | + # %

    3. Click Create.

    To open a flow context or model in Orchestrator, or open the Flow script composer, select the resource, then click Open.

    Note

    Communication settings in flow contexts can be defined only for approved package files or communications in the current tenant and domain.

    Duplicating a flow model or flow script

    If you want to create a copy of a flow model or flow script, you can duplicate it. This might be useful if you want a similar flow model or flow script with only minor modifications.

    To duplicate a flow model or flow script, in the Orchestration asset library view, in the Flow panel, select the flow model or flow script that you want to copy and click Duplicate.
