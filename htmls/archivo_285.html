10.1   Job status widget

    The Job status widget displays a graph of the flow instance data corresponding to the engine orchestration jobs in the current domain. The data can be filtered by flow model, input channel, status, or timeline.

    Note

    To view job status information, user roles must have the flow context write Resource access permission. If you have questions about resource permissions, talk to your system administrator.

    To add a job status widget:

    1. Click the Dashboards button.

    2. Click Add widget.

    3. Enter a name and, optionally, a description for the widget.

      Names do not support the following special characters: & / : * ! ? > < | + # %

    4. In the Widget type list, select Job status.

    5. In the Widget style list, select an initial size for the widget based on the number of tiles it covers. Options include Large (2x2), Medium (1x2), and Small (1x1).

      After you create the widget, you can customize its size by dragging the edges of the widget. Changing the size of a widget automatically changes its style, as well. For example, changing a 2x2 widget to 1x1 alters the style from Large to Small. If you want to resize the widget but do not want the style to change, then select the Lock widget style check box.

    6. In the Filter by list, select whether to show flow instances or jobs.

      • If you select Flow instances, use the Flow model list to specify which jobs are visible, based on their associated flow models.

      • If you select Jobs, use the Input channels list to specify which flows are visible, based on their associated input channel.

    7. In the Status list, select whether to include or exclude one or more of the following job statuses: Canceled, Complete, Deleted, Error, Holding, Started, or Waiting.

    8. In the Timeline list, select one of the following options: Last day, Last week, Last month (default), All time, or Custom.

      If you select Custom, use the From and To boxes to enter a range of dates. You can also click the calendar button to select a date.

    9. Click Create.

      After the widget is added, the dashboard changes to edit mode.

    10. Click the Dashboards button.

    11. Click Save.

    Tip

    For best performance, it is recommended to not exceed four million flow instances. If necessary, you can adjust the timing interval of the orchestration cleanup cron job to achieve this.
