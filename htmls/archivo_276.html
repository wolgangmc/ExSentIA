9.7.1   Setting up a sort index data source

  • Communication asset library view
  • Design asset library view
  • Orchestration asset library view
  • Monitor view
  • Rationalization asset library view
  • Dashboards view

    A sort index data source in Designer uses a sample file with a columnar layout. The sample file is then used to create the sort index data source and mapped in the data source editor for use by the engine.

    Complete the following tasks:

    1. Create a sort index data source object

    2. Map the the sort index data source

    Layout considerations for sort index data sources

    To have a valid sort index data source, keep in mind the following layout considerations for any sample file that you want to use to map a sort index data source:

    • Reserve a 20-character area within the text file for the sort index key information. You can use spaces or any character, such as X. The engine writes the key to the file during the pre-sort processing engine run and reads the key information during the post-sort processing engine run. You will map this area to the 'SYS_SortIndex' system variable.

    • Enter columns of information in the specific sequence and fixed lengths you want them to appear in the sort index data source. For example, if you want to use a five-digit ZIP Code, reserve five spaces. If you want to use the extended ZIP code, reserve ten spaces (to include the hyphen).

    • Enter data that uniquely identifies the customer and any data areas you plan to sort or change during the sorting step.

    • (Optional) Depending on how you want to sort data and, if you are using bundling, which documents you want to include in bundles, you might want to include space in the sample layout text file for specific system variables you can map later including a 15-character area for SYS_SortIndexDocument.

    Creating a sort index data source

    This task assumes you have added the sample file that you want to use to the Communications asset library. For information about how to add a sample file, see section 5.13 "Adding sample files" in OpenText Communications - Web Client User Help (CCWC250100-H-UGD)section 5.13 "Adding sample files" in OpenText Communications - Web Client User Help (CCWC250100-H-UGD).

    To create the sort index data source

    1. In the Communications asset library, click the Data tab.

    2. Click Add, then click Data source.

    3. In the Properties panel, enter a name and optionally a description.

      Names do not support the following special characters: & / : * ! ? > < | + # %

    4. In the Data source type list, select Sort index.

    5. In the Data source type list, select Columnar.

    6. To select the sample file to use, click Find sample file and then select the sample file that you want to use.

    7. Click Create.

    Mapping a sort index data source

    To map the sort index data source

    1. In the Communications asset library, click the Data tab.

    2. Select the sort index data source that you want to map.

    3. In the Properties panel, to open the sort index data source in Mapping view, click Open.

    4. In Mapping view, complete the following steps to map the sort index data source to the appropriate variables:

      1. Highlight the 20-byte area that you reserved for sort index keys, and map it to the 'SYS_SortIndex' system variable. This variable contains information to find the customer document in the sort data files that are created by the engine and cannot be changed or deleted during the sorting process.

      2. Map the remaining data areas with variables used in the customer data. You can also map other system variables, if applicable.
