    You can use the asset library panel in the Communication asset library view to add output queue objects to the asset library for use with a communication. Output queues function as containers for production settings, including those for output drivers and inserter objects.

    You can add multiple output queues to a communication set to produce output with a variety of high-volume settings and output driver settings, including a mix of print and electronic drivers, while running the engine only once in OpenText Communications. When a communication set has multiple output queues, you can designate one to be the primary queue.

    Creating an output queue object

    1. In the Delivery panel, click Add, then click Output queue in the selection list.

    2. Specify a name and, optionally, a description for the output queue.

      Names do not support the following special characters: & / : * ! ? > < | + # %

    3. In the Output list, select the output driver for this queue from the list of available outputs.

    4. Click Create.

    Generating queue report files for tracking and quality control
    Using breaks to create multiple output files

    You can use the options on the Breaks tab to divide output into multiple files (sometimes called document subsets). You can also use breaks for archival purposes. For example, you can create a separate output file for each customer, or you can separate the output into files of 5000 pages each.

    Note

    Break options are unavailable for output queues that use Messaging as the output type.

    The following break options are available:

    • Break options—Indicates where to place a convenience break in the output. Select from the following options:

      • None—No breaks added.

      • Customers—Adds breaks after the specified number of customers.

      • Documents—Adds breaks after the specified number of documents.

      • Designs—Adds breaks after the specified number of designs.

    • Count—Specifies the number of customers, documents, or designs after which to include a break. The default value is 1.

    • Can split customers—Allows documents or designs for a customer to be split into multiple output files. By default, this check box is not selected.

      Note

      Output queues that use HTML or Interactive as the output type automatically have the Break options set to Customers and the Count set to 1. These are fixed settings.

    Generating queue report files for tracking and quality control

    You can use the Report tab to configure the creation of a custom report file, often called a queue report or custom queue report. for the output queue. This report can be used as an audit or quality control file. Unlike regular report files, queue reports generate queue-specific information rather than global information about a communication. For example, you might want to track information about how many customers are processed through the queue or how many documents are sent through the queue. The report is generated as a JSON or XML file and can be imported into other programs for formatting and data manipulation. You must have created a report file object before you can add it to the output queue.

    The following queue report settings are available:

    • Layout—Specifies the data source to use as the layout for the report file.

      Note

      The only data source objects that can be used for a layout are Report objects that have their I/O timing option set to Queue break and end of queue or Completion of all customers.

    • Report file name type—Indicates if the report file name is set by a variable or static text.

    • Report file name—Specifies the name of the report file.

    • Break method—If you have set up convenience breaks for output, you can set the report file to break at the same time by selecting When queue breaks in the Break method list. To turn off this option, select None.
