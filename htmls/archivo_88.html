2.5.3.1   Configuring OpenText Core Journey integration

    Important

    This configuration is domain-specific. To use the same configuration in a different domain, you must recreate the configuration in that domain.

    1. In the Domains panel in Tenant management view, click the domain for which you want to enable OpenText Core Journey integration.

    2. In the Properties panel, click the CXI configuration tab.

    3. Select the Enable Journey integration check box.

    4. To specify OpenText Core Journey connection information, use the following boxes:

      • Journey URL—Enter the OpenText Core Journey URL to which you want to post data.

      • Journey client ID—Enter the client ID that is used to connect to OpenText Core Journey.

      • Journey client secret—Enter the client secret that is used to connect to OpenText Core Journey.

      • Journey tenant ID—Enter the OpenText Core Journey tenant to which you want to post data.

    5. To specify the event schema information, use the following boxes:

      • Schema source—Enter a name that will be used to identify the schema source. This name must be unique for each domain.

      • Schema version—Enter the version of the schema.

    6. To enable audit logging for this OpenText Core Journey integration, select the Enable audit logging check box.

    7. To clean up event data that has already been sent to OpenText Core Journey from the database, select the Enable cleanup check box. In the Cleanup interval box that appears, enter the number of days to wait before purging data that has already been sent to OpenText Core Journey.

      Note

      By default, the cleanup cron job for the Journey integration in your deployment runs every 50 minutes. You can change this frequency by using a Helm command to set the ei.schedule.cleanup parameter. For more information, see OpenText Communications - Cloud Deployment Guide (CCMCE250100-IGD)OpenText Communications - Cloud Deployment Guide (CCMCE250100-IGD).

    8. Optional Click Validate to test the configuration.

      Validation confirms that you have access to the OpenText Core Journey integration. If you receive a failure, contact your OpenText Core Journey administrator.

    9. Click Save.
