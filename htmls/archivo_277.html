9.7.2   Setting up a communication for pre-sort processing

  • Orchestration asset library view
  • Monitor view
  • Rationalization asset library view
  • Dashboards view

    To set up your communication for pre-sort processing, you must select an output queue to use for sorting and bundling. Then, optionally, you can define the bundling process to group customer documents, such as a group of customers at one address (also known as householding) or in a hierarchical arrangement.

    For pre-sort processing, complete the following steps:

    1. Select and define an output queue for pre-sort processing.

    2. Optional Define the bundling process to group customer documents.

    To set up an output queue for pre-sort processing

    An output queue is required for the pre-sort process.

    1. In the Communications asset library, click the Delivery tab.

    2. Select the output queue that you want to use for sorting and bundling.

    3. In the Properties panel, click the General tab.

    4. Select the Use sorting and bundling check box.

    5. In the Sort file box, select the sort index data source.

    6. Optional To use convenience breaks on the sort index data source and sort data files, select the Use convenience breaks check box.

    To use bundling to group customer information

    Bundling is an optional process that allows you to group customer documents, such as a group of customers at one address (also known as householding) or in a hierarchical arrangement. Bundling is controlled by applying at least one control variable on which to bundle the data. You can also create multiple levels of bundling applying multiple control variables. For example, you might use the customer state as the main bundling criteria, the customer city as a second-level variable, and postal codes as the third-level variable.

    1. In the Communications asset library, click the Delivery tab.

    2. Select the output queue that you want to use for sorting and bundling.

    3. In the Properties panel, click the General tab.

    4. Make sure that the Use sorting and bundling check box is selected.

    5. In the Bundles area, for each bundle level, complete the following steps:

      1. Click Add.

      2. In the Control variable box, select a variable to identify each level of bundling. The first control variable is the main bundle, or the largest container. If you include additional bundles, each subsequent variable is a subcategory, within the previous bundle. You can use any combination of system variables and user-defined variables.

        Note

        So that system variables can help you assign levels of bundling and derive totals for each level, there are five separate versions of each bundling variable, each with a suffix number ranging from 1 to 5. Because system variables have separate versions which allow you to apply a different variable for each bundling level, you can set each level of page or customer formatting differently. The following table lists system variables you can use with bundling:

        Variable

        Description

        'SYS_BundleCustomer1' to 'SYS_BundleCustomer5'

        Defines the current customer number in a given level of bundling

        'SYS_BundleCustomerTotal1' to 'SYS_BundleCustomerTotal5'

        Defines the total number of customers in a given level of bundling

        'SYS_BundleInBreak1' to 'SYS_BundleInBreak5'

        Gives the bundle sequence number within the current convenience break

        'SYS_BundleInQueue1' to 'SYS_BundleInQueue5'

        Gives the sequence number of the current bundle within the current output queue

        'SYS_BundlePage1' to 'SYS_BundlePage5'

        Defines the current page number in a given level of bundling

        'SYS_BundlePageTotal1' to 'SYS_BundlePageTotal5'

        Defines the total pages in a given level of bundling

        'SYS_BundleSheet1' to 'SYS_BundleSheet5'

        Defines the current sheet count in a given level of bundling. Duplex operation counts two pages as a sheet, and simplex operation counts each page as a sheet.

        'SYS_BundleSheetTotal1' to 'SYS_BundleSheetTotal5'

        Defines the total sheet count in a given level of bundling

      3. Optional In the Cover page box and Trailer page box, select a cover page and trailer page for each set of documents within the bundle.

        Tip

        Since you can place barcodes on cover and trailer pages, you could use the system variable 'SYS_BundleCustomer' to control an inserter for each bundle.

      4. Repeat step 5.a through step 5.c for each level of the bundle, up to five levels.
