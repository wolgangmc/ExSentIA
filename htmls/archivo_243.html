8.3.2.2.2   Creating a migration set

  • Communication asset library view
  • Design asset library view
  • Orchestration asset library view
  • Monitor view
  • Rationalization asset library view
  • Dashboards view

    When you add a migration set, you are importing a ZIP file that contains the files that you want to make available for rationalization.

    Before you create a migration set, you must first create the ZIP file that contains the content that you want to import. The ZIP file needs to contain one of the following:

    • One or more DOCX files

    • One or more DOCX files and any XML driver files that were used to create the DOCX files

    • One or more PDF files

    • One or more PDF files and any XML driver files that were used to create the PDF output

    • StreamServe template files

    After you add a migration set, you must rationalize the set in order to create the fragments that can be used in Designer and Content Author.

    The rationalization process identifies any imported content that matches approved DAS content, including partial as well as exact matches. This can help you to avoid duplicating content that already exists in DAS. Only text content is considered, and text formatting is ignored.

    To create a migration set:

    1. In the Migration sets panel in Rationalization asset library view, click Add.

    2. In the Properties panel, specify a name and, optionally, a description for the new migration set.

      Names do not support the following special characters: & / : * ! ? > < | + # %

    3. In the Migration type list, select the source of the content to be rationalized:

      • DOCX—Microsoft Word Open XML files

      • PDF—Portable Document Format files

      • Template—StreamServe 16.6.x templates

    4. In the DAS matches list, select which DAS communication type to use when comparing DAS content to the rationalized content. This can help you reduce the possible number of duplicate fragments that the rationalization process creates.

      • All communication types—Compares rationalized content to both communication set-related and design pack-related DAS content.

      • Communication set—Compares rationalized content to only communication set-related DAS components.

      • Design pack—Compares rationalized content to only design pack-related DAS components, clauses, text messages, and graphic messages. You can further narrow the duplicates by selecting a specific design pack that was imported to DAS in the Design pack list.

    5. Click Import files.

    6. In the file browser, select the ZIP file that contains the files you want to rationalize, and then click Open.

    7. Optional In the Current rationalization set tolerance area, use the slide or enter a value from 50 to 100, and then click Save. This value determines how alike that different fragments must be before Rationalization considers them to be a match—a higher number requires them to be more alike, and a lower number requires them to be less alike. You can also set this value before you rationalize the newly created migration set.

      The default tolerance value is 50. Values below 50 are not allowed.

    8. Click Create.

    In the Migration sets panel, the status indicator for the migration set is gray. To use the content from this migration set in a design, you must first run the rationalization process to identify the content fragments in the imported files.
