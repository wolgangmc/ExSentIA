    The AFP output driver produces the necessary resources to support the print stream for IBM's Advanced Function Presentation (AFP) format printers.

    After you create the output object, you can use the Properties panel to view and modify the following output properties. After making changes, click Save or the changes will be lost.

    General output settings

    The following sections provide information about the general output options.

    Search key type

    In the Search key type box, select the type of search key to apply to the output. For more information about using search keys, see Section 4.12 “Setting up search keys”.

    Resolution (DPI)

    You can customize the output resolution settings to ensure that output is clear and readable. The output resolution specifies the sharpness of text and graphics, measured in dots per inch (DPI). If you are generating printed output, use a DPI at or lower than the maximum resolution supported by the output device that you are using for the best processing performance.

    In the Resolution (DPI) list, select the DPI setting that you want to use from the available options.

    Simplex/Duplex

    Set up simplex or duplex output. In the Simplex/Duplex list, select one of the following:

    • Simplex – Prints only on one side of a page

    • Duplex – Prints on both sides of a page

    Color mode

    Specify the way color is produced in output.

    Color model

    Specify whether to use RGB or CMYK colors in output.

    Create AFP index

    If your organization uses an archival system that requires an index file, select Create AFP Index to generate the index file during the production run. Then, in the Index file box, enter the name and location of the index file that is created by the engine for the production run.

    Unprintable areas offsets

    For printed output, you can set the print stream so that you do not print on page edges. You might want to set these unprintable areas because of output device limitations, because you are using a pre-printed or non-standard stock, or because of post-processing trimming. Keep in mind that all desktop printers have an unprintable area, and the output is offset by the printer automatically. A centralized production printer does not have the same restrictions. If you want to print from edge to edge when you use a desktop printer, consult your desktop printer manufacturer documentation for information.

    To specify unprintable areas for page margins, in the Unprintable area offsets area, enter the values of the unprintable areas for the edges of your pages. If the page is duplex, you can define offsets for both the front and the back of the page. In the Front and Back boxes, enter the x (short side) and y (long side) measurements for each side of a printed sheet.

    BDT/EDT placement

    Depending on whether you want to create one large output file or to place a break between each customer record, you can specify where to place Begin Document (BDT) and End Document (EDT) structured fields in the AFP print stream.

    To specify where to place BDT and EDT structured fields, select one of the following options from the BDT/EDT placement list:

    • Around entire file – Create one long document containing the entire output.

    • Around each customer – Place a break between each customer record, so each customer record is considered a separate document by the printer.

    Character fidelity

    Use character fidelity settings to make sure fonts appear correctly in output. OpenText Communications pulls metrics, such as height and width, from the font. By default, the engine uses integer measurements, which are used most often by bitmap fonts. However, rounding errors can cause the cursor position in your output to be different from what you expected, especially if you use outline fonts. Usually, this issue can be corrected by increasing the output resolution. If this solution does not work, you can adjust the character fidelity.

    For best results, use the default settings for character positioning. If you are experiencing issues with overlapping text or incorrect spacing between words or characters, the following can also influence your character positioning:

    • Adjusting your resolution settings.

    • If you are using multiple output queues for production, some character fidelity settings might be affected.

    To adjust character fidelity settings:

    In the Character fidelity list, select one of the following options to specify how text is placed:

    • Auto, character – This option is the default. Use this setting unless you are having issues with overlapping text or incorrect spacing in your output.

    • Auto, word – Adds positioning commands for each word in the output only if required; otherwise, positioning commands are added for each line of text. This option should allow you to set a lower print resolution, which reduces the size of images in output.

    • Auto, line – Adds positioning commands for each line of text and automatically positions text within that line. This option produces the smallest output size.

    • Position every word – Applies placement logic to every word in the output, which increases print sizes. This setting forces OpenText Communications to apply a new x/y coordinate for every word, rather than one x/y coordinate per line of text.

    • Position every character – Applies placement logic to every character in the output, which significantly increases print sizes. This setting forces OpenText Communications to apply a new x/y coordinate for every character, rather than one x/y coordinate per line of text. If you choose this option for any output driver in a multiple queue production run, the setting is applied to the text on all charts, such as data labels.

    Text position adjustment

    If you are using more than one output queue, you can adjust the text position to accommodate for font differences between the output drivers that are associated with those output queues. You can use the text position adjustment feature to make output that uses an outline font look like output that uses bitmap fonts.

    In the Text position adjustment list, select one of the following options:

    • No adjustment – Text output is not changed.

    • Adjust at word boundaries – OpenText Communications inserts position commands as necessary at word boundaries.

    • Adjust characters – OpenText Communications inserts position commands as necessary on a per-character basis. Text is not searchable when you select Adjust characters.

    Resource settings

    The following list provides information about managing output resources.

    Font usage

    Specify whether to embed or reference fonts. In the Font usage list, select one of the following options to indicate whether fonts used in a communication should be included in the output:

    • Embed all fonts – All the fonts used in an application are embedded in the output. Using this option results in a larger file size, but it does not affect processing time and provides OpenText Communications the most control over the printed output.

    • Reference all fonts – None of the fonts in a communication are included in the output. Referencing all fonts can help make the print file as small as possible. Keep in mind that when you reference fonts, your output might look slightly different from your design. For example, if you send files to customers who do not have the same output device or viewing software, these customers might not have the same fonts, in which case the output device or viewing software might either substitute a different font, or they might receive errors.

    Font type

    Specify the type of font to embed in the communication. OpenText Communications supports any font loaded on your system and converts it to the correct font type for your output device.

    AFP output supports only OpenType fonts. This font is selected automatically.

    Image processing

    Specify where to place static images in the print stream. Static images can be placed at different places in the print stream. The options you use to specify where static images are placed depend on the output device or viewing software you will use. For example, some output devices can read resources from the top of the file into memory and then reuse them each time the image is called.

    To specify where to place static images:

    In the Image processing list, select one of the following options:

    • Include on each page – Includes a copy of all the images used in the application in your output file on each page where they occur

    • Place at top of print stream – Includes a copy of the images at the top of the print stream for insertion as needed. This option can increase production processing speed.

    Resource inclusion

    You can choose to include all available resources (for example, fonts and images) in your output, or you can include only the resources that are used.

    To control which resources are included in your output:

    Select one of the following options in the Resource inclusion list:

    • Default – All available resources that are designed in your communication are included in the output file, even if they are not used. For example, if you have used rules to exclude some content, the resources in that content are still included in the output file.

    • Used resources only – Only the resources used by your communication are included in the output file. If you select this option, programs can reference resources as needed rather than loading all available resources for each page. The Used resources only option can slow the engine run; however, it can also reduce the file size, depending on the number of resources used by your application.

    Create and include Formdef in print file

    Form definition (formdef) files are resources that define the general characteristics of the physical printed page, including duplexing and the number of copies. In OpenText Communications, you can create an inline formdef and include it at the top of the AFP print stream.

    If you choose to create output with both simplex and duplex pages, you must create an inline formdef.

    To create an inline form definition file:

    1. Select the Create and include Formdef in print file check box.

    2. In the Formdef name box, enter the name for the formdef file. The formdef file name cannot exceed eight characters. The default name is FORMDEFR.

    Additional settings

    This output includes additional common settings. For more information about any of these settings, see the following locations:

    • Paper bins – For information about setting up paper bins for an output, see Section 4.10 “Setting up paper bins”

    • Domains – For information about defining the domains to which a resource is assigned, see Section 3.13 “Associating resources with domains”.

    • Workflow – For information about managing an approval workflow, see Section 3.6 “Managing approval workflows”.

    • References – For information about viewing the linked resources connected to a selected resource, see Section 3.3 “Viewing resources in asset library views”.
