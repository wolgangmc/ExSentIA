    If you can approve resources that are associated with a domain, and the domain is configured to use the simple workflow, you can approve drafts without submitting them for approval.

    Note

    For more information about configuring user roles to approve resources, see Configuring roles and permissions.

    If a domain is configured to use the simple workflow, the resources associated with that domain have the following workflow states and change options:

    Current state

    Change options for simple workflow

    Draft

    Approve

    Approved

    		Make new draft

    When you initially enable the simple workflow, some resources might be in the Submitted for approval or Rejected state. In these cases, your change options are as follows:

    Initial state

    Change options for simple workflow

    Submitted for approval

    Approve or Return to draft

    Note

    In the simple workflow, Reject is not an option in the Submitted for approval state.

    Rejected

    Return to draft

    For more information about using the simple workflow, see Section 3.6 “Managing approval workflows”.
