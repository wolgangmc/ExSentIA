9.4.3   Filtering jobs and flows

    By default, the Monitor view panels display all available jobs or flow instances. To make it easier to locate a specific job or flow instance, you can filter the lists.

    To filter the displayed jobs list:

    1. Click the Filter button .

    2. In the Results filter dialog box, specify the following settings as needed:

      • ID—The ID of the orchestration job

      • Channel—The name of the input channel that initiated the orchestration job

      • Job type—The type of job, based on how it was initiated.

        Initiator Job type
        Published channel Communication
        Event Alert
        Orchestrator simulation Simulation

      • Job mode—Indicates if the job corresponding to the flow instance is batch or on-demand

      • Start—Use the From and To boxes to enter a range for the job start date and time. You can also click the calendar button to select a date and time instead of manually entering it.

      • Finish—Use the From and To boxes to enter a range for the job finish date and time. You can also click the calendar button to select a date and time instead of manually entering it.

      • Expiration date—The date when a waiting or held job expires.

      • Status—The status of the job

      • External ID—The external ID on the current job

        An asterisk (*) can be used as a wildcard character in the External ID field to match zero or more non-space characters. This ID can include multiple asterisks.

      • Triggered by—The OpenText Communications role that initiated the orchestration job

    3. Click Apply.

    To filter the displayed flows list:

    1. Click the Filter button .

    2. In the Results filter dialog box, specify the following settings as needed:

      • ID—The ID of the orchestration flow instance

      • Job ID—The ID of the orchestration job that was initiated by sending data to the input channel

      • Flow model—The name of the flow model that is used for the flow instance

      • Channel—The input channel that triggered the job and the starting point of the flow model in the flow instance

      • Flow type—Type of flow model that is used for the flow instance

      • Job mode—Indicates if the job corresponding to the flow instance is batch or on-demand

      • Start—Use the From and To boxes to enter a range for the flow instance start date and time. You can also click the calendar button to select a date and time instead of manually entering it.

      • Finish—Use the From and To boxes to enter a range for the flow instance finish date and time. You can also click the calendar button to select a date and time instead of manually entering it.

      • Expiration date—The date when a waiting or held flow instance expires

      • External ID—The external ID on the current flow instance

        An asterisk (*) can be used as a wildcard character in the External ID field to match zero or more non-space characters. This ID can include multiple asterisks.

      • Status—The status of the flow instance

    3. Click Apply.

    Using the calendar to select a date and time

    The calendar dialog box allows you to select a specific date and time for the start or finish of a job or flow instance. Keep the following points in mind:

    • The Clear button removes any previously set date and time, and closes the calendar dialog box.

    • If your browser is set to use a 12–hour format and you enter a time in 24-hour format, it is converted to 12–hour format.

    • The Clear time button resets the time to 12:00:00 AM for 12–hour format or 00:00:00 for 24–hour format.
