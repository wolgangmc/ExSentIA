9.5   Creating a job using a local driver file

  • Communication asset library view
  • Design asset library view
  • Orchestration asset library view
  • Monitor view
  • Rationalization asset library view
  • Dashboards view

    If you want to run an orchestration job using a local driver file that is not available in DAS, you can create a job manually in Monitor view. This process uploads a local driver file that you specify and it is used for the REST request that creates the new job when run.

    To run a job using a local driver file:

    1. In Monitor view, on the Jobs tab, click Add.

    2. In the New job dialog box, click the File selection button to open the file browser and select the driver file that you want to use, then click Open.

    3. Select an Input channel for the job. Only input channels from published flow contexts in the current domain are available.

      If the selected input channel is an HTTP(S) or Interactive fulfillment channel you must also specify which job processing mode (batch or on demand) to use in the Job mode list.

    4. Optional Enter an External ID to help identify the job.

    5. Optional If you want to create custom header entries, then you can add one or more name-value pairs.

      1. Click Next.

      2. Click the Variables button and fill out the following fields:

        • Name—Specify the name for the request header element.

          Names in the header request are case-insensitive and must be unique. An error is issued if you attempt to specify a duplicate name.

        • Value—Specify the value of the request header element.

      3. Click Add.

      To make any changes or remove a name-value pair, click the More actions button at the end of the row.

    6. Click Run.
