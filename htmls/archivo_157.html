    The system variables in the following table are related to data files.

    Note

    In the following table, if a system variable name is in bold, its value can be calculated using a rule or formula.

    Table 4.7: Data related system variables

    Variable name

    Description

    SYS_CustInvalidDataLevel

    An invalid data level for the current customer, set when the engine reads or computes a variable and detects invalid data. The level is based on the action set in the variable's validation. The flag indicates the maximum value read for a customer:

    • 0 = No invalid data

    • 1 = Invalid data, continue, or set to default

    • 2 = Invalid data, error

    • 10 = Invalid data in this document. Skips all other documents for the customer.

    Tip

    You can also set the ‘SYS_CustInvalidDataLevel‘ system variable using the MSG or MSGCHANGE engine switch or the Message function.

    For more information about the Message function, see section 7.4 "General functions" in OpenText Exstream Design and Production - Logic and Data Help (CCDPLD250100-H-UGD)section 7.4 "General functions" in OpenText Exstream Design and Production - Logic and Data Help (CCDPLD250100-H-UGD).

    SYS_ParentSectionName

    For hierarchical XML data, the name of the parent data segment of the current

    SYS_ParentSectionInstanceNumber

    For hierarchical XML data, the instance number of the parent of the current active data segment

    The ‘SYS_ParentSectionInstanceNumber‘ system variable applies only to data segments within data segments having the same name.

    SYS_RecordCount

    The total number of records read from all of the customer driver files in

    SYS_RecordCountDriver

    The total number of records read from the current customer driver file

    SYS_SegmentIndex

    The absolute index of the current data segment within all instances of segment data, regardless of data segment name

    SYS_SegmentLevelNumber

    For hierarchical XML data, the level within the hierarchy of the current data segment

    SYS_SegmentName

    The name of the record that caused the current data segment to begin

    If no segments are being processed, the value is empty.

    SYS_SegmentNameInstanceNumber

    The instance number of the current data segment within all data segment having the same name

    SYS_SortIndex

    A 20-byte variable used in sort index files for the indexing information.

    The ‘SYS_SortIndex‘ system variable is used only in post-sort processing.

    SYS_SortIndexDocument

    A 15-byte variable used in sort index files to select documents to include and exclude for certain customers

    The ‘SYS_SortIndexDocument‘ system variable is used only in post-sort processing.
