6.2   Exporting DAS resources

    You can use DAS functionality to create export packages that you can use to move resources from one DAS tenant in OpenText Communications 25.1. For example, if you are using multiple tenants to separate resources for development, testing, and production.

    To move resources, you use the Export resource option to create a resource package in the source DAS tenant, and then use the Import option to import resources from that exported package into the destination DAS asset library. DAS export packages can be created from any resource in the asset library. When you export a resource, that resource and all of its dependent resources are packaged into a ZIP file.

    Considerations when exporting DAS resources

    When you export DAS resources, keep the following considerations in mind: 

    • Workflow status of versioned resources—When you export DAS resources, you can choose whether to export the latest approved versions or the latest versions of a selected resource and its dependent resources. During import, the workflow status is retained for all resources.

    • User permissions for locked resources—If a export package contains a resource that is locked, then the import process breaks the lock and imports the resource in an unlocked state so that users in the target tenant can edit it.

    Creating a DAS export package

    1. In the OpenText Communications web client, select the resource that you want to export.

      Tip

      You can select any resource in the asset library views. When the export package is created, that resource and all of its dependent resources are packaged into the ZIP file.

    2. On the General tab in the Properties panel, click the Export resource button.

    3. In the Export dialog box, select a resource export type:

      • Latest approved version—Exports the latest approved versions of the selected resource and all of its dependent resources. If you select this option, and if a latest approved version doesn't exist for a resource to be exported, then an error is displayed.

      • Latest version—Exports the latest versions of the selected resource and all of its dependent resources, regardless of the workflow state. If you select this option, and if the latest version of a resource to be exported is not in an approved state, then the export package will also contain the latest approved version for that resource.

    4. If you want to add security to the export package ZIP file, then select Enable password protection and use the resulting fields to enter and confirm a password.

    5. Click Export.

      A ZIP file containing the exported resources is created and saved based on the default file download behavior for your web browser.
