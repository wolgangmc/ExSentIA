4.13   Creating language objects

    Language objects are resources that allow you to use multiple languages in a design, as well as provide spelling dictionary and accessibility reading language control.

    Creating a new language object

    1. In the Region panel, click Add, then click Language in the selection list.

    2. In the Properties panel, enter a name and, optionally, a description for the language object resource.

      Names do not support the following special characters: & / : * ! ? > < | + # %

    3. Click Create.

      The language object resource is created and the Properties panel displays the language properties.

    4. Use the Properties panel to define the following attributes:

      Property Description
      Spelling dictionary

      The language-specific dictionary to use when creating designs and for spell checking content.

      If the dictionary you want to use is not listed, you can add dictionaries in Tenant management view. This list also includes any custom dictionaries that you install.

      If you select a language that is pre-installed by OpenText Communications, the Days of the week and Months of the year fields are populated with the values found in the dictionary (.dic) file for that language. However, these can be edited, as needed. If you select a language with a custom dictionary, these fields are initially blank, but can be filled manually.

      Accessibility language control type

      Specifies a language for assistive technology tools to use when reading content.

      You can either use a language from the static list or use a variable:

      • Static—In the Accessibility language list, select a language from the default list provided by OpenText Communications.

      • Variable—In the Custom accessibility language list, select a variable that contains a custom ISO language code.

        Tip

        If you use a variable with a custom ISO language code, make sure to verify that your assistive technology tools read the custom code correctly. The engine does not validate the language code.

      Days of the week

      Provides the default names of the days of the week for the selected language.

      The full names are shown in the Long column of text boxes, and the abbreviated names are shown in the Short column of text boxes.

      If you make edits to the content in these lists or are adding content manually, make sure to start with the day corresponding to Sunday in the top text box, and proceed down the list in consecutive order, otherwise the wrong day of the week is used in output.

      Months of the year

      Provides the default names of the months of the year for the selected language.

      The full names are shown in the Long column of text boxes, and the abbreviated names are shown in the Short column of text boxes.

      If you make edits to the content in these lists or are adding content manually, make sure to start with the first month of the year in the top text box, and proceed down the list in consecutive order, otherwise the wrong month of the year is used in output.

    Note

    If you export a language object, any custom dictionaries are not included.

    Using language objects for design variations and data mapping

    When you create a language object, its name is automatically added as a static selection list item in the SYS_LanguageCustomer system variable, which controls which language object to use at engine run time. If the SYS_LanguageCustomer variable is then added as a variation in a communication set, any Designer designs or Content Author templates based on that communication set will include those language objects as a variation option.

    Note

    In order to use SYS_LanguageCustomer in Content Author, the Include for Content Author option needs to be enabled for the variable in the communication set.

    Keep the following recommendations in mind when creating variations using the SYS_LanguageCustomer variable: –

    • If you create a design variation using a language as an attribute, the properties of that language object dictate the spelling dictionary used during editing. If multiple languages are used in a variation, then the default English spelling dictionary is used.

    • In order to ensure you get the correct formatting and language in your output, either map the SYS_LanguageCustomer variable in your data source or use a targeting rule to set the correct values for the variable.

    • If you create a variation using both the SYS_LocaleCustomer and SYS_LanguageCustomer variables, the language setting for both should match. For example, both should be set to English or both to French.

    • If you include a language variation in a design, OpenText strongly recommends that you specify a default locale for the communication. A default locale can be designated from Designer, in Communication Composer, on the General properties panel of the communication.

    For more information on using variations in a communication, see OpenText Communications - Designer User Help (CCCD250100-H-UGD)OpenText Communications - Designer User Help (CCCD250100-H-UGD).

    If you map Date type variables that contain months or days values in a data source, and then use that variable in a communication design that is associated with one of the SYS_LanguageCustomer language variations, the engine will use the Months of the year and Days of the week entries in the appropriate language object. For example, if the data source contains the date “Saturday, Nov. 13, 2021” and is mapped to a date variable, then the output for French customers uses the days and months defined in the French language object as the values for the variable.

    Supported accessibility language codes

    If you want to use a variable as the Accessibility language control type for a language object, you need to create a scalar string variable that contains an ISO 639–1 or ISO 639–2 language code as its value.

    Important

    Keep in mind that accessibility languages are the languages that are available to be read by assistive technology tools. The list of accessibility languages does not represent the languages supported in OpenText Communications output.

    Duplicating a language object

    If you want to create a copy of a language object, you can duplicate it. This might be useful if you want a similar language object with only minor modifications.

    To duplicate a language object, in the Region panel, select the language object that you want to copy and click Duplicate. You can then update the settings for the new language object.
