2.4.7   Setting up orchestration profiles

  • Dashboards view

    You can use a profile to share configuration properties, such as authentication details, between multiple flow context items.

    For example, if you have multiple Azure channels that all use the same credentials, you can create a profile with those credentials and then reference that profile from each of the Azure channels.

    To create a profile:

    1. In the OpenText Communications web client, sign into the following URL as a tenant administrator user:

      https://<host>:<optionalPort>/design?tenant=<tenantName>

    2. To open Tenant management view, click the Tenant management button .

    3. In Tenant management view, click Profiles.

    4. From the domains list, select the domain where you want to add a profile.

    5. Click Add.

    6. On the Properties panel, specify a name and, optionally, a description for the profile.

      Names do not support the following special characters: & / : * ! ? > < | + # %

    7. In the Profile type list, select the type of connection profile you want to create.

    8. Optional To designate the profile as the default for this profile type, select the Default Profile check box. You can designate one default profile for each profile type.

      Note

      The first profile created for each profile type is automatically designated as the default profile; however, after additional profiles of that type are created, you can change which one is assigned as the default profile.

    9. Click Create.

    To configure or edit profile settings:

    1. In the web client, click the Tenant management button .

    2. In Tenant management view, click Profiles.

    3. Click the profile you want to configure.

    4. Click Open.

      The profile opens in Orchestrator.

    5. Enter the settings for the profile.

    6. Click Update.

    7. Click Save.
