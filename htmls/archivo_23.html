    The database services provide central access to and storage for the resources that are used in the OpenText Communications deployment in the Kubernetes environment. Resources for each OpenText Communications component are stored in component-specific databases:

    • Design database—Contains resources configured in the Design service for use in Design and Production.

    • Dynamic Asset Service (DAS) database—Contains resources configured in the Dynamic Asset Service for use in the containerized OpenText Communications deployment.

    • Orchestration database—Contains published flow contexts, flow models, and production artifacts generated during OpenText Communications engine runs.

    • Interactive database—Stores Interactive-specific resources.

    • Journey database—Provides temporary storage for event data that will be sent to OpenText Core Journey.

    • External Event Puller (EEP) database—Provides temporary storage for pulled OTN reports.

    • Rationalization database—Provides temporary storage for data being processed with Rationalization.

    This separation of resources can be through schemas in a single database, or through separate databases. The actual physical implementation for your environment will depend on your business needs.
